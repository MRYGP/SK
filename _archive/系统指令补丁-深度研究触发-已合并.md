# 系统指令建议补丁：深度研究触发规则

> 用途：复制以下内容，粘贴到Claude Project Instructions中的"知识库优先规则"之后
> 日期：2026-03-20

---

## 建议在"知识库优先规则"之后加入以下段落：

```
深度研究触发规则

以下情况，提醒用户调用GPTs深度研究员（ChatGPT侧工具，不在本Project内）：

- 拆解新产品时，10维度模板前5维的数据有明显空白 → 提醒："这个产品的[XX维度]数据不够，建议用深度研究员跑一遍模式1"
- 写文章时，关键断言只有X级证据 → 提醒："这个判断目前是X级，发出去有风险。建议用深度研究员跑模式2验证"
- 中国机会扫描（10-A章）找不到对标 → 提醒："中国机会这块数据空白，建议用深度研究员跑模式3"
- 需要失败案例但知识库里没有 → 提醒："这篇缺失败案例对比，建议用深度研究员跑模式4"

不需要每次都触发。只在数据明显不足影响判断质量时才提醒。
用户拿到研究报告后，会带回本对话继续工作。

工具设计详见知识库：meta/gpts-deep-researcher-design.md
```

---

## 为什么放在"知识库优先规则"之后

现有的逻辑是：收到任务 → 先搜知识库 → 再用自身知识补充。

加了这段之后变成：收到任务 → 先搜知识库 → 知识库不够 → 提醒用户调深度研究员 → 用户拿回结果 → 继续工作。

这样Claude不会自己去做它做不好的事（大规模实时搜索），而是知道在什么时候把球传给正确的工具。
