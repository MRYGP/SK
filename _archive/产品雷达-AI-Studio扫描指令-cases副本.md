## 一、为什么用Google AI Studio

Google AI Studio（Gemini）有三个你现在最需要的能力：

1. **联网搜索**：可以直接访问网页，不需要你手动复制内容进去
2. **免费额度充足**：日常扫描不需要付费
3. **长上下文**：可以一次处理大量信号源内容

跟Claude Project的分工：Claude做深度拆解和写文章，Gemini做信号扫描和初筛。各干各的强项。

## 二、Google AI Studio 系统指令

### 指令A：每日快速扫描（5-10分钟）

在Google AI Studio新建一个对话，粘贴以下系统指令：

```
你是三湘问道的AI产品雷达分析师。

你的任务是：帮我从指定的信号源中发现新的AI产品/工具/公司，做初步筛选，输出结构化结果。

## 你的工作流程

1. 我会给你一批信号源的URL或名称
2. 你去访问这些源，提取最近7天内提到的具体AI产品/工具
3. 对每个产品做初筛打分
4. 输出结构化结果

## 初筛5问（每题0或1分）

Q1 小团队吗？（≤50人得1分，大公司产品或团队规模未知得0分）
Q2 有中国市场机会吗？（能想到一个被忽视的中国用户群得1分）
Q3 跟以下方向相关吗？（AI产品拆解、内容工具、信息处理自动化、AI Agent/AI员工、产品情报系统、小团队做大产品——相关得1分）
Q4 有反直觉的地方吗？（一句话说不清为什么它成功/它的做法跟主流不同——得1分）
Q5 时间窗口还开着吗？（没有巨头已经入场做同样的事 / 中国还没有人在做——得1分）

## 判定规则

≥3分 = 🔴待拆解（重点关注）
1-2分 = 🟡观察（记录但不急着拆）
0分 = 丢弃（不记录）

## 输出格式（严格遵循）

对每个发现的产品，输出以下格式：

---
### [产品名]
- **一句话**：它做什么（不超过20字）
- **官网**：URL
- **团队规模**：已知数字或"未知"
- **融资**：已知信息或"未知"
- **发现来源**：哪个信号源提到的
- **5问打分**：
  - Q1 小团队：0/1（理由）
  - Q2 中国机会：0/1（理由）
  - Q3 方向相关：0/1（理由）
  - Q4 反直觉：0/1（理由）
  - Q5 窗口开着：0/1（理由）
- **总分**：X分
- **判定**：🔴待拆解 / 🟡观察 / 丢弃
- **一句话点评**：为什么值得关注或为什么不值得（你的判断）
---

## 重要规则

- 只提取**具体的AI产品/工具/公司**。忽略：泛泛的行业趋势、大公司常规更新（如OpenAI发新模型）、纯学术论文
- 信息不足打不了分的维度标"？"，不要硬编
- 如果同一个产品被多个信号源提到，标注"⚡强信号"
- 优先关注：小团队（≤50人）做出大成绩的案例、商业模式创新、被忽视的垂直市场
- 每个产品的"一句话点评"要有你自己的判断，不要只复述产品介绍
```

## 三、手动操作SOP

### 每日扫描（5分钟，每天做）

```
步骤1（2分钟）：打开Google AI Studio，发一句话
  "帮我扫描以下信号源的最新内容：
   - https://bensbites.com（最新1期）
   - https://www.producthunt.com/topics/artificial-intelligence（今日前5）
   看看有没有新的AI产品值得关注。"

步骤2（1分钟）：看AI输出
  重点看🔴待拆解和⚡强信号
  其余扫一眼就行

步骤3（2分钟）：手动补充中文平台
  刷一下猫不白（抖音）最新视频标题
  如果有新产品，追加到对话里：
  "猫不白今天推了一个叫XXX的工具，做YYY的。帮我也做一下5问打分。"

步骤4：复制AI输出的表格，粘贴到product-radar.md的「📊本周新发现」区
```

### 每周深度扫描（30分钟，周一做）

```
步骤1（1分钟）：打开Google AI Studio，粘贴指令B的完整内容
  直接发送，等AI跑完

步骤2（10分钟）：审核AI输出
  重点审：
  - 🔴待拆解的产品，打分有没有明显偏差
  - ⚡强信号是不是真的值得关注
  - "本周最值得关注的3个产品"你同不同意

步骤3（5分钟）：手动补充中文平台信号
  回顾本周看到的抖音/小红书/B站内容
  把记下来的产品追加到对话里让AI打分

步骤4（5分钟）：整理输出
  把Part 3的表格复制到product-radar.md
  🔴待拆解的产品同时追加到「🔴待拆解」区

步骤5（5分钟）：决策
  看本周🔴待拆解的产品列表
  决定哪个值得写文章、哪个先观察
  在product-radar.md里标注

步骤6（4分钟）：归档
  把本周的完整AI输出保存为：
  cases/2026/雷达周报/radar-weekly-YYYY-MM-DD.md
```

### 每月垂直扫描（按需，有特定方向想深挖时做）

```
步骤1：选一个垂直方向（如"AI Agent"）
步骤2：从信号源清单里找到对应的垂直信号源URL
步骤3：粘贴指令C + 对应URL，发送
步骤4：审核输出，重点看"赛道全景"和"给三湘问道的建议"
步骤5：如果发现值得拆解的产品，进入正常的拆解流程（场景A）
```

## 四、中文平台的特殊处理

抖音/小红书/B站没有URL可以让AI直接访问。处理方式：

### 方式一：抖音视频→豆包解析

```
步骤1：看到猫不白/Topbook/林亦LYi推荐了一个新AI产品
步骤2：复制视频链接，发给豆包
步骤3：让豆包提取视频里提到的产品名、功能、评价
步骤4：把豆包的输出粘贴到Google AI Studio的雷达对话里
步骤5：让Gemini做5问打分
```

### 方式二：手动记录→批量处理

```
一周内看到中文平台的AI产品，随手记在一个文本文件里。
格式随意：
"猫不白推了XXX，做AI简历的"
"AI工具库推了YYY，AI做合同的，看起来不错"
"科技猎手说ZZZ很好用，AI做数据分析"

周末把文本文件内容全部粘贴到Google AI Studio：
"以下是本周从中文平台收集的AI产品信息，帮我做5问打分和结构化整理。"
```

### 方式三：Google AI Studio直接搜

```
直接在Google AI Studio里问：
"帮我搜索最近一周中国市场新出现的AI产品/工具，
 重点关注：抖音/小红书/B站上被推荐的AI工具、
 36氪/IT桔子上新融资的AI公司、
 即刻社区讨论度高的AI产品。
 用5问打分。"

Gemini有联网能力，能搜到一部分中文内容。
不全，但比完全不搜好。当作补充，不当作主力。
```

## 七、第一次怎么启动

**今天就做（15分钟）：**

1. 打开Google AI Studio
2. 粘贴指令A的系统指令
3. 发一句话："帮我扫描以下3个信号源最近7天的内容，提取新AI产品：
   - https://bensbites.com
   - https://www.producthunt.com/topics/artificial-intelligence
   - https://www.reddit.com/r/Solopreneur/"
4. 看输出结果
5. 如果格式和质量满意 → 下周一正式开始每周深度扫描
6. 如果打分偏差大 → 调整指令中的5问描述，再跑一次

**连续跑2周之后你会知道：**
- Gemini的5问打分准不准（需不需要调Prompt）
- 每周大概能捞到多少个新产品
- 哪个信号源的产出质量最高（可以调整优先级）
- 中文平台的补充流程顺不顺畅

**然后决定：**
- 是继续手动操作
- 还是让实习生B接手做成自动化脚本（Phase 1）

## 八、指令调优备忘

如果发现以下问题，按对应方式调：

| 问题 | 调什么 |
|------|--------|
| 打分太宽松（什么都3分以上） | 在5问里加"严格标准：如果信息不足以判断，默认给0分" |
| 打分太严格（什么都0-1分） | 在5问Q3里扩大相关方向的范围 |
| 产出太多（一次30个以上） | 在指令里加"只输出≥2分的产品，0-1分的只列名字不展开" |
| 产出太少（一次不到5个） | 扩大扫描范围，加入更多信号源URL |
| 重复太多（每周看到同样的产品） | 在指令开头加"以下产品已在库中，请跳过：[列出已有产品名]" |
| 中文产品覆盖不够 | 单独跑一次指令C，方向设为"中国AI产品最新动态" |
