# 公众号文章模板 v1.1

**用途**：写文章时快速参照的结构骨架
**配套**：语言规范、禁用词、证据处理等规则见「公众号写作指南.md」
**字数**：1500-2500字

---

## 标题（三选一）

- 反直觉型：「大家都以为X，但其实Y——[产品名]做对了什么」
- 数字型：「[产品名]：[N]个人做到[$X]，靠的是这个反常识打法」
- 对比型：「[大公司]忽视的[用户群]，被[N]人小团队抓住了」

---

## 开头钩子（150字内）

从读者已知的痛点或误解开始，不从产品介绍开始。

```
[行业/职业]的人都知道[普遍认知]。
但[产品名]的[N]个人做了一件反常识的事：[核心行为]。
结果是[令人意外的结果]。
这篇文章拆解他们怎么做到的。
```

---

## 一、产品是什么（200字）

- 一句话定位：帮[具体用户]解决[具体问题]
- 关键数据（A/B级证据）：用户数 / 收入 / 团队规模
- 不写官网介绍语，写「我用一句话向朋友解释」

---

## 二、反直觉洞察（400字）⭐ 给合伙人看

- 先说「通常大家认为……」，再说「但这个产品的逻辑是……」
- 用具体数据或行为佐证
- 结尾点出为什么这个洞察重要

---

## 三、它怎么做到的（400字）

- 拆解1-2个关键机制（飞轮/定价/获客/技术）
- 有失败对比：「[案例]也试过，但因为[原因]失败了」
- 找不到失败案例就写「目前未找到直接对标失败案例」

---

## 四、时间窗口判断（300字）⭐ 给跟投人看

```
这个窗口，我判断还有[N]个月。
理由是：[1-2个具体信号]。
如果你在[这个方向]，现在是[行动建议]的时候。
```

---

## 五、小团队能学什么（300字）⭐ 给资源伙伴看

```
如果你是1-5人团队，可以学的是：
[打法1]：怎么做，成本多少，预期结果
[打法2]：怎么做，成本多少，预期结果
不建议学的：[什么做法在小团队不适用，为什么]
```

---

## 结尾（100字）

三类人钩子各一句：
```
如果你也在研究[这个方向]，欢迎直接来聊——我们在认真押注这件事。
在[具体场景]里，我们目前最需要的是[X类资源/连接]。
我们判断这个窗口还有[N]个月，理由在文章里，你怎么看？
```

最后：1个读者能在评论区直接回答的具体问题。

---

## 发布前自检

- [ ] 每句话≤30字
- [ ] 专业术语有白话解释
- [ ] 数据标注来源，X级已删除
- [ ] 有≥1个失败案例对比
- [ ] 「能学什么」只写1-10人能做的
- [ ] 无禁用词
- [ ] 结尾有三类人钩子 + 评论引导问题
