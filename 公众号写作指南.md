# 公众号写作指南 v2.0

**适用**：三湘问道·AI应用拆解工作室，所有对外发布的公众号文章
**核心目标**：同时服务三类读者——合伙人、资源伙伴、跟投人

---

## 一、写作身份设定

你不是媒体记者，不是产品经理。
你是一个在AI奇点时代认真押注的人，在向同频者广播信号。

这个身份决定了语气：不是介绍，是判断。不是报道，是押注。

---

## 二、三类读者与对应信号

| 读者 | 他在判断 | 文章要给的信号 | 对应段落 |
|------|---------|--------------|---------|
| 合伙人 | 这个人值不值得押注十年 | 反直觉洞察 + 独立判断框架 | 「反直觉洞察」部分 |
| 资源伙伴 | 有没有合作交集 | 具体赛道判断 + 我们在做什么 | 「小团队能学什么」部分 |
| 跟投人 | 机会是不是真机会 | 押注逻辑 + 时间窗口 + 证据严谨 | 「时间窗口判断」部分 |

**关键**：三类人读同一篇文章，每个核心段落都要给到对应信号。

---

## 三、文章结构规范

### 标题（必须是其中一类）
- **反直觉型**：「大家都以为X，但其实Y——[产品名]做对了什么」
- **数字型**：「[产品名]：[N]个人做到[$X]，靠的是这个反常识打法」
- **对比型**：「[大公司]忽视的[用户群]，被[N]人小团队抓住了」

禁止：「XX大揭秘」「你不知道的XX」「全网最全」类标题

### 开头（150字内）
从读者已知的痛点或行业误解开始，不从产品介绍开始。

**模板**：
```
[行业/职业]的人都知道[普遍认知]。
但[产品名]的[N]个人做了一件反常识的事：[核心行为]。
结果是[令人意外的结果]。
这篇文章拆解他们怎么做到的。
```

### 正文结构（共5段）
1. 产品是什么（200字）：一句话定位 + 关键数据
2. 反直觉洞察（400字）：核心，给合伙人看
3. 它怎么做到的（400字）：机制拆解 + 失败案例对比
4. 时间窗口判断（300字）：押注逻辑，给跟投人看
5. 小团队能学什么（300字）：1-10人能执行，给资源伙伴看

### 结尾（100字，三类人各一句）
```
如果你也在研究[这个方向]，欢迎直接来聊——我们在认真押注这件事。
在[具体场景]里，我们目前最需要的是[X类资源/连接]。
我们判断这个窗口还有[N]个月，理由在文章里，你怎么看？
```
最后加一个评论引导问题（读者能在评论区直接回答的具体问题）。

---

## 四、语言规范

### 必须遵守
- 每句话不超过30字（超过立刻拆开）
- 专业术语必须有括号内白话解释
  - 错：「LTV/CAC比值达到3.2」
  - 对：「用户生命周期价值（LTV）是获客成本（CAC）的3.2倍」
- 数据只用A/B级证据，C级加「预估」，X级直接删除
- 「如果你要做」部分只写1-10人团队能执行的路径

### 禁止用词
- 颠覆、革命、硬核、干货、赋能
- 深度好文、全网最全、强烈推荐
- 可能、也许、感觉（用数据代替模糊判断）
- 「AI将改变XX行业」类空洞断言

### 推荐用法
- 具体数字代替形容词：不说「增长很快」，说「3个月涨了10万用户」
- 对比代替描述：不说「很便宜」，说「$19/月 vs 竞品$99/月」
- 行为代替评价：不说「用户很喜欢」，说「付费续费率82%」

---

## 五、证据使用规范

| 等级 | 可信度 | 使用方式 |
|------|--------|---------|
| A级 | 90%+ | 直接引用，注明来源 |
| B级 | 70-90% | 引用，注明「据XX报告」 |
| C级 | 50-70% | 加「预估」「推断」，不作决策依据 |
| X级 | <50% | **直接删除，不保留** |

---

## 六、失败案例要求

每篇文章必须有1个失败案例对比，放在「它怎么做到的」段落内。

- 找到真实案例：「[公司名]做过类似的，但因为[原因]失败了」
- 找不到：「目前未找到直接对标失败案例，以下为推断风险」
- 禁止：编造失败案例

---

## 七、发布前自检清单

- [ ] 标题是反直觉型/数字型/对比型之一
- [ ] 开头从读者痛点开始，不从产品介绍开始
- [ ] 有1个反直觉核心洞察
- [ ] 有1个失败案例对比（或标注「未找到」）
- [ ] 有时间窗口判断和押注逻辑
- [ ] 「小团队能学什么」只写1-10人能做的
- [ ] 所有数据有来源，X级已删除
- [ ] 每句话不超过30字
- [ ] 专业术语有白话解释
- [ ] 没有禁用词
- [ ] 结尾有三类人钩子 + 评论引导问题
