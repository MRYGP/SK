# 公众号文章模板 v2.0

> v1.1→v2.0变更：从"一个固定模板"变成"写作底线 + 5种结构参考" 原因：30篇规划中至少有5种不同类型的文章，固定模板会把每篇写成一个样 配套：语言规范见「公众号写作指南.md」，经验教训见「公众号内容生产经验手册.md」

## 第一部分：写作底线（每篇必过，不管什么类型）

| #    | 底线                                                       | 来源         |
| ---- | ---------------------------------------------------------- | ------------ |
| 1    | 开头3句话内有一个让人停下来的信息（反直觉/数字/对比/故事） | 教训4        |
| 2    | 至少一组正反对比                                           | 教训1        |
| 3    | 每句话≤30字                                                | 写作指南     |

## 第二部分：5种结构参考

### 类型A：产品拆解（004-007, 012-014等）

> 拆一个具体AI产品。30篇里数量最多的类型。

```
标题：反直觉型 / 数字型 / 对比型（三选一）

开头（150字）：从读者已知的误解开始，不从产品介绍开始
  "所有人都以为[X]。但[产品名]的[N]个人做了[反常识的事]。"

一、产品是什么（200字）
  一句话定位 + 关键数据（A/B级）
  不写官网介绍语，写"我向朋友解释"的版本

二、反直觉洞察（400字）⭐ 给合伙人
  先说通常大家认为…… 再说这个产品的逻辑是……
  用数据或行为佐证

三、它怎么做到的（400字）
  拆1-2个关键机制
  有失败对比

四、时间窗口（300字）⭐ 给跟投人
  窗口还有多久 + 判断依据（具体信号）

五、小团队能学什么（300字）⭐ 给资源伙伴
  1-2个1-10人能执行的打法 + 不建议学的

结尾（100字）：三类人钩子 + 评论引导
```

**按需模块**（不是每篇都要，看"那一刀"需要什么）：

- 财富公式分析
- 推演公式分析
- RRF分析链
- 中国机会判断
- 四路决策

**原则：每次只装一颗子弹。**

### 类型B：工具/公式教学（002, 003等）

> 教读者一个框架或方法。重点是"跑给你看"。

```
开头（150字）：一个问题或痛点
  "为什么[看起来成功的人/公司]其实[没有得到预期结果]？"

一、现象（300字）
  用1-2个案例展示问题
  正反对比（同类型，不同结果）

二、工具/公式（400字）
  介绍框架，用白话解释每一项
  不要超过3个要素（多了记不住）

三、跑给你看（500字）⭐ 核心段
  用一个真实案例当场演示
  最好用已发过的案例（读者有记忆锚点）

四、读者怎么用（300字）
  "如果你要判断[某个事情]，可以这样用这个工具"
  给一个具体的5分钟能做的动作

结尾：预告下一篇 + 评论引导
```

**教训3铁律：工具类文章没有"跑给你看"就是废文。**

------

### 类型C：失败案例/死法图谱（008等）

> 拆多个失败案例，提取共性死因。证明"我不只会夸好的"。

```
开头（150字）：一个触目惊心的数字
  "[N]个AI产品，每个融了千万美金，全死了。"

一、死法1（300字）
  [产品名]：做了什么→为什么死→FM编号
  一句话总结死因

二、死法2（300字）
  同上结构

三、死法3-5（根据篇幅，每个200-300字）

四、共性规律（400字）⭐
  "这5个死法，背后是同一个错误：[一句话]"
  回到推演公式/财富公式做解释

五、小团队怎么避免（300字）
  不是"小团队能学什么"，是"小团队怎么不死"
  具体的检查清单（3条以内）

结尾：评论引导
```

**注意：不是幸灾乐祸。语气是"我们从别人的代价里学到了什么"。**

------

### 类型D：身份/宣言/阶段总结（009, 016, 030等）

> "我是谁""我在做什么""我看到了什么"。不套拆解框架。

```
开头：一个具体的场景或时刻
  不从"我是XX"开始，从一个画面开始
  "那天下午，我盯着屏幕上的[某个东西]，突然意识到[某件事]"

正文：故事驱动，不是论证驱动
  允许暴露困惑和不确定
  理论自然带出，不列清单
  必须包含：我是谁、我在做什么、我在找谁

结尾：明确的行动邀请
  不是"欢迎关注"，是"如果你也在[具体方向]，来聊"
```

**教训4：这类文章放在读者已经通过拆解验证了你的判断力之后。009放在第9篇，不是第1篇。**

------

### 类型E：元分析/规律提炼（010, 022, 026等）

> 从多个已拆解案例中提炼共性规律。最难写的类型。

```
开头（150字）：一个数字或发现
  "拆完[N]个AI产品后，我发现了[N]条所有人都踩的坑"

一、规律1（400字）
  先说规律（一句话）
  再用2-3个已拆案例佐证（读者已经认识这些产品）
  对比：符合规律的成功了，违反的失败了

二、规律2（400字）
  同上结构

三、规律3（400字）
  同上

四、这意味着什么（300字）⭐
  "如果这三条规律成立，那[某个方向]的窗口比大家以为的[大/小]"
  连接到我们的押注

结尾：评论引导（让读者用这些规律去判断某个产品）
```

**关键：元分析的素材全部来自已发布的文章。每个规律至少引用2篇前文。读者感觉"他不是在凑数，背后有一个体系"。**

### 类F：MTP构思招募型（第六种，2026-04-19新增）

> **定义**：把一个还没做的产品构思，写成一篇能让B端资源人举手的公众号文章。
>  **和其他类型最大的区别**：其他类型在讲「已经发生的事」，F型在讲「我想做但还没做的事」，目的是提前验证资源是否存在。
>  **方法论文档**：`core/SKILL-MTP构思招募法.md`（必读，含7段骨架+10条检查清单）

```
标题：表达对象的具体状态，不是构思名称
  参考：「2.5亿中国人，每天照镜子，但没有一个人知道自己到底有多严重」

第1段：一个具体的卡住的人（300字内）
  有名字/年龄/具体生活场景，必须基于真实观察

第2段：所有人都在解错题（200字内）
  现有市场都在解什么题，真正卡住人的题没人解

第3段：这个构思要做的事（200字内）
  不写完整商业计划，只写核心动作一句话

第4段：这个构思能成立的前提（300字）
  3-5个关键假设+每个假设怎么验证——把推演暴露给读者

第5段：我可能踩的坑（2-3个自我攻击）
  这篇MTP和普通内容最大的区别——求被反驳

第6段：我在找什么样的人（极其具体）
  不是「欢迎交流」，是「如果你有X，私信告诉我Y」

第7段：如果没人举手（100字）
  明确承诺：没人举手就放弃+3个月后复盘
```

**核心和A/B/C/D/E类型的区别**：其他五种在讲「已经发生的事」，F型在讲「我尝试了还没做的事」。011是三湘问道的第一个MTP，虽然当时还没有这个名字。

## 第四部分：选类型的决策树

```
这篇文章要：
├── 拆一个具体产品？ → 类型A
├── 教一个工具/公式？ → 类型B
├── 拆多个失败案例？ → 类型C
├── 说"我是谁/我在做什么"？ → 类型D
├── 从多个案例提炼规律？ → 类型E
└── 把一个还没做的构思对外招募B端资源？ → 类型F（必先读SKILL-MTP构思招募法.md）
```

如果一篇文章横跨两个类型（比如"用工具拆一个产品"），选主要目的那个。002的主要目的是教工具（B），不是拆papi酱（A）。
