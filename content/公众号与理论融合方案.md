# 公众号内容与理论融合方案

> 日期：2026-03-22
> 目的：把RRF框架作为30篇内容的"脊柱"，重新设计理论引入节奏
> 原则：不推倒30篇大纲，只做"脊柱植入"

---

## 一、核心逻辑：RRF不是一篇文章，是一条暗线

**错误做法：** 写一篇"我发明了RRF理论"。读者不认识你，不会买账。

**正确做法：** RRF在文章中从不被命名，但读者读完30篇后会发现——这个人每次判断产品生死，用的都是同一条逻辑。这条逻辑会被读者自己总结出来，比你讲给他听更有力。

**具体方法：三层渗透。**

| 层级 | 做什么 | 读者感知 |
|------|--------|---------|
| 第一层（暗线） | 每篇拆解都用RRF判断链做分析，但不说"这是RRF" | "他每次都在问同样的三个问题，好像很管用" |
| 第二层（显化） | 第010篇正式提炼"三个决定生死的问题" | "原来他一直在用这套东西，我现在看懂了" |
| 第三层（展开） | 017-022每篇展开RRF的一个组件 | "每个问题背后都有深层理论支撑" |

## 二、30篇大纲的具体调整

### 第一拳（001-003）：不改

001-003已经成稿，且逻辑自洽：
- 001用实战证明判断力
- 002亮出财富公式（与RRF的F-飞轮中"被留住"相关）
- 003亮出推演公式（通用分析工具）

唯一调整：002在讲"被留住"时，可以轻轻埋一句种子——"为什么有些产品用户来了就走？后面会专门拆。"这为后面的飞轮概念埋伏笔。

### 第二拳（004-008）：加入RRF暗线

每篇拆解用v4.0模板做分析（内部），但输出到公众号时不用"RRF"这个词。

**具体怎么渗透：**

每篇拆解的"反直觉洞察"段落，自然带出RRF三个问题中的一个：

| 编号 | 产品 | 重点带出RRF的哪一层 | 怎么带 |
|------|------|-------------------|--------|
| 004 | Gamma | F-飞轮 | "50人做到21亿估值，不是因为功能多。是因为每个用户做出来的PPT本身就是Gamma的广告——看到的人问'这用什么做的'，然后也来用。这个循环是它的命。" |
| 005 | Lovable | R-规则制定 | "Lovable没融过像样的钱，但它在定义一个规则：AI时代的前端开发应该长什么样。跟它比，其他低代码工具都在旧规则里内卷。" |
| 006 | YOODLI | R-重新定义问题 | "所有人都在做'AI帮你写稿子'。YOODLI做的是'AI帮你练说话'——它没有进入写作的红海，它重新定义了问题。" |
| 007 | Paid | R-重新定义（成本维度） | "传统SaaS按月收费，用户为功能付钱。Paid把问题从'你要不要买这个工具'改成'你愿不愿意为结果付费'——这不是定价创新，是问题本身变了。" |
| 008 | 失败案例专题 | F-飞轮反面（FM015） | "这5个产品死法不同，但有一个共性：增长全靠买量。一旦停止烧钱，用户立刻消失。它们没有自己会转的轮子。" |

**读者到这里的状态：** 已经隐约感觉到"这个人每次都在看三样东西——产品改了什么问题、它在控制什么规则、它有没有自转的增长循环"。但他们说不清楚，只是觉得"这个人的分析角度总是比别人深一层"。

### 第三拳（009-010）：010改为RRF正式亮相

009不改（身份文"50岁重新出发"）。

**010大幅调整——这是RRF的正式亮相：**

原标题：《拆完8个AI产品后，我发现了3条所有人都踩的坑》
→ 不够有力。"踩的坑"是被动的。

**新标题方向：《拆完8个AI产品，我发现判断一个产品能不能活，只需要问3个问题》**

结构：
1. 开头用前8篇的案例做引子——"我拆了Gamma、YOODLI、Paid……每个看起来不同，但我发现它们活着的原因惊人地相似。"
2. 亮出三个问题（不叫RRF，就叫"三个问题"）：
   - 它把问题改了吗？（R-Redefine）
   - 它消失了谁会痛苦？（R-Rule，消失测试）
   - 它有没有自己会转的轮子？（F-Flywheel）
3. 用360 vs 拼多多做"终极对比案例"——两个都改了问题，但一个有轮子一个没有。
4. 结尾：这三个问题，就是我接下来每一篇拆解都会问的。你拿去试试。

**这篇文章是公众号最重要的一篇。** 因为：
- 它把前8篇的暗线显化，读者会有"恍然大悟"的感觉
- 它给后面所有文章提供了"期待框架"——读者知道你每篇都会回答这三个问题，他们会开始自己先猜，再看你的答案
- 它是RRF的"轻量版公开发表"——用案例带出，不讲理论
- 它自带传播力——"判断一个产品能不能活只需要3个问题"这个句式很适合朋友圈分享

### 第四拳（011-016）：每篇拆解都显式回答三个问题

010亮出了框架，011-016的拆解就可以明牌用了。

每篇结构中加一个固定段落：**"三问检验"**——用1-2段话快速回答这个产品在"改问题/定规则/转飞轮"上的表现。不需要每篇都长篇分析，有时一句话就够。

**016（阶段押注宣言）的特殊调整：**

原来只是"哪些方向我们会押注"。
现在可以升级为："用这三个问题过滤了15个产品之后，我们决定押注这些方向——因为它们能回答三个问题中的全部三个。"

这比"我们觉得这个方向有前途"有力100倍。因为读者已经信了你的判断框架，你用框架推出的结论他们更容易接受。

### 第五拳（017-022）：从"散装理论"变成"三个问题的展开"

**这是最大的结构性调整。**

原来：017-022是6篇独立的理论文章，各讲各的。
现在：017-022是"三个问题"的深度展开，每篇深挖一个问题背后的理论。

| 编号 | 原规划 | 新规划（融合RRF） | 对应RRF |
|------|--------|------------------|---------|
| 017 | Loom + 初创企业生存法则 | **"它消失了谁会痛苦？"——从Loom看规则制定者的三重境界** | R-Rule |
| 018 | 拼多多 + 破坏式创新 | **"它把问题改了吗？"——拼多多不是'砍一刀'赢的，是改了一道题** | R-Redefine |
| 019 | 换题思维 | **为什么先驱往往成为先烈？——改对了题，但没来得及定规则** | R→R的转化 |
| 020 | 网络效应 | **"飞轮画得再好看，转一圈如果亏钱，就是绞肉机"——Ofo教训** | F-Flywheel + 财务约束 |
| 021 | 从0到1 | **飞轮基因论：有些公司从出生就没有自转的可能** | F-飞轮基因 |
| 022 | 维度跨越 | **不是你的对手太强，是你在错误的维度上竞争** | R-Redefine（六维度） |

**读者到这里的状态：**
- "三个问题"不再是一个口号，每个问题背后都有一整套理论支撑
- 读者自己已经能用"三个问题"做基本判断了
- 你在读者心中的定位从"拆解博主"升级为"有一套判断体系的人"

### 第六拳（023-030）：不改大方向，微调一处

**025（读者提交的产品拆解）的调整：**

不是你拆给读者看，而是：你给读者一个模板（三个问题的简化版），让他们自己先回答，你再点评。

这就是飞轮在转——读者用你的框架产出内容，你点评后内容回流到公众号，吸引更多读者来用框架。

## 三、理论体系的"一张图"

读者读完30篇后，脑子里应该形成这张图：

```
             判断一个产品能不能活
                    |
        ┌──────────┼──────────┐
        |          |          |
     问题1       问题2       问题3
   它改了什么？ 它消失了    它有没有
              谁痛苦？   自转的轮子？
        |          |          |
   ┌────┴────┐   ┌─┴──┐   ┌──┴────┐
   |         |   |    |   |       |
  破坏式   维度  规则  生存  网络   飞轮
  创新    跨越  制定  法则  效应   基因论
  (018)  (022) (017) (017) (020)  (021)
   |                        |
  换题                    财务约束
  思维                    绞肉机
  (019)                   (020)
```

**工具层（贯穿始终）：**
- 财富公式（002）—— 判断"被看见×被信任×被留住"
- 推演公式（003）—— 跑历史数据做预测
- 失败模式库（008）—— 识别死法

**每个理论都不是孤立的，都是"三个问题"的子弹。**

## 七、给读者的"理论体验路径"

一个读者从001读到030，他的体验是：

**001-003：** "这个人判断力不错，给了我两个工具。"

**004-008：** "每篇拆解都比别人深一层。他好像总在看某些相同的东西，但我说不清是什么。"（暗线在工作）

**009：** "原来他50岁了，跟AI聊了三年。难怪。"

**010：** "啊！原来他一直在问三个问题！我回去看前面的文章，真的每篇都在问这三个。"（恍然大悟时刻——这是最强的"晕"感）

**011-016：** "现在我也开始用这三个问题看产品了。"（读者开始主动使用框架）

**017-022：** "每个问题背后居然有这么深的理论。但他讲的案例我前面都见过，所以秒懂。"

**023-025：** "我自己也拆了一个产品投稿给他。"（飞轮开始转）

**030：** "这个人有一套完整的判断体系。我要跟他一起走。"

**这就是"信号 > 流量"的终极实现——30篇文章，吸引的不是路人，是同频者。**
