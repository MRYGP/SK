# SKILL-问题跃迁触发条件

## 一、问题为什么需要跳出

SK 库现有所有 SKILL 都回答"如何深入某个问题":
- 推演 SOP:深入推演一个想法
- 项目审问清单:深入评估一个项目
- 真实产品体检 SOP:深入拆解一个产品
- AI 研究报告评估法:深入验证一份报告

**这些 SKILL 帮你从一个问题挖到底。但它们都假设"问题本身是对的"**。

如果问题本身错了——比如"AI 销售陪练怎么做" vs 真问题是"该不该做 AI 销售陪练"——再深入也只会越挖越偏。

**问题跃迁 = 停止深入当前问题,跳到更上位的问题**。

## 二、五种触发跃迁的信号

以下任一信号出现,**应该立刻停止深入当前问题,跳到上位问题**:

### 信号 1:深入越久越觉得"哪里不对"

当前问题继续推演,但出现"虽然能算下去,但越算越觉得方向不对"的感觉。
- 真实场景:5/14 拆 11x 越拆越觉得"它的洞察用不到我们身上" → 跳到"两层断层"
- 跃迁动作:问"我深入的这个问题,真的是真问题吗?"

### 信号 2:深入到细节后无法回答"为什么做这件事"

具体动作清晰,但"我为什么要做这件事"反而模糊了。
- 真实场景:5/12 在 13 个场景里筛 AI 销售陪练 vs DCC2026,越筛越说不清"我们为什么要做这个方向"
- 跃迁动作:跳到"我们现在的真问题是什么"

### 信号 3:不同来源给出不一致答案,且都很有道理

四个 AI / 多个专家给的答案不一致,但每个都看起来合理。
- 真实场景:5/14 GPT 推荐 #19 实时记账,Kimi 强烈否决,且各自论据都成立
- 跃迁动作:意识到这不是"谁说得对",是"我用的判据本身不够上位",跳到判据层

### 信号 4:出现"我们好像在绕"的感觉

一直在产出,但产出和"原本想干嘛"距离越来越远。
- 真实场景:5/12-5/14 SK 库高质量产出,但 5/9 锁定的 5 通电话没打
- 跃迁动作:跳到"我现在做的这件事,服务于哪个上位目标"

### 信号 5:被外部信息(GPT / 报告 / 别人意见)推着深入

不是自己决定要深入,是被一个新信息推着深入。
- 真实场景:GPT 给"销售内容生成器"建议,差点被推着深入这个候选
- 跃迁动作:跳到"这件事是不是我真的想做"

## 三、跃迁的三个上位方向

跳出当前问题时,问以下三个上位问题之一:

### 上位 1:真问题是什么(跳到问题定义层)
当前问题 → 真问题
- 例:"AI 销售陪练怎么做" → "我接下来 3 年的真任务是什么"
- 例:"#19 实时记账行不行" → "我对小团队的约束设错了吗"

### 上位 2:判据是什么(跳到判据层)
当前判断 → 用什么判据判断
- 例:"哪个候选最好" → "我用什么标准判断候选"
- 例:"两层断层够不够" → "什么是产品形态原型"

### 上位 3:目标是什么(跳到目标层)
当前动作 → 这个动作服务于什么目标
- 例:"写公众号文章" → "公众号文章服务于什么"
- 例:"扫描 24 个候选" → "扫描的目的是什么"

## 四、跃迁不是放弃,是换战场

**关键区分**:
- 跃迁 = 暂时停止当前问题,跳到上位问题后,带着上位问题的答案再回来
- 放弃 = 把当前问题扔了不管

每次跃迁都应该回得来:
- 跳到上位问题 → 找到上位答案 → 回到原问题 → 看是否还该做 / 怎么做
- 如果上位答案让原问题变得没意义,那是"问题消失",不是放弃

## 五、SK 库已有 SKILL 与本 SKILL 的关系

本 SKILL 是元 SKILL,触发以下行为:
- 触发后,可能跳到「SKILL-坐标系校正判断」(判据层)
- 触发后,可能跳到「SKILL-反漂移检查清单」(防偏离)
- 触发后,可能修订「北极星文档」(目标层)

但本 SKILL 本身不是上述任何一个——它是"什么时候用上述工具"的元判据。

## 六、风险:跃迁滥用 = 永远在跳

如果每次稍有不确定就跃迁,会陷入"永远在元层面飘,从不深入"的陷阱。

**跃迁的合理频率**:
- 一个深度问题(比如一个候选拆解)中,合理跃迁次数 = 1-2 次
- 超过 3 次 = 可能在用跃迁回避深入
- 0 次 = 可能在硬挖错误方向
