# 产品评估决策清单 · v1.0
> 建立日期：2026-04-15
> 调用时机：任何新产品方向进来，先过这张清单，再进入深度推演
> 原则：必须条件一个不满足就停，可以慢慢来的不影响启动

---

## 一、必须满足的条件（🔴 一个不过就不做）

### 1. 单位经济必须能算通
**检验问题**：假设客户只来一次，这门生意能活吗？  
**算法**：单次成本 < 单次收入 × （1 + 合理利润率）  
**反面案例**：熊猫不走（配送成本80元，客单200元，年均1次 → 数学上必死）  
**FM预警**：FM003 伪需求陷阱

### 2. 需求频率 ≥ 月频
**检验问题**：用户多久用一次？停止外部推动后，用户还会回来吗？  
**底线**：至少月频以上。年频是红线，周频是理想。  
**反面案例**：文和友（打卡一次=消费完成）、妙鸭相机（年均拍一次）  
**FM预警**：FM015 无飞轮增长依赖

### 3. 消失测试：用户会痛苦
**检验问题**：这个产品消失了，用户有多痛苦？迁移成本是什么？  
**判断标准**：  
- 用完即走 → 不做  
- 有数据/关系沉淀，走了会痛苦 → 可以做  
**正面案例**：Yoodli（企业方法论灌进去了，重建整套才能走）  
**FM预警**：FM008 竞品快速复制

### 4. 有自转机制，不全靠买量
**检验问题**：如果停止花钱推广，30天后还有新用户进来吗？  
**至少需要一条**：飞轮/口碑传播/数据锁定/社区  
**正面案例**：Gamma（用户产出物自带品牌，看到的人注册）  
**FM预警**：FM001 流量红利消失

### 5. 时间窗口 ≥ 12个月
**检验问题**：国内已有多少竞品？巨头有没有开始做？  
**判断标准**：  
- 窗口已关（竞品成熟/巨头入场）→ 不进  
- 6-12个月 → 快速决策，不能等  
- 12-18个月 → 正常节奏推进  
- 18个月以上 → 优先  
**FM预警**：FM013 时机窗口错过

### 6. 不碰监管红线
**检验问题**：医疗诊断建议/金融投资建议/个人隐私数据——有没有触碰？  
**高风险领域**：AI医疗（需资质）、AI金融（需牌照）、AI教育（未成年人）  
**原则**：先查政策，再做产品。灰色地带不碰。  
**FM预警**：FM009 监管政策变化

## 二、可以慢慢完善的条件（🟢 不完美没关系）

| 条件 | 为什么可以后补 |
|------|-------------|
| 功能完整度 | MVP先做最小化场景，不求全覆盖 |
| 用户规模 | 先找10个愿意付钱的，不求万人 |
| 品牌知名度 | 内容线可以边做产品边建，不是先决条件 |
| 技术精度 | MVP阶段允许半人工半自动 |
| 变现模式细节 | 先验证有人愿意付钱，定价可以迭代 |
| 团队规模 | 2-3人先跑通验证，再扩人 |

## 三、复制的三个层次

不是复制越像越好，而是叠加越多维度越有竞争壁垒。

### 层次1：直接复制（大多数人在做的）
看到海外产品 → 做中文版 / 中国版  
**竞争优势**：几乎没有。3个月后出现10个竞品。  
**示例**：看到MYHAIR AI → 做中国脱发诊断AI

### 层次2：横向复制（跨赛道应用模式）
提取产品的核心模式 → 应用到相邻赛道  
**竞争优势**：赛道不同，暂时领先，但模式同样可以被复制  
**示例**：MYHAIR AI的「拍照→AI个性化诊断」模式 → 皮肤AI、指甲AI、植物诊断AI

### 层次3：模式叠加（目标层）
把多个成功产品的换题模式叠加到一个新产品上  
**竞争优势**：组合罕见，壁垒是模式叠加，不是单一功能  
**示例**：MYHAIR AI诊断模式 + Yoodli「帮用户练」模式 = 皮肤健康训练AI

## 四、4个可复用的换题模式（从案例库提炼）

> 这4个产品不是要被直接复制，而是提炼出了可叠加到任何产品的「底层模式」

### 模式P1：产出物即分发（来自Gamma）
**本质**：用户核心产出物天然携带品牌，自动带来下一个用户。  
**叠加到**：任何用户会把产出物发给别人的场景（报告/诊断结果/设计稿/分析）  
**叠加后的壁垒**：增长不花钱，越转越快  
**触发问题**：用户做出来的东西，他会发给谁？发出去之后，看到的人会想用吗？

**P1适用边界（2026-04-17从Gamma重拆补全）**：

| 适用条件 | 不适用条件 |
|---------|----------|
| 用户核心任务 = 「把东西发给别人」（文档/报告/设计/诊断结果/分析） | 用户核心任务 = 「自己用」（健身记录/日记/私人财务管理） |
| 产出物被收件人看到时，发送者已经证明了产品价值 | 产出物是私密的/不被别人看到的（密码管理/个人日历） |
| 品牌露出不干扰收件人使用（角落/底部/非核心位置） | 品牌露出妨碍收件人（如水印遮住主要内容） |
| 看到产出物的人也有同样的需求（做报告/做演示/做问卷） | 看到产出物的人与制作者需求完全不同 |
| Freemium可行（免费版让品牌出现，付费版可以去掉） | 产品本身没有免费层（付费才能使用→品牌曝光机会少） |

**典型适用**：演示文稿工具（Gamma）、问卷工具（问卷星）、设计工具（Canva）、AI诊断报告工具、AI分析报告工具、合同/法律AI工具  
**典型不适用**：个人健康追踪、密码管理、个人财务管理、内部工具（不对外发送产出物）

**对Cactus的提示**：用户核心行为=写论文→论文最终会被提交/发表，P1有机会；但学术规范不接受AI标注，P1在Cactus可能不适用，更可能是P3（帮用户练）

### 模式P2：门外汉上场（来自Lovable）
**本质**：不是让高手更快，而是让被现有工具排除在外的人第一次进来。  
**叠加到**：任何现在需要专家才能做的领域（法律/医疗/财税/专业分析）  
**叠加后的壁垒**：天花板更高，竞争对手不在同一赛道  
**触发问题**：这个领域里，哪些人现在完全没有资格上场？

### 模式P3：帮用户练（来自Yoodli）
**本质**：从替用户做，到帮用户练。用户的能力是沉淀，不是消耗品。  
**叠加到**：任何「知道≠会做」的场景（沟通/销售/护肤/运动/语言）  
**叠加后的壁垒**：用户越练越离不开，切换成本是自己练出来的能力  
**触发问题**：用户用完这个产品，他比之前更强了吗？还是更依赖了？

**P3适用边界（2026-04-17从Yoodli重拆补全）**：

| 适用条件 | 不适用条件 |
|---------|----------|
| 用户「知道理论」但「用不出来」的鸿沟真实存在 | 用户只需要信息（知道了就够用了） |
| 练习场景可以被模拟（有角色可扮演/有场景可还原） | 技能依赖物理存在（外科手术，无法完全模拟） |
| 用户有动机重复练习（频次月级以上） | 一次性任务，练了就用（如婚礼致辞，无需月频） |
| 进步可被量化（有评分维度） | 进步无法量化（如纯创意能力的提升） |

**P3 vs P1的本质区别**：
- P1（Gamma）：用户走了，裂变链断了（产品损失）
- P3（Yoodli）：用户走了，用户自己损失（沉没成本是自己的成长记录）
- P3的壁垒是对用户的尊重，而非锁死——这是比P1更高质量的护城河

**典型适用**：企业销售培训/考公面试练习/外语口语/运动技术训练（部分）  
**典型不适用**：个人健康追踪（只需信息）/密码管理/一次性创作任务

**对Cactus的提示**：追问「学生是知道写作结构用不出来（P3）还是根本不知道怎么写（P2）」；学术规范不接受AI标注可能让P1不适用；Cactus是低压迭代型练习，与Yoodli高压即时型不同，评分维度能否建立是关键。

### 模式P4：激励对齐（来自Paid）
**本质**：重新设计收费方式，让卖方利益和买方结果方向一致。  
**叠加到**：任何现在按时间/功能收费但价值模糊的专业服务  
**叠加后的壁垒**：差异化在商业模式本身，竞品很难跟进  
**触发问题**：你的产品做得越好，你赚得越多吗？如果不是，激励结构错了。

## 五、使用流程

```
新产品方向进来
    ↓
Step 1：过6个必须条件（有一个不过 → 停，换方向）
    ↓
Step 2：判断复制层次（你在哪一层？目标到层次3）
    ↓
Step 3：叠加换题模式（P1/P2/P3/P4能叠加哪几个？）
    ↓
Step 4：进入深度推演（SKILL-推演SOP-v1.3 + AI产品复制推演框架）
    ↓
Step 5：红队攻击（SK框架红队法，至少3轮自我攻击）
    ↓
Step 6：MVP设计
```
