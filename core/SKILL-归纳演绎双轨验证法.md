# SKILL · 归纳演绎双轨验证法

## 一、核心命题

**单轨跑的判断都不可靠**。

- 只跑案例归纳(轨 A) → 幸存者偏差 + 归因谬误("亚马逊卖书成功 = 我也卖标准化品类")
- 只跑第一性原理(轨 B) → 真空推导 + 忽视人性摩擦("从物理推导最优解是磁悬浮")
- 两轨并行但不校验 → 确认偏误(选案例时已经被原理预设,推原理时已经被案例污染)

**只有两轨独立跑、强制碰撞、能共存的结论,才值得下注。**

## 二、五步操作

### 步骤 1·轨 A 独立跑(案例归纳)

- 找 **3 个成功案例 + 3 个失败案例**(强制对称,防幸存者偏差)
- 用 `core/SKILL-案例研究五步法` 拆解每个案例:
  1. 当时的天时(基础设施状态)
  2. 创始人发现的断层(卡住的人 + 场景)
  3. 第一动作是什么(最小切口)
  4. 什么条件让这个动作有效
  5. 什么条件缺失时这个动作失效(反事实)
- **关键约束**:拆解时不问"这对我有什么启发",只记录事实

### 步骤 2·轨 B 独立跑(第一性原理)

- 问三个归零问题:
  - **物理层归零**:这个需求的物理本质是什么?(出行的本质是位移,不是车)
  - **人性层归零**:用户做这个决策时,系统 1 在怕什么?系统 2 在算什么?
  - **经济层归零**:如果摩擦力归零,最优解长什么样?
- **关键约束**:禁用类比。每用一个"这就像当年的亚马逊",就引入了一个未经验证的假设

### 步骤 3·强制碰撞(两轨结论对照)

| 对照项 | 轨 A 结论 | 轨 B 结论 | 状态 |
|---|---|---|---|
| 切口选择 | | | ⚠️ 冲突 / ✅ 一致 |
| 核心能力 | | | |
| 时机判断 | | | |

**冲突是价值所在,不是失败**。冲突意味着:
- 轨 A 遗漏了关键案例(样本偏差)
- 或轨 B 遗漏了人性/制度摩擦力(真空推导)
- 或两轨都对但适用边界不同

### 步骤 4·用 D030 错位母模型解释冲突

冲突出现时,过五层错位:
- **对象错位**:案例服务的人群和我场景的人群是否不同?
- **阶段错配**:案例时基础设施状态和现在是否不同?
- **标尺错配**:案例用"规模"标尺赢,我是否该用"速度"标尺?
- **语义错位**:案例的"好产品"和我场景用户的"好产品"是否同一含义?
- **身份错位**:案例创始人是"资源型",我是否是"洞察型"?

### 步骤 5·输出"带边界条件的决策"

不是"两轨一致 → 做",而是:

> "在条件 A、B、C 成立时,轨 A 和轨 B 都指向 X 方向;
> 如果条件 D 不成立,轨 B 的推导失效,应退回轨 A 的备选方案 Y。"

## 三、必须避免的死法

### 死法 1·只跑轨 A
看到亚马逊卖书成功 = 我也卖书。但同时代有 100 个卖书网站都死了。
归纳只告诉你"成功者做了什么",不告诉你"失败者是不是也做了同样的事"。

### 死法 2·只跑轨 B
从物理层推导最优解 = 磁悬浮。但推不出市场采用最优。
D029 客户采用六因子里的认知预演和物理嵌入,是第一性原理推不出来的人性摩擦力。

### 死法 3·假双轨(确认偏误)
潜意识选支持轨 B 的案例 + 潜意识让轨 A 案例暗示轨 B 推导 = 双向污染。
**判别**:如果两轨结论"惊人地一致",反而要警惕——可能是污染。

## 四、与 SK 库现有资产的整合

| 现有资产 | 在双轨中的角色 |
|---|---|
| SKILL-案例研究五步法 | 轨 A 的执行手册 |
| DOC-D022 重新定义问题 | 轨 B 的归零起点 |
| DOC-D029 客户采用六因子 | 轨 B 推导后的人性摩擦力校验 |
| DOC-D030 错位母模型 | 两轨冲突时的解释层 |
| core/failure_modes.yml | 轨 A 必须包含失败案例的强制来源 |
| SKILL-问题跃迁触发条件 | 两轨持续冲突 → 触发"是否问题本身错了"的跃迁 |

## 五、核心使用场景

- 评估候选项目(该不该做)
- 推演产品方向(怎么做)
- 复盘失败/卡顿(为什么做不成)
- **任何怀疑自己被 AI 框架推演带跑的时候**

## 六、一句话

**案例告诉你"什么条件下什么动作有效",第一性原理告诉你"条件归零时最优解是什么"。
双轨不是互相印证,是互相找茬。只有两轨打架后还能共存的结论,才值得下注。**
