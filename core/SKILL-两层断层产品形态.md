# SKILL-两层断层产品形态

## 一、什么是两层断层

在"高决策成本 + 信息不对称 + 决策频次不高"的市场里,用户从"有需求"到"买对"中间有两层断层:

**断层一 · 供给可达性断层**
用户已经知道要什么,但不知道哪个供给最适合自己,或者最适合的供给在物理上 / 信息上不可达。

**断层二 · 认知翻译断层**
用户面对多个候选供给时,无法把抽象的功能 / 参数 / 条款翻译成"对我意味着什么",所以无法做出有信心的对比和选择。

## 二、判断两层断层是否存在的三个条件

1. **决策成本足够高**:错选后果不可忽略(健康/资金/时间不可逆消耗)
2. **信息不对称足够深**:供给方掌握的专业语言,用户原生不具备
3. **决策频次足够低**:用户没有重复学习机会,每次决策都是"第一次"

三条都成立 → 两层断层存在
任意一条不成立 → 不适用本形态

## 四、形态选择:做断层一 vs 做断层二 vs 做两层

- **只做断层一**:本质是匹配 / 渠道生意,如美团 / 58 同城。门槛在流量和供给覆盖,小团队难入。
- **只做断层二**:本质是教育 / 内容生意,如丁香医生 / 小红书评测。门槛在内容产能和持续性,容易被信息平台收编。
- **做两层叠加**:本质是诊断 + 翻译,如 MYHAIR AI / 植发 MTP / 口腔笑容预演。**这是 SK 库目前主要押注的位置**。两层叠加的护城河来自"诊断能力 + 翻译能力"双重壁垒,且天然带"中立角色"位置(详见 2026-04-25 推演流水账:诊断 vs 匹配 vs 三方错位)。

## 六、和"寄生 / 嵌入"判据的关系

两层断层产品天然适合寄生:

- **寄生在已识别的增长曲线上**:GLP-1 爆发 → 减肥两层断层;低空经济成型 → 飞行培训两层断层;AI 扩散 → AI 工具选型两层断层
- **寄生在已成型的信任入口上**:寄生在医院 / 公立机构 / 专业 KOL 的信任流量,不自建权威
- **寄生在已建好的支付链上**:不自建支付 / 履约,接现有支付链

寄生位置选错 → 两层断层产品会退化为内容生意或匹配生意。

## 七、使用流程

遇到一个候选方向 / 朋友带来的项目 / 新行业扫描,按本 SKILL 跑三步:

**步骤一:三条件自检**
该市场是否同时满足"决策成本高 + 信息不对称深 + 决策频次低"?三条全 YES 才继续,任一 NO 该方向不适用本形态。

**步骤二:断层定位**
该方向的断层一(供给可达)和断层二(认知翻译)各自有多严重?哪一层已被现有方案覆盖?哪一层是空的?

**步骤三:寄生位置确认**
打算寄生在哪条已识别增长曲线 / 信任入口 / 支付链上?寄生位置若不清楚,本方向退回观察池。

三步全过 → 进入项目审问清单 Part A-D
任一不过 → 不适用,不浪费产能。
