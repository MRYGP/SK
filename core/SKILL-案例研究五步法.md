# SKILL-案例研究五步法

> 版本：v1.0 · 2026-05-03
> 来源：三湘问道·AI应用拆解工作室 中华饮食记忆品牌出海项目实践沉淀
> 适用范围：所有领域的成功/失败案例研究，不限于AI产品拆解
> 与SK现有工具的关系：本文件是 `product-teardown-template.md` 中"成功/失败机制"章节的上游工具；每次发现新案例时先跑五步法，再决定是否值得做完整10维度拆解

---

## 为什么案例研究容易走偏——三个陷阱

**陷阱1：把"它做到了"当成"我也能做到"（幸存者偏差）**

Etsy展示的是活下来的店铺，不展示死掉的。你看到的成功案例，是从数万个同类尝试中存活下来的那1%。
解法：每找一个成功案例，必须同时找一个同类失败案例对照。只看成功不看失败，等于只读了一半地图。

**陷阱2：抄表面不抄引擎**

看到竹编米勺月销高，就去做竹编——抄的是品类，不是"制作过程视频+低客单价+TikTok算法"的增长引擎。品类本身没有护城河，引擎才有。
解法：每个成功案例必须找到"增长引擎"——是什么让它卖出去的？不是产品本身，是机制。

**陷阱3：把C级经验当A级决策依据**

大量出海案例是自媒体二手传播，数字未经验证。"某中国女生独立站年营收$2.86M"——找不到实名店铺，就是X级，不能进财务模型。
解法：只用A/B级数据做关键决策；C级数据只用来激发方向，不用来做财务预测。

## 五步法（每次遇到新案例时执行）

### Step 1：身份核查（2分钟）

**问**：这个案例是真实可查的吗？

检查清单：
- 有没有店铺名/品牌名/创始人名？（"某越南团队"≠真实案例）
- 数据来源是官方平台还是自媒体转载？
- 核查工具：直接搜Etsy店铺名 / Google品牌名 / LinkedIn创始人

**通过标准**：有具体可查的主体名称，数据至少B级（可信媒体/平台可见）

**结果判断**：
- 通过 → 进Step 2
- 不通过 → 标X级，降权处理（可用于方向激发，不进财务模型或强结论）

**注意**：找不到≠不存在。是"我没查到"，不是"它没发生"。

---

### Step 2：增长引擎提炼（5分钟）

**问**：它是靠什么卖出去的？

填空句（必须填完，填不完说明案例研究不够深）：

> 它的增长靠的是 ___（内容平台/口碑传播/Etsy SEO/B2B背书/节日礼品），
> 触发了买家的 ___（情感/猎奇/身份认同/功能需求/仪式感），
> 在 ___（Etsy/独立站/TikTok Shop/线下）完成转化，
> 复购靠的是 ___（系列化/节日/口碑/会员/定制）。

**判断**：
- 如果"增长靠的是"填的是"产品好"或"价格低"→ 没有提炼到引擎层，继续挖
- 如果"完成转化"填的是TikTok，但产品页在Shopify → 要核查漏斗衔接，这是常见死亡点

**本步输出**：一句话描述增长引擎（例："靠Etsy自然搜索流量+情感叙事包装+华人节礼场景触发转化"）

---

### Step 3：失败模式匹配（3分钟）

**问**：这个案例有没有踩过SK失败模式？

对照 `core/failure_modes.yml` 逐条检查，重点看：
- FM003 伪需求：用户真的愿意付钱吗？还是只是"看起来有需求"？
- FM006 平台依赖：它的生存是否完全依赖某一个平台的规则？
- FM008 快速复制：如果它成功了，义乌/山寨版多快能跟进？
- FM009 监管政策变化：有没有已经发生的政策变化会影响它的模式？

**判断**：
- 成功案例踩了FM风险但仍然成功 → 标注"存在但被控制的风险"，说明它用什么机制绕开了
- 失败案例 → 标注对应FM编号，提炼"死亡触发条件"（不是死亡原因，是具体触发了什么）

**注意**：FM009（监管政策变化）特别容易被忽略，因为研究者不知道"已经发生了什么"。
出海类项目：必须专门核查近12个月的政策变化（关税/平台准入/外汇/目标市场法规）。

---

### Step 4：可迁移性评估（5分钟）

**问**：这个案例对我们有多大参考价值？

评分矩阵：

| 维度 | 相似度（高/中/低） | 说明 |
|------|-----------------|------|
| 产品形态 | | 同是实物摆件/文化产品/内容品牌？ |
| 目标用户 | | 同样面向海外华人/文化爱好者/礼品市场？ |
| 平台渠道 | | 同样用Etsy/独立站/TikTok？ |
| 团队规模 | | 同样是1-3人小团队？ |
| 启动资金 | | 同样是<10万人民币级别？ |
| 增长引擎 | | 引擎类型相同（内容驱动/SEO/B2B背书）？ |

评分规则：
- 4个以上"高" → 强参考，增长引擎和失败模式可以直接借用
- 2-3个"高" → 中等参考，抽取核心机制，不照搬执行细节
- 1个以下"高" → 仅作方向参考，不做具体动作依据

---

### Step 5：提炼行动动作（3分钟）

**问**：这个案例给我们的下一步是什么？

**格式**（必须是具体动作，不能是原则）：

> [案例名] → [可抄的具体机制] → [我们执行时的调整] → [验证方式和时间节点]

示例：
> FakeFoodWorkshop → 找机构合作换背书 → 联系1个澳大利亚华人文化中心提供免费样品 → 30天内是否有回复和展示机会

**反例（不合格的动作）**：
- "学习FakeFoodWorkshop的成功经验" → 太抽象，没有可执行性
- "做好产品质量" → 是原则，不是动作

## 快速判断：这个案例值不值得深挖？

满足以下任一条 → 立刻深挖（优先级P0）：
- 直接竞品出现"月销估算>50件"的新信号
- 发现有创始人面孔+真实故事的同类文化品牌
- 出现大规模平台封号/新政策变化的信号

满足以下条件 → 记录备用（优先级P1）：
- 相似文化品类（非中餐）在Etsy的新爆款
- 亚裔文化周边品牌的新产品线或定价变化

以下情况 → 存档不深挖（优先级P2）：
- 其他亚洲国家文化产品的泛案例，无具体数据
- 没有可核实数据的自媒体出海故事

## 持续案例研究的节奏

**每周：竞品监控（30分钟）**

搜索关键词（根据项目调整）：
- Etsy关键词：竞品品类的核心词（用eRank查搜索量）
- Reddit：r/Etsy、r/Entrepreneur 新帖
- 小红书：出海/Etsy卖家相关内容

记录内容：
- 新上架listing/价格变化/评论数增长
- 有没有新玩家进入？已有玩家有没有爆款信号？

**每月：案例库更新（2小时）**

每月最后一天，扫描：
- Etsy发现页（按类目浏览，看新店铺）
- 近期出海媒体报道
- AI深度研究工具跑一轮竞品扫描

发现新案例 → 用五步法处理 → 符合条件则追加到案例库

**每季度：深度案例拆解（半天）**

每季度选1个最相关竞品或对标，做完整SK10维度拆解。
输出：案例拆解底稿（存入 `cases/2026/深度底稿/`）

## 案例→决策的正确路径

```
发现案例
  ↓
Step 1 身份核查（2分钟）
  ↓ 通过
Step 2 增长引擎提炼（5分钟）
  ↓
Step 3 FM失败模式匹配（3分钟）
  ↓
Step 4 可迁移性评估（5分钟）
  ↓ 强参考（4个以上"高"）
Step 5 提炼行动动作（3分钟）
  ↓
写入案例库 + 加入下周执行清单
```

如果可迁移性评估是"中等参考"或"仅方向参考"：
→ 提炼核心机制，不照搬执行细节
→ 存入案例库备用，不进入本周执行清单

## 关键教训（来自2026-05-03出海项目实践）

**教训1：最重要的信息往往是"已经发生的政策变化"**

8个AI研究工具同时跑，只有1个发现了美国De minimis小包免税已于2025年5月取消。这个已发生的政策变化推翻了整个毛利模型。教训：Step 3的FM009（监管政策变化）检查，对出海类项目是最高优先级，不是走形式。

**教训2：报告最漂亮的来源给出了最危险的建议**

Gemini给出的结论最乐观、格式最完整，但关键数字建立在错误的税务假设上，建议"直接切入"。数据等级不能靠"看起来专业"判断，要看来源和可核实性。

**教训3：深度>广度**

KIMI一份深度报告（50KB）抵过Grok+Perplexity+GPT三份（合计约25KB）的决策价值。报告数量≠研究质量。
