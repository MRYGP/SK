# SKILL · 两库配对找方向法

## 四条前置认知（防止这套方法变成哲学创业）

1. **方向常是"跑出来的"，不是"算出来的"**。本方法只负责选一个有复利土壤的切入点，不负责预知十年后的大机会。大机会在积累中涌现（路径B原则3：破坏式/颠覆式机会是迭代涌现的，不是预设的）。亚马逊的 AWS 是卖货中跑出来的，不是第一天算出来的。

2. **"大机会"= 切入点积累的东西（数据/能力/关系）会复利、能长出比切口更大的东西**。判断的是"土壤"，不是"果实"。禁止用"我觉得这是十年大机会"推演断言，必须有已验证样本佐证（某真实公司已在更大尺度上跑通这个结构）。戴一下"=卖书"即此理：假发本身不一定大，但"AI修复视觉决策认知断层"的能力会复利。

3. **要的不是"总量增长的市场"，是"结构性增长"**。"哪个市场增长"是公开信息，人人能查，从它出发=到达和所有人一样的红海。真正要找的是：某个"门外的人"刚刚因为某个技术翻转第一次能被服务（拼多多=五环外人群因智能机普及第一次能网购；Anthropic=企业刚开始需要可靠可控能进合规的AI）。即 ljkl 坐标系轴三：什么刚翻转？谁是门外的人？

4. **大机会被卡住，先分清"等技术型"还是"缺卖书型"**。
   - 等技术型：障碍是算力/数据/技术不成熟 → 只能等，现在动手也是白烧钱。
   - 缺卖书型：技术其实够用，障碍是还没找到"卖书"（一个现在就成立的切入点）→ **可立即启动**。
   关键动作不是等条件成熟，是**找那个卖书，用卖书把自己熬到条件成熟**——卖书在赚钱（活着）、在攒数据（等数据够）、在等算力变便宜（等技术够）。亚马逊1994年算力远不够做"卖一切+云"，但他没等，他做卖书，用卖书熬到了AWS。
   **卖书的硬标准**：必须是"AI出现之前就一直有人为它付钱、AI只是做得更好"的真实传统需求（永恒需求型），不是AI造出来的新鲜感需求（新鲜感会衰减，养不活你熬到大机会那天）。
   判法：面对一个大机会愿景，先问"它是缺技术还是缺卖书？"——缺卖书型才去找切入点配对；纯等技术型，记录进观察池，不投入。

## 两库的分工（各缺一半，必须配对）

- **ljkl 122家/26格** = 大机会的结构（哪个格子美国满/中国空），缺"小团队切入点"——它的高增长幸存者几乎全在明牌-技术锁大战场，小团队进不去。
- **SK ~100产品** = 切入点的动作（怎么找没服务的人/怎么嵌工作流/那一刀），缺"大机会"——多是小团队能做的小生意，天花板有限。
- **配对** = 用 ljkl 找结构 × 用 SK 找动作 × 重构到中国左下腹地（巨头不屑弯腰捡）。这是拼多多式"抄结构×抄动作×换战场"。

## 配对四步（一次只做一个格子，禁止批量覆盖）

1. **选格子**：选一个 ljkl"美国满/中国空"的大格子，优先选有"结构性增长+门外人"的（用前置认知3判断，不是用总量增速）。
2. **找切入点（走场景K）**：找这个大机会在中国的"左下切入点"——巨头不屑、中国空、小团队能进场、且是真实漏斗里的真实卡点。数据>推演，禁止凭空设计切入点。
3. **配 SK 动作**：在 SK 产品库已拆过的产品里，找哪个的底层动作（不是表面形态）能用在这个切入点上。
4. **过审问清单**：项目审问清单 问4（复制×换地两步筛）+ 问5（切入点×大机会双层检验）。两关都过才值得推演。

## 禁止（每条都是今天验证过会死的）

- 禁止排序200家选第一名抄（覆盖式 + 抄形态，Yoodli/考公已验证会死）
- 禁止从"哪个市场增长"出发（公开信息=红海）
- 禁止凭推演给大格子配切入点（哲学创业最舒服的温床——前置认知2的防滑钉在此）
- 禁止一次处理多个格子（违背"扫描为承诺，不为覆盖"）
- 禁止把"切入点能活"当终点（那只是菜地/求生存层，要同时看复利土壤）
