### 6b) 飞轮检验（v4.0新增，参考DOC-D022）

> 飞轮 = 增长的自我强化循环。漏斗 = 需要不断注入资源的消耗。
> 360只有漏斗（买量→变现→流失→再买量），拼多多有飞轮（更多用户→更多农户→更低价格→更多用户）。

**飞轮四条件检验**：

| 条件 | 问题 | 本产品情况 |
|------|------|-----------|
| ①多边参与 | 除了平台和用户，还有哪些参与方？它们之间是否相互依赖？ | |
| ②核心价值随规模强化 | 规模扩大时核心卖点是变强还是稀释？还是越差（如广告骚扰）？ | |
| ③退出成本递增 | 用户/参与方用得越久，离开的代价是否越大？具体是什么代价？ | |
| ④单圈正收益 | 飞轮转一圈，经济模型是正的还是负的？LTV/CAC趋势如何？（如果转一圈亏钱，飞轮越快死越快——Ofo/WeWork教训） | |

**飞轮判断**：
- [ ] 真飞轮（四条件全满足）
- [ ] 半飞轮（满足1-3条，有改进空间）
- [ ] 绞肉机（①②③满足但④不满足——转得越快亏得越多）
- [ ] 无飞轮/纯漏斗（需持续买量或补贴维持增长）

### 9b) RRF分析链（v4.0新增，参考DOC-D022 v2.1）

> RRF = Redefine（重新定义问题）+ Rule（规则制定）+ Flywheel（飞轮构建）
> 这三步决定了一个产品的天花板：只做了R的是一次性爆款，做到RR的能站稳，做到RRF的能持续增长。
>
> ❗ 边界提醒：RRF适用于平台型/网络型商业模式。纯硬科技、专业服务、强监管行业、Pre-PMF阶段、消费硬件不适用。如本产品不在适用范围，写"本RRF不适用，原因：___"并跳过本节。

**R — 重新定义问题**

这个产品重新定义了什么问题？

| 维度 | 行业原来的定义 | 这个产品的新定义 |
|------|--------------|----------------|
| 用户 | 原来服务___ | 现在服务___ |
| 价值 | 原来解决___ | 现在解决___ |
| 边界 | 原来是___ | 现在变成___ |
| 时间 | 原来是___ | 现在变成___ |
| 成本 | 原来___付费 | 现在___付费 |
| 竞争 | 原来的对手是___ | 现在的对手是___ |

重新定义了几个维度？___个。
- 0-1个：优化，不是颠覆
- 2-3个：有颠覆潜力
- 4个以上：范式转移级别

**检验句**：这个产品的成功，是否让原来的行业领导者"看起来像在做蠢事"？
→ 是 / 否。如果"否"，它可能只是在优化，不是在重新定义。

**R — 规则制定**

它是规则套利者还是规则制定者？

- **套利信号**：封闭控制、依赖单一变现渠道、没有培育第三方生态
- **制定信号**：开放能力/API/标准、有第三方深度依赖它生存、生态在自发扩张

判断：套利者 / 制定者 / 过渡中

如果是制定者，在哪个层次？
- [ ] L1 流量规则（控制入口/分配注意力）
- [ ] L2 商业基础设施规则（供应链/交易/分配标准）
- [ ] L3 技术标准规则（协议/架构/接口）

**"消失测试"**：如果这个产品明天消失，谁会痛苦？
→ 没人痛苦 = 套利者。有人的生意依赖它 = 制定者。

**AI产品专用——依赖度检验（v2.0新增）：**
如果底层模型提供商（OpenAI/Anthropic/Google）明天把你的核心功能内置化，你还剩什么？
→ 什么都不剩 = 二阶套利者（如Jasper AI）
→ 剩工作流锁定/行业数据/用户关系 = 有防御性（如Harvey/Cursor）

**F — 飞轮**

（已在第6b章完成，此处引用结论即可）

飞轮判断：真飞轮 / 半飞轮 / 无飞轮

**RRF综合判断**：

| 维度 | 状态 | 意味着什么 |
|------|------|-----------|
| R-重新定义 | ✅/❌ | |
| R-规则制定 | 套利/制定/过渡 | |
| F-飞轮 | 真/半/无 | |

**产品段位判断**：
- RRF全满（含第④条） = 有成为伟大公司的基因（拼多多、亚马逊）
- RRF全满但第④条不满 = 绞肉机，越快越危险（Ofo、WeWork）
- 有R无RF = 一次性爆款风险（360、Jasper AI）
- 有RF无第一个R = 做得好但不是颠覆者（守成型公司）
- 全无 = 不值得拆解（或不在RRF适用范围）
