### 9b) RRF分析链（v4.0新增，参考DOC-D022）

> RRF = Redefine（重新定义问题）+ Rule（规则制定）+ Flywheel（飞轮构建）
> 这三步决定了一个产品的天花板：只做了R的是一次性爆款，做到RR的能站稳，做到RRF的能持续增长。

**R — 重新定义问题**

这个产品重新定义了什么问题？

| 维度 | 行业原来的定义 | 这个产品的新定义 |
|------|--------------|----------------|
| 用户 | 原来服务___ | 现在服务___ |
| 价值 | 原来解决___ | 现在解决___ |
| 边界 | 原来是___ | 现在变成___ |
| 时间 | 原来是___ | 现在变成___ |
| 成本 | 原来___付费 | 现在___付费 |
| 竞争 | 原来的对手是___ | 现在的对手是___ |

重新定义了几个维度？___个。
- 0-1个：优化，不是颠覆
- 2-3个：有颠覆潜力
- 4个以上：范式转移级别

**检验句**：这个产品的成功，是否让原来的行业领导者"看起来像在做蠢事"？
→ 是 / 否。如果"否"，它可能只是在优化，不是在重新定义。

**R — 规则制定**

它是规则套利者还是规则制定者？

- **套利信号**：封闭控制、依赖单一变现渠道、没有培育第三方生态
- **制定信号**：开放能力/API/标准、有第三方深度依赖它生存、生态在自发扩张

判断：套利者 / 制定者 / 过渡中

如果是制定者，在哪个层次？
- [ ] L1 流量规则（控制入口/分配注意力）
- [ ] L2 商业基础设施规则（供应链/交易/分配标准）
- [ ] L3 技术标准规则（协议/架构/接口）

**"消失测试"**：如果这个产品明天消失，谁会痛苦？
→ 没人痛苦 = 套利者。有人的生意依赖它 = 制定者。

|------|------|-----------|
| R-重新定义 | ✅/❌ | |
