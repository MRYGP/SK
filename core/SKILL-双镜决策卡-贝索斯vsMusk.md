# 双镜决策卡 · 贝索斯 vs Musk

## 怎么用
对任一候选方向，左右各跑一遍，最后只回答中间那个"拍板问题"。
不是为了"哪个镜子对"，是为了用两个相反尺度逼出方向的真实软肋。

## 左镜 · Musk（看成本/技术结构）
1. 第一性成本被重做了吗？（重做了底层成本/物理结构 还是只套了AI壳？）
2. 有没有别人造不出的"硬件级的件"（技术/成本硬护城河）？
3. 能否垂直整合、拥有关键栈？
4. 零数据点不烧钱——有没有真实物理数据点？
5. 窄切口→横向复制的必然性通路？
- Musk 红旗：撮合市场、无硬护城河、靠叙事自我说服、TAM 不够大。

## 右镜 · 贝索斯（看需求不变量/平台通路）
1. 押的是十年不变的人性需求吗？（不变量）
2. 是不是"卖书"式切口——低货值/高频/可标准化/能积累用户与数据？
3. 现金流能否自造血、熬十年（Day1/长期主义）？
4. 是否从客户痛点倒推（顾客痴迷），而非从供给侧聪明想法出发？
5. 决策可逆吗（双向门先低成本试错）？
- 贝索斯红旗：一单一单手工定制做不大、从供给侧出发、切口与终局搞反次序。

## 中间 · 必须自己拍板的那个问题
> 这个方向的护城河，到底来自【成本/技术壁垒】还是【时间+情感+数据沉淀的需求不变量壁垒】？
- 信前者 → Musk 赢，做有硬护城河的窄品类引擎。
- 信后者 → 贝索斯赢，按永恒需求型慢熬心智/数据壁垒（对应项目审问清单 C问题5②补·永恒需求型天花板）。

## 两镜的共识（可信度最高，先做这三条）
1. 现有那份精致 BP/形态，两个人都会推翻——别先优化 BP。
2. 先拿真实数据点再放大——零数据点的 BP 等于零。
3. 切口与终局分离——做最小可行切口、心里携带终局、绝不一上来做平台。

## SK 阶段警告（5-04 铁律）
Musk/贝索斯的选择尺度多为阶段三（永久资本期）判据，SK 是阶段一。
只借两人的思考方式（剥第一性 / 押不变量 / 先拿数据 / 切口终局分离），
不借其项目尺度（万亿TAM门槛 / 多线并进 / 靠叙事融资 / 慢熬十年）。
