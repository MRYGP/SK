# SKILL-视觉决策商品的 AI 试戴层

## 一、什么是视觉决策商品

**核心定义**:
> 决策维度是视觉(好不好看 / 适不适合我),但购买路径是文字 + 他人示范(评价 / 模特图 / 商家描述)。决策维度和购买路径之间存在系统性错位。

**典型样本**:
- 服装(对着镜子穿才知道好不好看 / 但购买靠模特图)
- 假发(戴上才知道适合 / 但购买靠盲选)
- 眼镜(戴上才知道脸型 / 但购买靠他人推荐)
- 美甲(做完才知道喜不喜欢 / 但选款靠图片)
- 化妆品色号(画上才知道适合 / 但购买靠 swatch)
- 家具(摆放才知道空间感 / 但购买靠 3D 渲染)
- 装修风格(完工才知道效果 / 但决策靠效果图)
- 美容医美(做完才知道适合 / 但决策靠案例图)

**反例(不是视觉决策商品)**:
- 食品(味觉决策,AI 试戴不适用)
- 电子产品(功能决策)
- 书籍(认知决策)
- 虚拟商品(无实体)

## 二、两层断层的具体形态

视觉决策商品的两层断层(对应 SKILL-两层断层产品形态):

**断层一·供给可达性**:
- 用户不知道哪款适合自己
- 海量 SKU 中无从选起
- 现状解法:看模特图 + 评价 + 赌

**断层二·认知翻译**:
- 用户脑内的"用了之后的样子"不可见
- 现状解法:买回家试,不行就退

**1.5 层断层(本 SKILL 新发现)·场景到 SKU 的不可见匹配**:
- 用户知道场景(婚礼 / 约会 / 通勤),但不知道场景对应哪款 SKU
- 这一层在两层断层之间,负责"从场景需求翻译到具体产品"
- **传统电商完全没有覆盖**(只按品类/价格筛选,不按场景筛选)
- AI 时代可以填补:输入"周五参加婚礼" → AI 推荐 + 试戴

## 三、AI 试戴层的产品形态

**最简形态**:
1. 用户上传一张自己的照片
2. 用户选择 / 输入场景或款式
3. AI 生成"用了之后的样子"

**完整形态**:
1. 用户场景输入("我下周要参加婚礼,想要庄重款")
2. AI 推荐 3-5 个候选 SKU(场景到 SKU 匹配 = 1.5 层断层)
3. AI 生成"我戴每款的样子"(断层二解决)
4. 用户对比 → 选定 → 跳转购买(断层一解决)

## 四、护城河:什么是真的护城河,什么不是

**❌ 不是护城河**:
- AI 视觉生成能力(GPT-5o / Sora 2 / Veo 4 会让这件事通用化)
- 用户上传照片的速度(任何产品都能做)
- UI 设计(可复制)

**✅ 真护城河**:
- **场景到 SKU 的匹配数据库**(每次用户试戴 → 购买 → 反馈,累积成"什么人 × 什么场景 × 什么 SKU 的最优解"数据)
- **特定人群的信任关系**(化疗患者 / 严重脱发用户等高焦虑人群,一旦建立信任,不轻易迁移)
- **合作商家的独家供货**(头部 SKU 的独家上线权)

## 七、典型对照:理发店 AI 试戴

理发店 AI 试戴和假发 AI 试戴的对照:

| 维度 | 理发店 | 假发 |
|---|---|---|
| 决策核心 | 剪完之后好不好看 | 戴上之后好不好看 |
| 不可逆性 | 高(长不回来) | 低(可退) |
| 现场感 | 强(可调整) | 弱(全靠想象) |
| AI 试戴渗透 | 已开始(2024-2025) | 几乎零(2026) |
| 试戴-成交转化 | 30-50%(行业数据) | 未知,待验证 |

**核心洞察**:**两者底层是同一种焦虑——决策的不可逆 + 视觉不可见**。理发店已经验证了用户对 AI 试戴的接受度,假发只是同形态的另一应用,**心理教育成本极低**。

## 八、使用流程

遇到一个候选方向时,先过本 SKILL 的三问:

1. **是不是视觉决策商品?**(决策维度是视觉,但购买路径是文字 + 他人示范)
2. **两层断层是否成立?**(决策成本高 + 信息不对称深 + 决策频次低)
3. **AI 试戴层是否空缺?**(已有玩家做到什么程度?)

三条全过 → 这是 AI 试戴层的机会
任一不过 → 不是本 SKILL 适用范围

## 九、风险

**最大风险:被基座模型 / 大平台吞噬**
- GPT-5o / Sora 2 / Veo 4 会让"试戴生成"成为通用能力
- 抖音 / 淘宝 / 拼多多可能内置 AI 试戴
- 应对:**护城河不在生成能力,在场景匹配数据库**

**次大风险:用户不愿意上传照片**
- 隐私顾虑 + 外貌焦虑
- 应对:可以做"模特库 + 脸型匹配"的弱版本,牺牲个性化换隐私

**第三风险:试戴效果差导致信任崩塌**
- 试戴生成的图和实物差距大 → 用户感到被骗
- 应对:在 demo 阶段必须做大量真实测试,确保试戴质量

## 十、待验证

- 假发 demo 上线后,用户上传照片的意愿率到底有多少(预测 30-50%,需验证)
- 化疗患者 vs 严重脱发用户,谁更愿意上传?(预测化疗患者更愿意,需验证)
- 试戴-购买转化率(预测 5-15%,需验证)
- 1.5 层断层(场景到 SKU 匹配)的实际付费意愿(目前未知)
