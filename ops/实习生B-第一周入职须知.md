## 你在做什么

我们在做一个老年AI管家产品的MVP（最小可用版本）。

一句话：**让老年人在微信里加一个"好友"，发语音就能跟AI聊天。**

MVP只做三件事：
1. 自由对话（老人说什么AI答什么，记住对话历史）
2. AI早安图（每天早上自动发一张个性化问候图）
3. AI写真（老人发照片，AI返回艺术风格版本）

你的任务：**4周内把这三个功能做出来，给10个真实老人用。**

不需要做App。不需要做精美界面。不需要考虑10万人同时用的情况。微信公众号对话就是全部载体。

## 第一周任务拆解

### Day 1-2：基础框架

**目标：用户在公众号发一条消息 → 你的服务器收到 → 调用大模型 → 返回回复给用户**

技术步骤：
```
1. 搭建一个HTTP服务（FastAPI/Flask），部署到云上
2. 配置微信公众号的服务器URL指向你的服务
3. 实现微信消息接收（解析XML）
4. 实现消息转发到大模型API
5. 实现大模型回复转发回微信
6. 处理语音消息：微信语音→语音识别（微信自带或调ASR API）→文字→大模型→文字回复
```

**Day 2结束时的验收标准**：
- [ ] 你在公众号里发一条文字消息，5秒内收到AI回复 ✅
- [ ] 你发一条语音消息，10秒内收到AI文字回复 ✅
- [ ] 回复内容像人话，不是乱码或报错 ✅

### Day 3-4：对话记忆

**目标：AI记住每个用户说过的话**

技术方案（MVP够用）：
```
每个用户一个JSON文件（或数据库记录）
格式：
{
  "user_id": "openid_xxx",
  "conversations": [
    {"role": "user", "content": "今天天气怎么样", "time": "2026-03-25 08:30"},
    {"role": "assistant", "content": "今天长沙晴天...", "time": "2026-03-25 08:30"}
  ]
}

每次用户发消息：
1. 读取该用户的历史对话
2. 把最近20条对话 + 新消息一起发给大模型
3. 大模型回复后，追加到历史记录
```

**为什么只保留最近20条**：大模型有上下文长度限制，而且太长会变慢变贵。20条够MVP用了。

**Day 4结束时的验收标准**：
- [ ] 你上午说"我叫张三"，下午问"我叫什么"，AI能答对 ✅
- [ ] 换一个微信号测试，AI不会把两个人的对话搞混 ✅

### Day 5：AI人设提示词

**目标：让AI像一个耐心的朋友，不像客服**

这是最重要的一步。提示词决定了AI的"性格"。

**基础人设提示词**（你在这个基础上调）：

```
你是一位温暖、耐心、有生活智慧的朋友。
你跟一位老人在微信上聊天。

你的风格：
- 说话简短，每句话不超过30个字
- 用口语，不用书面语。说"挺好的"不说"非常好"
- 会主动关心对方，但不啰嗦
- 对方说了开心的事，你真心替他高兴
- 对方说了烦心事，你先听，不急着给建议
- 偶尔用表情（😊🌞👍），但不多用
- 记住对方说过的事，下次聊天时提到

你不做的事：
- 不说"作为AI，我..."
- 不用"您"，用"你"
- 不讲大道理
- 不在一条消息里说超过3句话
- 不推销任何东西
```

**Day 5做的事**：
1. 把这个提示词配进系统
2. 模拟10种老人可能发的消息，测试AI回复是否自然：
   - "今天天气真好"
   - "我孙子考上大学了"
   - "最近膝盖有点疼"
   - "帮我看看这个保健品是不是骗人的"
   - "你是机器人吗"
   - 发一条语音（随便说什么）
   - 发一张照片
   - "你能帮我做什么"
   - "无聊"
   - "我想我老伴了"（敏感话题测试）

3. 每条回复打分（1-5分），不自然的调提示词，再测

## 第二周任务

### Day 1-2：早安图功能

**逻辑**：
```
每天早上8:00 → 遍历所有关注用户 → 给每个人发一张图 + 一句问候语

问候语个性化：
- 如果知道用户名字："张叔，早上好！今天天气不错，出去走走？"
- 如果不知道："早上好！新的一天，心情好最重要 😊"

图片来源（按难度选一个）：
- 简单版：准备30张早安图模板，每天随机选一张（实习生A提供素材）
- 进阶版：调用图片生成API，每天生成一张（DALL-E/通义万相/文心一格）
```

**建议MVP先用简单版**——30张图够用一个月了。把精力省给AI写真功能。

### Day 3-5：AI写真功能

**逻辑**：
```
用户发一张照片 → 识别为图片消息 → 调用图片风格迁移API → 返回艺术风格版

风格选择（MVP做1-2种就够）：
- 油画风格
- 水墨风格
- 卡通风格

技术方案（选一个）：
- 通义万相（阿里，有风格迁移API，中文生态好）
- Midjourney API（质量最好，但需要第三方代理，成本较高）
- Stable Diffusion + ControlNet（开源，需要GPU服务器）
- 文心一格（百度，有免费额度）

建议MVP用通义万相或文心一格——有免费额度，接口简单。
```

**Day 5结束时的验收标准**：
- [ ] 发一张自拍照，30秒内收到油画风格版 ✅
- [ ] 图片质量不变形、不恐怖、老人看了会说"哦！好看" ✅
- [ ] 发文字消息还是正常对话，不会触发写真功能 ✅

## 记住一件事

**你在做的不是一个技术项目，是在验证一个假设。**

假设是：老人愿不愿意跟AI对话？对话后觉不觉得好？

技术是手段。如果4周后10个老人每天主动跟AI聊好几轮，这个假设就验证了。如果没人来，我们需要换方向。

所以比"代码漂亮"更重要的是"老人用起来自然"。提示词的打磨可能比架构设计更影响结果。
