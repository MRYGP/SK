## 字段说明

| 字段 | 说明 |
|------|------|
| MTP编号 | 顺序编号，MTP-001起 |
| 对应文章 | 知识库里的文件名 |
| 发布日期 | 公众号实际发布日期（不是写完日期） |
| 目标B端资源人 | 这篇MTP在找什么样的人（一句话） |
| 举手数 | 截止当前收到的有效举手人数 |
| 举手人身份摘要 | 举手人的职业/位置/能提供什么资源（不写姓名） |
| 后续对接 | 是否约了通话/见面，进展如何 |
| 复盘截止 | 必须发复盘文章的最晚日期（发布日+3个月） |
| 3个月复盘结论 | 做/不做/转型，以及原因 |

## 使用纪律

1. **举手≠成功**：举手只是入口，必须在举手后7天内完成第一次对话
2. **0举手=强信号**：不是失败，是市场在告诉你这个方向的资源问题没有解法，比花6个月做了才发现要好得多
3. **复盘不可跳过**：不管举手多少，3个月后必须回到公众号写复盘——这是MTP机制的核心，没有复盘MTP就退化成了普通内容
4. **复盘模板**：
   - 有举手 → 「MTP-00X之后我聊了X个人，下一步是Y」
   - 0举手 → 「MTP-00X之后没人举手。这个方向放弃。这是我学到的3件事：1/2/3」
