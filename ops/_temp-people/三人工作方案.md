## 三、MVP线：老年AI管家

### MVP目标（第一个月）

**只验证一个假设：老人愿不愿意跟AI对话，对话后觉不觉得好。**

不做完整产品。做一个最小的东西，让10个老人试用2周。

### MVP形态

**微信公众号对话**——老人关注公众号，发语音或文字，AI回复。

为什么选公众号不选其他：
- 老人已经会用微信，不需要学新东西
- 公众号有官方API，合规
- 搭建成本最低，一个人一周能做出来

### MVP只做三件事

| 功能 | 描述 | 技术实现 |
|------|------|---------|
| **自由对话** | 老人说什么AI答什么，记住对话历史 | 微信公众号API + 大模型API（Claude/GPT） + 对话记忆存储 |
| **AI早安图** | 每天早上自动发一张个性化问候图 | 定时任务 + 图片生成API |
| **AI写真** | 老人发一张照片，AI生成艺术风格写真 | 图片上传 + 风格迁移API |

### 实习生B的任务（按周拆解）

**第一周：技术选型+环境搭建**

| 天 | 任务 | 产出 |
|----|------|------|
| Day 1-2 | 注册微信公众号（服务号），申请API接口权限 | 公众号就绪 |
| Day 1-2 | 调研大模型API方案：Claude API vs GPT API vs 国内模型（豆包/Kimi） | 选型报告（1页：哪个模型、多少钱、怎么接） |
| Day 3-4 | 搭建基础框架：用户发消息 → 调用大模型 → 返回回复 | 能跑通"发一条消息，收到AI回复" |
| Day 5 | 加对话记忆：每个用户的对话历史存储（简单的文件/数据库） | 用户A发的消息，AI下次还记得 |

**第二周：三个功能逐一实现**

| 天 | 任务 | 产出 |
|----|------|------|
| Day 1-2 | 自由对话的提示词优化——AI的"人设"要像一个耐心的朋友，不是客服 | 发10条不同类型的消息，AI回复都自然 |
| Day 3 | AI早安图功能——每天8点自动给关注用户发一张图 | 定时任务跑通 |
| Day 4-5 | AI写真功能——用户发照片，返回艺术风格版本 | 发一张照片，30秒内收到写真 |

**第三周：内部测试+修bug**

| 天 | 任务 |
|----|------|
| Day 1-2 | 杨+两个实习生自己先用3天，找问题 |
| Day 3-4 | 修bug、优化回复速度、调提示词 |
| Day 5 | 让3-5个非团队成员试用（实习生的家人、朋友的父母） |

**第四周：找10个真实老人测试**

| 天 | 任务 |
|----|------|
| Day 1 | 杨联系那个老年俱乐部，安排10个老人试用 |

| Day 3-14 | 观察数据：每个人每天发几条消息？发了什么？AI回复质量怎么样？有没有人主动发朋友圈？ |

实习生A不写代码，但负责**微信公众号的非技术部分**：

## 七、给实习生的"你需要知道的事"

### 给实习生A（内容运营）

**你的核心KPI**：
- 每篇文章发布后2小时内完成所有冷启动动作
- 评论区零漏回
- 每周五交数据周报

**你需要读的文件**（读一遍就够）：
- `公众号写作指南.md`——知道文章的质量标准
- `001-launch-plan.md`——知道冷启动的完整方案
- `research-wechat-visibility.md`——知道公众号算法怎么运作

**你不需要做的**：
- 不需要写文章（杨写）
- 不需要决定选题（杨决定）
- 不需要跟读者深度交流（有价值的私信转给杨）

### 给实习生B（产品开发）

**你的核心KPI**：
- 第二周结束时，MVP三个功能全部能跑
- 第三周结束时，内部测试通过，没有阻断性bug
- 第四周结束时，10个老人能正常使用

**你需要读的文件**（认真读）：
- `product-design-draft-v0.2.md`——产品设计文档，理解我们在做什么
- `north-star-elderly-ai.md`——北极星，知道方向

**你需要做技术决策的几个点**：
1. 用哪个大模型API？（成本、中文能力、响应速度权衡）
2. 对话记忆怎么存？（简单的JSON文件就够MVP用）
3. 图片生成用什么API？（Midjourney/DALL-E/国内方案）

**你不需要做的**：
- 不需要考虑扩展性（MVP只给10个人用）
- 不需要做精美UI（公众号对话就够）
- 不需要做付费系统（第一个月全免费）

## 八、一个月后的检查点

**内容线**：
- 001-003发布完毕，004在写 ✅/❌
- 新关注数 ≥ 50 ✅/❌
- 有效私信（三类读者之一）≥ 5 ✅/❌
- 多平台至少铺了知乎+即刻 ✅/❌

**MVP线**：
- 10个老人完成了2周测试 ✅/❌
- 核心假设验证结果：老人愿不愿意跟AI对话？ → 回答
- 日均对话条数 → 数字
- 有没有人主动分享给朋友？ → 有/没有

**根据结果决定下一步**：
- 如果老人愿意用 → 扩大到50人，开始设计付费模式
- 如果老人不愿意 → 分析原因，调整方向或放弃
- 如果内容线跑得好、MVP线还没ready → 先集中精力在内容线
- 如果内容线和MVP线都跑通了 → 内容里开始写"我们自己在做的方向"（对应大纲第015篇）
