## 零、先理解你在做什么

### 你的工作不是"帮老板发文章"

你的工作是**在文章发出去的前2小时内，确保对的人看到它并点了推荐**。这2小时的动作决定了这篇文章是活还是死。

### 公众号的流量机制（必须知道）

```
文章发出去
    ↓
系统先推给几百人看（初始流量池）
    ↓
这批人反应好（看完了、点推荐、评论了）
    ↓
系统推给更多人（几千人）→ 继续好 → 继续推 → 越来越多人看到
    ↓
反应不好 → 文章沉底，不再推了
```

**前2小时决定生死。** 系统看的是刚发出去那段时间的数据。

### "推荐"是最重要的按钮

读者点文章底部的"推荐"（♡），这篇文章就出现在他微信好友的信息流里。一个人点了推荐，他的几百个好友都可能看到。而且这些好友跟他是同类人——做AI的人推荐的文章，他的朋友圈也是做AI的人。比系统随机推荐精准100倍。

### 新号有30天"新手保护期"

系统会给新号额外的流量倾斜。**这30天极其宝贵。** 只发AI产品拆解相关的内容，不发其他的，不断更。

## 工作线1：文章流水线

### 完整流程

```
杨写好底稿，发给你
    ↓
第1步：你标注底稿（30分钟）
    ↓
第2步：你发给写作GPTs改写（10分钟操作+等结果）
    ↓
第3步：你检查改写稿（15分钟）
    ↓
第4步：你发给读者GPTs检测（10分钟操作+等结果）
    ↓
第5步：你把检测结果+改写稿发给杨审核（等杨回复）
    ↓
第6步：杨确认OK → 你排版+发布+冷启动+分发
    ↓
第7步：你记录数据，整理反馈
```

---

### 第1步：标注底稿

**标【圣物】：** 找到那些"读到时让你停了一下"的句子——短、尖锐、有画面感。前面加 `【圣物】`。

```
【圣物】这不是炫耀。这是求生。
【圣物】买的不是健康，是"被重视"。
```

**标【锚点】：** 找有冲击力的数字，加 `【锚点】`。

```
【锚点】巅峰期月活2亿
【锚点】单场直播销售额破2.67亿
```

标完后存副本：`XXX-底稿-已标注.md`

---

### 第2步：发给写作GPTs改写

打开ChatGPT → 找到"三湘问道·写作工坊"GPTs，发送：

```
这是第[XXX]篇，类型是[拆解型/工具型/死法型/身份型/元分析型]。
在30篇规划里的角色是：[这篇的定位]。
那一刀是：[最核心的判断——拿不准问杨]

底稿里标了【圣物】的句子不能改措辞，只能调位置。
底稿里标了【锚点】的数字至少保留2-3个。

以下是底稿全文：
[粘贴底稿]
```

---

### 第3步：检查改写稿

**立即检查4项**（不通过就打回）：

| # | 检查项 | 不通过怎么办 |
|---|--------|------------|
| 1 | 【圣物】句子还在吗？ | 告诉GPTs"以下句子被删/改了，请恢复原文" |
| 2 | 【锚点】数字还在吗？ | 同上 |
| 3 | 结尾三元素在吗？（评论问题+下篇预告+系列标识） | 告诉GPTs"缺了XX，请补上" |
| 4 | 字数在3000以内？ | 告诉GPTs"超了XX字，请砍到3000以内" |

---

### 第4步：发给读者GPTs检测

打开ChatGPT → "三湘问道·第一读者"GPTs → 粘贴改写稿全文。

重点看：A类风险有没有、截图句几个（少于3个是大问题）、长度测试、作者vs读者比例。

---

### 第5步：提交杨审核

**用这个模板发给杨：**

```
杨哥，[XXX]改写稿+检测结果出来了。

【改写稿】
[粘贴全文]

【我的检查】
- 圣物句子：全部保留 / XX句被改了（附对比）
- 锚点数字：保留了X个
- 结尾三元素：齐全
- 字数：XXXX字

【读者GPTs检测摘要】
- A类风险：无 / 有（列出）
- 截图句：X个（列出）
- 传播力：一句话
- 最尖锐的问题：[抄GPTs给的]

【我的建议】
- 可以直接发 / 建议改XX

等你确认。
```

杨回复"OK" → 进第6步。

### 第6步：排版+发布+冷启动

#### 发布前（提前2小时）

```
□ 用编辑器排版（标题、小标题、引用块、加粗、分隔线）
□ 检查错别字、格式、图片
□ 预览发给杨确认
□ 杨说OK → 准备发布
```

#### 发布（20:30）

```
□ 点发送
□ 打开文章链接确认发布成功
```

#### 发布后0-5分钟：朋友圈

```
□ 杨的号发一条（用杨写好的文案 + 文章链接）
□ 你自己的号也发一条（如果愿意）
□ 截图存档
```

**为什么发朋友圈？** 不是"宣传"，是给系统喂第一口数据。你的微信好友会直接点进来看，他们的行为（看没看完、有没有点推荐）直接决定系统给不给你更多流量。

#### 发布后5-15分钟：群发

```
□ 发到3-5个微信群
□ 每个群用不同引导语（不要发一模一样的话）
```

**群发模板**（直接复制修改）：

AI/创业类群：
> 写了一篇AI产品拆解，发现了一件挺反直觉的事——[文章里最吸引人的一句话]。5分钟读完，想听听你们的看法。[链接]

朋友/同行群：
> 花了几个月研究的东西，终于写出来了。帮我看看，觉得有道理就帮我点个"推荐"，对新号特别重要。[链接]

行业资源群：
> 最近在研究[行业]方向，写了一篇深度分析。如果你也在关注这个方向，聊聊。[链接]

**为什么群发？** 朋友圈能触达的人有限。微信群可以一次触达几百人。选群标准：群里有我们的目标读者（做AI的人、创业者、投资人）。

#### 发布后15-30分钟：私信

```
□ 私信10个种子用户（杨给名单）
□ 模板：[称呼]，写了一篇关于[主题]的拆解，5分钟能读完。
  如果觉得有意思帮我点个"推荐"，对新号冷启动特别重要。[链接]
```

**为什么私信请"点推荐"？** 10个人里5个点推荐，每人200好友 = 1000个潜在读者。如果这1000人里50个又点推荐，50×200 = 10000。这就是裂变。"看了"对系统没用，"点了推荐"才触发裂变。

#### 发布后30分钟-2小时：评论区

```
□ 发杨写的第一条评论
□ 之后每一条读者评论都要回复
□ 回复风格：真诚、简短、有内容。不要"谢谢关注"
□ 拿不准的截图问杨
```

**为什么回复评论？** 评论越多，系统越认为这是好文章。每回复一条，对方收到通知，可能再次打开文章——又给系统一次"有人在看"的信号。

## 冷启动效果怎么看

不看阅读量（新号一开始一定低，正常）。看这三个：

| 指标 | 好的信号 | 不好的信号 |
|------|---------|-----------|
| 推荐数 | ≥10个 | < 5个 |
| 评论数 | ≥5条 | 0条 |
| 私信 | 有人主动聊内容 | 只有客气话 |

前5篇每篇带来5-10个真正感兴趣的关注者，到第10篇就有50个种子用户。50个种子用户的推荐能力已经很强了。

## 常见问题

**Q：写作GPTs改写后金句全没了？**
直接告诉它"以下标了【圣物】的句子被改了，请恢复原文"，把原句粘贴给它。

**Q：读者GPTs标了A类风险？**
不要自己判断。截图发给杨，让杨决定。

**Q：雷达连续一周没新发现？**
正常。不要为了凑日报降低标准。写"无新发现"就行。

**Q：评论区有人喷？**
事实性质疑截图问杨。纯喷不回复。

**Q：群发会不会让人觉得烦？**
如果发的是"帮我转发谢谢"会烦。如果发的是"写了一篇拆解，发现了一件事——所有AI产品都在做错同一步"，这有信息量，不烦。

**Q：杨的底稿我看不懂？**
不需要全看懂。你的工作是标注、走流水线、检查、发布。内容判断是杨的事。但如果觉得"读者肯定看不懂"的地方，提交时告诉杨。
