## 一、你负责的三件事

### 事1：信号源更新（每周）

信号源是我们发现值得拆解的AI产品的来源。它会过时，需要定期维护。

**文件位置**：`D:\sk\radar\signals.md`（三梯队 + 五垂直方向 + 中文平台）

**每周做**：
- 扫一遍第一梯队信号源，判断有没有失效的（账号停更、内容质量下降）
- 有新发现的好信号源 → 填入对应梯队，写一行说明"看什么、多久更新一次"
- 每月末做一次全量review：哪些质量下滑可以降级，哪些新源可以升级

**判断标准**：
- 升级到第一梯队：每天有内容、命中率高（每5篇至少1篇有价值）、有独立判断不只是转发
- 降级或删除：连续2周没有有价值的内容，或已停更

**交付**：每次更新后在微信群说一句"信号源已更新，新增XX，删除XX"。不需要写报告。

---

### 事2：产品雷达扫描（每周）

扫描全球新AI产品，找值得关注的，存入产品库。

**核心文件**：
- `D:\sk\radar\product-radar.md`——产品库主文件（唯一真相）
- `D:\sk\radar\产品雷达-AI-Studio扫描指令.md`——扫描指令v1.2（直接粘贴进AI Studio用）
- `D:\sk\cases\2026\产品对标库-38个AI产品复制价值排名.md`——38个产品打分表

**每周操作流程**：
```
1. 打开信号源（Ben's Bites / Product Hunt / @levelsio等第一梯队）
2. 发现新产品 → 过初筛5问（在product-radar.md里）
3. ≥3分 → 填入🔴待拆解区
   1-2分 → 填入🟡观察池
   0分 → 丢弃（不记录）
4. 有条件时：用AI Studio扫描指令跑一次批量扫描
5. 更新「📊本周新发现」区
6. 微信群发周报（模板见下）
```

**周报模板**（每周日发）：
```
【雷达周报·第X周】
本周扫了：[信号源名称]
新发现：
1. [产品名]：[一句话]，初筛X分，来源[哪里]
2. ...
无新发现（如果真的没有就写这句）
product-radar.md已更新
```

**重要原则**：宁可少报，不要凑数。没有新发现就写"无新发现"。

### 事3：工作流工具探索（持续）

探索哪些AI工具/自动化工具能用到我们的工作流里。

**背景**：我们有两条路线并行——内容线（公众号）+ 产品线（未来MVP）。  
你的工具探索直接服务于这两条线：哪些工具能让雷达扫描更快、内容流水线更顺、或者将来MVP能用。

**工具范围**（按优先级）：
1. **自动化工具**：Coze、n8n、Make——把重复的工作流串起来
2. **AI扫描工具**：各AI Studio、Perplexity——让雷达扫描更高效
3. **内容工具**：辅助王的排版/分发流程的工具
4. **MVP相关**：等赛道确定后再定向探索

**探索方式**：
- 每周选一个工具，动手试，不只是看介绍
- 试完写一段话（3-5句）：能做什么、适合我们工作流的哪个环节、用起来麻不麻烦
- 存入 `D:\sk\ops\工具探索记录.md`（文件不存在就新建）

**判断标准**（一个工具值得深入的条件）：
- 能替代现在某个需要人工重复操作的步骤
- 上手时间 ≤ 1天
- 不依赖特殊资质或审批

## 二、和王一起做的事：仓库自动化

**背景**：仓库（D:\sk）是我们所有工作的基础设施。  
目前很多维护动作是手工的——雷达扫描结果要手动填、文件更新要手动追踪。  
你们两个一起讨论并逐步把这些变成能跑的工具。

**优先做什么**（按价值排序）：
1. **雷达扫描结果自动存档**：扫描完自动追加到product-radar.md，不用手动粘贴
2. **内容发布流水线工具化**：把王的排版/发布/分发流程做成checklist工具或脚本
3. **案例库维护提醒**：完成一篇拆解后自动提醒更新case-cards.md和case-index.md

**工作方式**：
- 每周一次讨论（15-30分钟）：上周做了什么、卡在哪里、这周做什么
- 杨旁观，你们两个主导
- 每个自动化模块做完，写一个「输入什么→输出什么→替代了什么手工动作」的说明

## 五、遇到问题怎么办

**卡住超过2小时**：微信群说，不要自己闷头耗。

**找不到某个文件**：先看README.md（D:\sk\README.md），那是导航地图。

**不知道某个工具值不值得探索**：问杨一句，5秒有答案。

**发现信号源里有一个很有意思的产品**：先过初筛5问，打分，填入product-radar.md，再在群里发一句"发现了一个有意思的：[产品名]，初筛X分"。不用等到周报再说。
