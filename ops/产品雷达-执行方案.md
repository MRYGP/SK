# 产品雷达 · 执行方案

> 日期：2026-03-23
> 解决的问题：SK知识库有分析能力，没有发现能力
> 产品库位置：`cases/2026/product-radar.md`

---

## 一、为什么需要

现在的管道：
```
杨想到一个产品 → 加入拆解顺序表 → 等着拆
```

问题：来源单一、更新频率低、错过窗口。

需要的管道：
```
雷达每天扫描 → 初筛入库 → 杨每周选拆解目标 → 拆解 → 对标 → 四路决策
```

---

## 二、实习生A每天15分钟

```
8:00-8:15
□ 扫Product Hunt昨日Top5
□ 扫即刻AI相关帖子
□ 扫36氪AI融资新闻
□ 发现新产品 → 5问初筛 → 填入product-radar.md的"本周新发现"
□ 微信群发一句话："今天发现[X]，初筛[N]分，角度：[一句话]"
```

不需要深度研究。5分钟判断一个产品值不值得关注。

---

## 三、杨每周一30分钟

```
□ 看product-radar.md里本周新增
□ 从"待拆解"区选1-2个下周拆解目标
□ 调整优先级（有没有强信号产品应该插队？）
□ 跟30篇大纲对照：下一篇该写什么产品？
```

---

## 四、每月一次1小时

```
□ 观察池清理：超30天没新信号的移除
□ 待拆解区review：窗口关了没？
□ 已拆解review：四路决策还成立吗？
□ 信号源review：哪个来源产出最多？加强或砍掉
```

---

## 五、跟现有系统的接口

```
产品雷达（发现+初筛）
      ↓
待拆解队列（product-radar.md）
      ↓
场景A（v4.0模板+RRF拆解）
      ↓
四路决策
      ↓
如果"复制" → 场景G推演 → 产品设计方法论
```

---

## 六、第一个月先手动

不用自动化工具。手动扫的好处是实习生A会建立"什么产品值得关注"的直觉。
第二个月如果量太大，再考虑RSS/AI辅助。

---

## 七、给实习生A的补充说明

**你需要新增的日常任务**：

在原来的每日任务之后加15分钟：
```
原有：排版/发布/冷启动/评论区/数据记录
新增：8:00-8:15扫描AI产品信号源
```

**初筛的感觉**：
不需要"分析"。就是看到一个产品，问自己5个问题，每个问题1秒钟给答案。做多了就快了。

**什么算"有反直觉的地方"**：
你看到这个产品的时候，心里冒出"为什么？""怎么可能？""这也行？"——就是。
如果看完只觉得"哦，又一个AI工具"——那就不是。
