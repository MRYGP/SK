# 月度文档健康度审计协议

> 版本:v0.1(2026-04-26 起稿)
> 用途:防仓库腐烂——识别并处置"会过时但没人触发"和"已死但没被判定"的文档
> 调用者:Claude,每月最后一天主动跳一次。不依赖杨提醒。

---

## 为什么需要这个协议

仓库中的文档存在四象限:
         会过时         不会过时
  ┌──────────────┬──────────────┐
高频触发│  A 必须维护    │  B 安全       │
├──────────────┼──────────────┤
低频触发│  C 危险!      │  D 死了/可能死  │
└──────────────┴──────────────┘

- **C 是最危险的**——会过时但没人触发,等到要用时是错的。
- **D 是最浪费的**——建好之后没被触发,占着位置。

本协议核心动作:**每月最后一天扫一次,判定每个可疑文档的归宿**。

---

## 触发时机

- **主触发**:每月最后一天,Claude 主动跳。
- **唤醒触发**:杨随时说"跳文档健康度审计",Claude 立刻跳。
- **未跳警报**:过了月末 7 天仍未跳,下次对话末尾对账时显式警报"N 月度审计未跳"。

---

## 审计范围(分三块)

### 块 1:C 类扫描(会过时 + 低频触发)

对以下文件逐个检查"上次修改日期"与预期频率是否脱节:

| 文件 | 预期更新频率 | 警报阈值 |
|------|--------------|----------|
| `cases/2026/case-cards.md` | 每次拆解后追加 | 超过 2 个未补的案例 |
| `cases/2026/case-index.md` | 每 5 个案例同步 | 低于最新文章编号 5+ |
| `ops/MTP追踪.md` | 每篇 MTP 发布后 24h | 某篇 MTP 状态与文章状态不一致 |
| `meta/三湘问道-系统全貌图.html` | 2 周 | 超过 1 个月 |
| `README.md` | 有新文件加入时 | core/ 或 ops/ 文件计数与实际不一致 |
| `meta/CLAUDE系统指令.md` | 有新 SKILL/清单入库时 | 某个 SKILL 未出现在关键词表 |
| `radar/signals.md` | 季度 | 季度未复查 |
| `content/公众号内容大纲-30篇规划.md` | 每发布 5 篇 | 实际与规划偏离 > 5 篇 |
| `cases/2026/角色定位与变现路径.md` | 季度 | 超过 1 季 |
| `cases/2026/诊断空白系列-写作方案.md` | 系列写完后 | 系列完成但未报退 |

**处置**:每个脱节项输出状态 + 建议动作(补齐/重写/降级/归档)。

---

### 块 2:D 类扫描(可能死亡的文档)

对近 30 天未被引用 / 未被修改 的文件逐个检查:
对每个文件:
如果 上次修改 > 30 天前:
如果 未被 CLAUDE系统指令 引用:
如果 未被 其他文档 引用:
标记为 "D 类倒亡候选"

**每个倒亡候选出三选一判定**:

1. **复活**:分配下次触发条件(加到 CLAUDE系统指令关键词 / 其他文档引用 / 明确使用场景)
2. **归档**:移到 `_archive/`,在原位放占位说明
3. **删除**:确实没用(严重需要杨确认,不默认删)

**判定原则**:
- 近 90 天有 3 次以上被引用 → 复活
- 例多看以后可能用但现在不用 → 归档
- 完全过时 / 已被更新版本替代 → 询问杨是否删除

**安全门**:什么情况下不归档?
- 哲学/论域上"偶尔被重读"价值高的文档(如厄股都是某些理论文档)
- 同一主题只有一份的(即使低频也是唯一资产)
- 未来 6 个月内明确要启用的(如已重分配编号的 016 MTP 等)

---

### 块 3:theory/ 被引用度审计

理论库包含 28 个 DOC 文件,每个查:

1. 被 CLAUDE 系统指令"搜索关键词表"引用了吗?
2. 被本月任何其他文件(拆解/文章/推演流水账)引用了吗?

**输出三档名单**:
- 高频(近 90 天被引用 ≥ 3 次)——保留
- 低频(1-2 次)——保留但记录
- 零频(0 次)——交杨决策:加入关键词 / 归档 / 删除

**不要主动删理论文件**。理论是低频但高价值资产,交杨决策。

## 与其他协议的关系

本协议是"月度扫描",与以下协议互补:

- `ops/对话末尾检查清单.md` → 每次对话末尾对账(高频 / 微观)
- `ops/进度追踪.md` → 当前窗口货状态(动态 / 中观)
- `ops/月度文档健康度审计协议.md` → 本文件(低频 / 宏观)

三者频率递减,覆盖范围递增。

## 验证期

本协议在第一次跑后(1 个月后,2026-05-31)评估是否有效。

如果有效→加到 CLAUDE 系统指令的"对话开始时的首要动作":如果今天是月末,跳本协议。
如果无效→修订或废弃。
