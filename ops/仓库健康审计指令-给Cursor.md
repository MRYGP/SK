## 检查项目（按优先级排序）

### 🔴 P0：断链检查（最高优先级）

扫描所有 `.md` 文件中出现的文件路径引用（格式：`` `path/to/file.md` `` 或 `` D:\sk\... ``），检查：

1. 引用的文件是否在仓库中实际存在？
2. 引用的路径是否正确（注意 `D:\sk` 和 `E:\sk` 的区别）？

**输出格式：**
```
[断链] 发现于：文件名:行号
  引用路径：xxx
  问题：文件不存在 / 路径前缀错误（E: vs D:）
```

---

### 🔴 P0：多版本共存检查

检查是否有同一内容的多个版本文件共存（非归档状态）：

- 文件名中含有 `v1`/`v2`/`v3`/`V1`/`V2`/`V3` 或 `-1`/`-2`/`终稿`/`旧版` 等标记
- 同名文件存在多个（如 `011-xxx-v2.md` 和 `011-xxx-v3.md` 同时存在于根目录）

**注意：** `_archive/` 目录内的是归档文件，不算问题。只检查活跃目录。

**输出格式：**
```
[多版本共存] 目录：xxx
  - 文件A（大小，最后修改时间）
  - 文件B（大小，最后修改时间）
  建议：保留较新版本，旧版移入 _archive/
```
