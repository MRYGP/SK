# 对话末尾检查清单

> 版本:v0.1(2026-04-25 起稿,验证 1-2 周后定型)
> 用途:Claude 每次对话末尾必须显式跑一遍并输出对账,防止仓库维护遗漏
> 协议:Claude 每次回答的最后一段必须显式输出"─── 对账 ───"块,让用户能验证

---

## 触发时机(底线,不可跳过)

满足以下任一条,Claude 必须跑检查清单:

- 本次回答动了仓库中任一文件(用 edit_file)
- 本次回答提出了要 Cursor 动手的动作
- 本次回答提到了"下一步""后面再""还需要"
- 本次对话超过 5 轮
- 用户明确表示即将结束对话

不要问"要不要给你对账",直接输出。问了等于默认不跳,违反协议。

---

## 检查清单(按顺序跑)

### A. 新冒出的清单角度

- 本次对话冒出几条新角度?(0 / 1-2 / 3+)
- 都已写入 `ops/清单草稿-未验证教训.md` 了吗?
- 没写入的列出来,要么本次写,要么登记到 `ops/进度追踪.md`

### B. 新冒出的待办文件操作

逐项过:
- 需新建文件:[列出 / 无]
- 需重命名:[列出 / 无]
- 需修改文件:[列出 / 无]
- 需删除文件/目录:[列出 / 无]

每项判定:
- 立刻能做的(edit_file 能处理) → 是否已做?
- 需 Cursor 处理的 → 是否已给 Cursor 指令?
- 暂不做的 → 是否写入 `ops/进度追踪.md`?

### C. 新冒出的理论/方法论修订

- 涉及哪些 SKILL/审问清单/系统指令?
- 修订是立刻做还是延后?延后的写入 `ops/进度追踪.md` 了吗?

### D. 反漂移自检

- 本次对话原始出发点是什么?(一句话)
- 当前讨论焦点和原始出发点的论域一致吗?
- 是否有"用聪明替代行动"的迹象?
- 是否触发"修改出发点的合法性 3 判据"(清单条 37)?

### E. 接力棒

- 本次对话留下了什么没做完的?(一句话)
- 下次对话开头要做的第一件事?(一句话)
- 是否已同步到 `ops/进度追踪.md` 的"当前窗口"节?

---

## 输出格式(给用户看)

每次对话末尾,Claude 输出一段:
─── 对账 ───
A 清单角度:[已处理 / 待处理:列出]
B 文件操作:[已处理 / 待 Cursor / 已登记进度追踪]
C 理论修订:[已处理 / 已登记进度追踪]
D 反漂移:[本次出发点 / 是否一致 / 是否漂移]
E 接力棒:[未完成 / 下次开头 / 进度追踪同步状态]
─── ✓ ───

不能是隐式对账("我心里过一遍了")。必须是显式输出。没输出 = 违反协议。

---

## 与 ops/进度追踪.md 的关系

本检查清单是"如何检查"的协议。
`ops/进度追踪.md` 是"检查出来的东西往哪里放"的状态文件。

两者需配合使用:
- 检查清单 → 定义什么要检查
- 进度追踪 → 记录检查出来的东西、演变状态、窗口衔接

Claude 不仅要跑检查清单,还要在对话末尾同步进度追踪的"当前窗口"节。

---

## 协议本身的迭代

本协议从 2026-04-25 起验证 1-2 周。

每次发现遗漏类型 → 立刻补充到本文件第十节。

验证期结束时:
- 如果有效 → CLAUDE 系统指令加一条最简短引用("对话结束前必须执行 ops/对话末尾检查清单.md")
- 如果无效 → 修订或废弃
