# 清单草稿 · 未验证教训

> 版本：持续更新
> 用途：**零摩擦捕获任何一次对话中冒出的"新发现"，不判断、不精炼、不组织**
> 升级规则：出现3次 → 升级到 `清单候选-待升级教训.md`

---

## 捕获格式

一行一条。必须包含：
- 日期
- 一句话表述
- 来源（哪次对话/哪个项目）

例：
```
2026-04-19 | 反向推理比正向推理应优先 | 来源：KAIROS拆解对话
```

## 更新规则

**每次在对话中冒出新想法时**：
- 立刻来这里加一行，带日期和来源
- 不要判断它对不对、重不重要
- 不要组织结构
- 就是流水账

**每周日 / 每月末回看时**：
- 看看哪些重复出现了
- 出现3次以上的 → 升级到 `清单候选-待升级教训.md`
- 超过6个月仍然只出现1次的 → 划掉（说明不重要）
