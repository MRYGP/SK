# 给实习生A：什么是冷启动，为什么你做的事情很重要

> 写这篇是因为你每天在执行的"发朋友圈、群发、私信、评论区运营"，
> 不是杂活，是整个公众号能不能活下去的关键动作。
> 读完这篇你会知道：为什么要这么做，每个动作背后的逻辑是什么。

---

## 一、冷启动是什么

一句话：**新号没有粉丝，文章发出去没人看，怎么让第一批人看到。**

这就像开了一家新店，装修好了，菜也做好了，但开在一条没人经过的巷子里。冷启动就是想办法把第一批客人拉进来。

为什么叫"冷"？因为公众号刚注册的时候，关注数是0，没有任何读者基础。系统也不认识你，不知道该把你的文章推给谁。你发一篇文章出去，如果没人看、没人互动，它就沉底了，永远不会被更多人看到。

---

## 二、公众号的流量机制（你需要知道的核心逻辑）

2025年以后，公众号的流量不再只靠粉丝数。现在是这样运转的：

### 文章发出去后会经历"流量池晋级"

```
你发了一篇文章
    ↓
系统先推给一小批人看（几百人）——这是"初始流量池"
    ↓
如果这一小批人的反应好（点开看了、看完了、点了推荐、评论了）
    ↓
系统就把文章推给更多人（几千人）——晋级到下一个流量池
    ↓
如果继续表现好 → 继续晋级 → 越来越多人看到
    ↓
如果反应不好 → 文章沉底，不再推了
```

**关键点：前2小时决定生死。** 系统看的是文章刚发出去那段时间的数据。如果前2小时没人看、没人互动，文章就被判"死刑"了。

### "朋友推荐"是最重要的流量入口

微信里有一个功能：读者读完文章后点底部的"推荐"（♡按钮），这篇文章就会出现在他微信好友的信息流里。

这意味着什么？**一个人点了推荐，他的几百个微信好友都可能看到这篇文章。**

而且这些好友跟他是同类人。如果一个做AI的人点了推荐，他的朋友圈里大概率也有做AI的人。这比系统随机推荐精准100倍。

---

## 三、你做的每件事，背后的逻辑是什么

### 发朋友圈 → 制造第一批读者

文章刚发出去的时候，系统给的初始流量池可能只有几百人。但如果你同时发了朋友圈，你微信里的好友会直接点进来看。这些人就是系统看到的"第一批读者"。

他们的行为（看没看完、有没有点推荐）直接决定系统会不会给你更多流量。

**所以朋友圈不是"宣传"，是给系统喂第一口数据。**

### 群发到微信群 → 扩大第一批读者的规模

朋友圈能触达的人有限（取决于你有多少好友）。微信群可以一次触达几百人。

选群的标准：群里有我们的目标读者（做AI的人、创业者、投资人）。在这些群里发，来看的人更可能认真读、点推荐。

**群发不是广告，是把文章送到对的人面前。**

### 私信10个人请他们"点推荐" → 启动社交裂变

这是最关键的一步。

假设10个人里有5个读完了并点了推荐。每个人的微信好友里有200个人可能看到这篇文章。5个人 × 200 = 1000个潜在读者。

如果这1000个人里又有50个点了推荐，50 × 200 = 10000。

**这就是为什么我们要专门"请人点推荐"，而不只是"请人看看"。** "看了"对系统没有太大作用，"点了推荐"才会触发裂变。

### 评论区写第一条评论 → 降低互动门槛

人的心理是这样的：看到一篇文章下面0条评论，不太想当第一个评论的人。但如果已经有一条评论了，第二个人就更容易跟上。

你写的第一条评论还有一个作用：**引导讨论方向。** 比如我们001的第一条评论是问"你家里的老人最开心的时刻是什么"——这个问题很具体，读者可以直接回答，不需要思考"我该说什么"。

### 回复每一条评论 → 告诉系统"这篇文章有活跃度"

系统判断文章好不好的一个重要指标是"互动率"。评论越多，系统越认为这是一篇值得推的文章。

而且**每回复一条评论，发评论的人会收到通知，有可能再次打开文章**——这又给系统增加了一次"有人在看"的信号。

## 四、新号的"新手保护期"

公众号新注册后有大约30天的"新手保护期"。这段时间系统会给你额外的流量倾斜——也就是说，同样质量的文章，新号比老号更容易被推荐。

**这30天极其宝贵。** 如果这30天内你建立了"AI产品拆解"这个清晰的标签（系统通过你发的内容来打标签），后续推荐流量会持续来。

如果这30天断更了、或者发的内容方向很杂（今天AI拆解、明天美食、后天旅游），系统不知道你是干嘛的，就不会推荐你。

**所以前30天的纪律：只发AI产品拆解相关的内容，不发其他的，不断更。**

## 五、用一张图总结

```
你的工作
   |
   ├── 发布前：排版 + 准备好标题/首图/群发文案/私信名单
   |
   ├── 发布后0-10分钟：朋友圈 + 群发 + 私信
   |         → 目的：制造"第一批读者"，喂给系统第一口数据
   |
   ├── 发布后10-30分钟：写第一条评论
   |         → 目的：降低互动门槛，引导讨论方向
   |
   ├── 发布后30分钟-2小时：回复每一条评论
   |         → 目的：提高互动率，告诉系统"这篇有人聊"
   |
   └── 发布后24-48小时：知乎改编 + 即刻发帖 + 小红书卡片
             → 目的：用同一篇内容在不同平台触达不同的人，引流回公众号
```

## 六、你怎么知道冷启动做得好不好

不看阅读量。新号的阅读量一开始一定很低，这是正常的。

看这三个：

| 指标 | 好的信号 | 不好的信号 |
|------|---------|-----------|
| 推荐数 | ≥10个 | < 5个 |
| 评论数 | ≥5条 | 0条 |
| 有没有人私信 | 有人主动私信聊内容相关的事 | 只有朋友客气说"写得不错" |

推荐数是最重要的。因为每一个推荐都会带来潜在的新读者。

## 七、常见问题

**Q：群发会不会让人觉得烦？**

如果你发的是"帮我转发谢谢"，会烦。但如果你发的是"写了一篇拆解，发现了一件事——所有AI产品都在做错同一步"，这是有信息量的，不烦。关键是**你发出去的每条消息本身要有内容，不是单纯的求转发。**

**Q：为什么不花钱投广告？**

两个原因：一是我们预算有限；二是花钱买来的读者不精准。我们要的不是随便什么人来看，是三类特定的人（合伙人、资源伙伴、跟投人）。通过社交推荐来的人天然更精准。

**Q：前几篇阅读量很低怎么办？**

正常。完全正常。我们001的成功标准不是10万+，是"10个人读完觉得值得关注"。如果前5篇每篇带来5-10个真正对AI感兴趣的关注者，到第10篇你就有了50个种子用户。50个种子用户的推荐能力已经很强了。

**Q：多平台分发（知乎/即刻/小红书）有什么用？**

公众号是主战场，其他平台是"引流入口"。不同的人在不同的平台活跃。有人天天刷知乎不看公众号，有人天天逛即刻。我们在这些地方出现，把人引到公众号来关注。

## 八、一句话记住

**你的工作不是"帮老板发文章"。你的工作是"在文章发出去的前2小时内，确保对的人看到它并点了推荐"。**

这2小时的动作决定了这篇文章是活还是死。
