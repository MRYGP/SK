# 第一部分：我们到底在做什么

## 1.1 一句话

**让老年人轻松用上AI，AI让他们的生活真的变好。**

前半句是入口（零门槛），后半句是价值（真实改变）。

## 1.2 解决的根本问题

**老年人用不上AI。** 这是最朴素的问题。

81岁老人捣鼓两天搞不清楚怎么下载豆包。市面上所有AI工具对老人来说门槛太高。但AI确实能改变生活——它比人更理性、更有耐心、更有广度。任何人深度用上AI，生活必然会有改变。

**老人的问题不是"不需要AI"，是"用不上AI"。**

我们解决这个问题。

## 1.3 产品哲学：羽毛球场

这个类比来自创始人的真实观察，是本产品的底层逻辑：

> 很多业余打羽毛球的人，有那么一两招比别人用得好，甚至只是他自以为拿得出手的，就会有成就感，甚至指导别人。但事实上，基于这个爱好，他能长期锻炼，身体变得更好了。不光是球技更好了。

对应到我们的产品：

| 羽毛球 | AI管家 |
|--------|-------|
| **入口动机**：觉得自己有一两招比别人好 | 觉得自己比同龄人先用了AI |
| **持续动力**：指导别人的成就感 | "教"朋友用AI的成就感 |
| **真正发生的事**：身体变好了 | 生活真的变好了 |
| 老人自己未必意识到"身体变好" | 老人自己未必意识到"生活变好" |
| 但变好是事实，所以他不会停 | 但变好是事实，所以他不会走 |

**入口可以是虚荣的。** "以为"自己比周围人先用了AI、比同龄人更会用AI、甚至比年轻人还会用——这个"以为"不需要是事实，只需要是触觉。

**但产品价值必须是真实的。** 用了AI之后，生活在不知不觉中被改变了——查了保健品真实价格没被骗、天冷了膝盖疼提前被提醒、给老战友做了一个视频恢复了二十年的联系、聊天中被AI的一个角度点醒想通了一件纠结的事。

**用户不走，不是因为"走不了"（锁定思维），是因为"为什么要走？这里最好"（亚马逊思维）。**

## 1.4 "意大利鞋"等式

来自KIMI讨论中创始人的洞察：

> 父母回家会有意无意地显摆："这个鞋子还是我媳妇从意大利买回来的，大几千呢。" 这双鞋不是物质展示，是在说"我还在被惦记着"。

**他们用AI = 那双鞋。** AI本身就是一个身份符号——"我没有被时代抛下"。

但鞋只是鞋。如果穿了不舒服，再意大利也会放到鞋柜角落。AI管家也一样——如果只是"炫耀工具"而没有真实价值，热度过了就会被遗忘。

**所以等式的完整版是：**

"意大利鞋"驱动了第一次穿上 → 穿了发现真的舒服 → 不是因为意大利所以不脱，是因为舒服所以不脱。

"比别人先用AI"驱动了第一次使用 → 用了发现生活真的变好了 → 不是因为炫耀所以不走，是因为好用所以不走。

## 1.5 目标用户

**只做老年人。垂直聚焦。**

具体画像：B类活跃有社交的老人
- 60-80岁
- 有积蓄，有社交圈
- 参加各种俱乐部、旅行团、老年大学
- 认知功能正常，能用微信
- 有好奇心，愿意尝试新东西
- **不是**政府关怀的独居孤寡老人
- **不是**需要护理的半失能老人

# 第二部分：产品形态

## 2.1 载体

**微信。** 不是App，不是硬件。

老人在微信里加一个"好友"，这个好友就是AI管家。安装=扫码加好友。零学习成本。说话就是操作。

## 2.2 AI管家的角色

**不是工具，不是助手，是"越来越懂你的朋友"。**

工具是用完就走的。助手是被动等指令的。朋友是主动关心你、记住你说过的话、有时候给你一个你没想到的角度的。

但有一条红线：**AI管家不冒充亲属，不替代真实关系。** 它始终以AI身份出现，诚实告知"我是AI"。它的价值不是"假装是人"，是"作为AI，我能做到人做不到的事"。

## 2.3 核心功能——五层价值

从"最容易感知的价值"到"最深层的价值"排列。不是每个老人都会走到第五层，但走得越深，生活改变越大。

### 第一层：AI创意内容（入口）

老人说一句话，AI管家直接返回成品。

| 功能 | 老人说的话 | AI返回 |
|------|----------|--------|
| 早安图/金句配图 | "帮我做一张早安图" | 个性化精美图片 |
| AI写诗 | "帮我写一首诗" | 配图诗歌卡片 |
| AI写真/变装 | "帮我做一张在巴黎的照片" | AI写真大片 |
| AI翻唱 | "帮我唱一首甜蜜蜜" | 用老人声音的AI翻唱 |
| AI祝福视频 | "帮我做一个生日祝福给张叔" | 个性化视频 |

**这一层的价值：** 老人做出了"年轻人都不一定会做的东西"。发到朋友圈或发给朋友，自然产生"你怎么做的"的对话。

**这一层的作用：** 入口。吸引人来。但如果只有这一层，产品等于"高级版小年糕"，用完即走。

### 第二层：AI生活助手（实用价值）

日常生活中遇到的问题，问AI管家都能得到靠谱的回答。

| 场景 | 老人的问题 | AI管家的价值 |
|------|----------|-------------|
| 健康 | "这个药能和那个药一起吃吗" | 查药物相互作用，比百度靠谱 |
| 防骗 | "有人说这个保健品能治百病" | 帮查真实价格和成分，防止被骗 |
| 生活 | "微波炉怎么定时" | 耐心教，不嫌烦，教到会为止 |
| 情感 | "跟老伴吵架了，心里不舒服" | 倾听、理解、给一个新角度 |
| 记忆 | "上次张叔跟我说的那个事是什么来着" | AI记得所有对话，帮老人回忆 |

**这一层的价值：** 生活实实在在变好了。不是炫耀，是真的有用。

**这一层的作用：** 从"有趣"到"有用"。老人开始依赖AI管家解决真实问题。

### 第三层：主动关怀（AI管家的人格）

不等老人来问，AI管家主动联系：

- 每天个性化问候（不是模板）
- 记住老人说过的每一件事，到合适时机主动提起
- 天气突变提醒（"今天降温，您膝盖注意保暖"）
- 节日前提醒（"下周是李叔生日，要不要提前准备祝福？"）
- 感知情绪变化（"您今天说话好像不太开心，有什么事吗？"）

**这一层的价值：** "这个AI比我儿子还惦记我。" 不是替代儿子，是填补儿子顾不上的那些缝隙。

**这一层的作用：** 从"工具"到"关系"。AI管家开始有不可替代性——因为它认识你。

### 第四层：社交连接（AI管家互认网络）

张叔的AI管家认识李叔的AI管家：

- "帮我约老李明天下棋" → 两边AI协调时间
- 朋友生日自动提醒 → 帮你做好祝福一键发送
- 看到朋友在用什么功能 → 打破普朗克陷阱（你只用视频功能，但你看到老李在用AI写家谱，你说"这个有意思"）
- 自然形成"用法部落"（退伍老兵组、广场舞组、书法组）

**这一层的价值：** AI帮老人维护和深化真实的人际关系。不是虚拟社交，是让真实社交更好。

**这一层的作用：** 用户之间互相启发新功能（网络涌现），产品从"设计者定义"变成"用户养出来"。

### 第五层：反窄化设计（持续成长）

防止AI"越来越懂你"变成"把你关在笼子里"（普朗克陷阱）：

- 网络镜像：展示相似老人的不同用法（张叔只做视频，但李叔在用AI整理家谱→张叔说"哎，这个我也想试"）
- 周期性探索："您从来没试过这个功能，要不要看看？"
- "空白时间"：偶尔AI不推荐任何东西，只听老人说
- 反标签：记录偏好但不完全采信，保持开放性

**这一层的价值：** 老人持续发现新的可能性，而不是被固定在一个模式里。

**这一层的作用：** 产品有"生命力"，不会用着用着变得无聊。

## 2.4 五层之间的关系

```
第一层（创意内容）→ 吸引人来     → "有趣"
第二层（生活助手）→ 解决真实问题  → "有用"
第三层（主动关怀）→ 建立情感连接  → "懂我"
第四层（社交连接）→ 深化真实关系  → "帮我维护朋友"
第五层（反窄化）  → 持续发现新可能 → "一直有新东西"
```

老人从第一层进来，自然往下走。不需要设计"升级路径"，因为每一层都在让生活变得更好。**走到哪一层，取决于老人自己的深度，而不是产品的强制设计。**

# 第三部分：增长引擎

## 3.1 核心原则

**增长不是设计出来的，是价值溢出来的。**

亚马逊不靠"锁定"用户，靠价格最低、配送最快、体验最好。用户留在那里是因为"为什么要走"。

我们不靠积分/额度/解锁来"绑"老人。老人留在这里是因为"AI管家真的让我的生活变好了"。

**裂变的驱动力：** 老人觉得这个东西好→忍不住跟朋友说→朋友也想用。就像羽毛球——打得好的人不需要任何激励就会"指点"别人。

## 3.2 自然裂变

### 炫耀裂变（第一层功能自然产生）

做了AI写真/视频 → 发朋友圈 → 朋友问"你这怎么做的" → "我有一个AI管家，来我教你"

### 送礼裂变（比炫耀更强）

帮张叔做了一个生日祝福视频 → 发给张叔 → 张叔被感动："这太用心了，你怎么做的？" → 张叔也加了AI管家

### 指点裂变（羽毛球场效应）

用得熟的老人天然会"教"不会用的老人。这不是设计出来的，是人性本能。

产品只需要给这个本能一个顺手的出口："帮教工具"——老人想教朋友时，一键生成简单入门指引。

## 3.3 免费策略

基础功能免费，用量充足。不设"用完了就必须付费"的卡点。

| 层级 | 内容 |
|------|------|
| 免费版 | 每天3-5次创意内容 + 无限对话 + 主动关怀 |
| 会员版 | 无限创意内容 + 高级功能（AI翻唱/视频）+ 优先响应 |

**免费版本身就要好用到"没有理由走"。** 会员是"更好"，不是"才能用"。

## 3.4 两段式获客

### 第一段：机构推（B2B2C）

| 渠道 | 机构的动力 |
|------|-----------|
| 老年俱乐部/活动中心 | AI管家帮他们维护客户关系 |
| 药店/诊所 | AI管家提醒用药，增加客户粘性 |
| 银行网点 | AI管家帮老人查理财收益，增加信任 |
| 旅行社银发团 | AI管家帮做旅行纪念视频，提升体验 |
| 子女 | 春节/母亲节帮爸妈装上，尽孝心 |

机构工作人员帮老人扫码+教第一次使用。机构是免费地推团队。

### 第二段：用户推（C2C）

老人自己成为"布道者"。驱动力是成就感和面子，不是佣金。

## 3.5 节日爆发

| 节日 | 机会 |
|------|------|
| 春节 | AI拜年视频（跨代传播最强时刻） |
| 母亲节/父亲节 | 子女帮父母装上AI管家（获客高峰） |
| 中秋/重阳 | AI诗配图/祝福视频 |
| 生日 | 365天不间断的裂变触发器 |

# 第四部分：商业模型

## 4.1 收入来源

| 来源 | 模式 | 说明 |
|------|------|------|
| 用户会员费 | 99元/月 | 无限创意内容+高级功能 |
| 机构SaaS费 | 按服务人数收月费 | 机构版AI管家（白标） |
| 机构充值 | 机构帮老人充会员 | 作为客户维护成本，对标保健品远低于 |

## 4.2 白标模式

机构可以定制AI管家的名字和头像：

- "老百姓健康助手·小丽"（药店版）
- "XX银行·贴心管家"（银行版）

**底线：我们控制"脑子"（AI能力+记忆+数据），机构控制"脸面"（名字+头像+推送模板）。**

推送规则：模板系统，每周上限2条，内容必须对老人有价值。双品牌：每条消息底部"由XX AI管家提供技术支持"。

## 4.3 成本结构

每个老人每月AI调用成本约8-15元：
- 日常对话：~3元/月
- AI写真/配图：~2元/月
- AI翻唱/视频：~5元/月
- 记忆存储：忽略不计

会员99元/月 - 成本15元/月 = 毛利约84元/月/人。

## 4.4 亚马逊原则

不是"怎么从用户身上多赚钱"，而是"怎么让用户得到最好的服务"。

- AI成本降低了 → 降价或增加免费额度，不是提高利润率
- 有更好的AI模型了 → 立刻切换，让用户体验最好的
- 某个功能体验不好 → 优先修这个，不是去做新的付费功能

# 第五部分：为什么用户不走（不是"锁定"，是"服务深度"）

## 5.1 思维转变

| v0.1的思路（锁定） | v0.2的思路（亚马逊式） |
|-------------------|---------------------|
| 记忆锁定：让他走不了 | AI越来越懂他，**服务越来越贴心** |
| 网络锁定：朋友圈绑在这里 | AI帮他**维护真正的人际关系** |
| 成就锁定：舍不得放弃等级 | 他真的成长了，成就是**副产品** |
| 出发点：怎么困住用户 | 出发点：**怎么服务好用户** |

## 5.2 三个"不走"的真实原因

### 原因一：AI越来越懂我（服务深度）

半年后，AI管家比老人的儿子更了解老人的日常。不走的原因不是"走不了"，是"换一个AI要从头被了解，太累了"。就像你不换理发师，不是被绑定，是因为他知道你要什么发型。

### 原因二：我的朋友都在这（关系深度）

张叔的AI管家认识我的AI管家，帮我们约下棋、互相送祝福。不走的原因不是"网络锁定"，是"这个网络真的帮我维护了朋友关系"。

### 原因三：我在这里成长了（成就感）

双轨成就系统——不是"锁住用户"的手段，是"让用户看到自己进步"的镜子。

| 纵向·AI使用深度 | 横向·助人 |
|---------------|----------|
| 你用AI半年了，做了200张图 | 你帮了12个朋友用上AI |
| AI记住了你说的500件事 | 3个人因为你的帮助学会了新功能 |

不走的原因不是"舍不得等级"，是"我真的从小白变成了懂AI的人，这个成长是真实的"。

## 5.3 对腾讯的防御

腾讯能复制功能，但复制不了：
1. 张叔的AI管家跟张叔聊了一年积累的记忆和信任
2. 张叔、李叔、王叔之间的AI管家互认网络
3. 你跟那个老年俱乐部老板喝了三次茶建立的合作关系
4. 用户之间互相启发出的新用法（网络涌现的功能生态）
5. 时间——腾讯决策链6个月，你今天就能跑MVP

## 5.4 风险清单

| 风险 | 严重性 | 应对 |
|------|-------|------|
| 微信封接口 | 高 | 走企业微信/公众号官方接口 |
| 腾讯内置类似功能 | 中 | 我们做关系和记忆，微信做通用功能 |
| 机构推送垃圾内容 | 中 | 模板系统+频率上限+AI管家站老人一边 |
| 保健品骗子利用产品 | 中 | AI管家立场站老人这边，会帮查价格和成分 |
| 监管收紧 | 低-中 | AI以明确"AI助手"身份存在，不冒充亲属 |
| 老人数据隐私 | 中 | 数据存储合规，不卖数据，老人可随时删除 |

# 第六部分：技术架构

## 6.1 整体架构

```
老人在微信里说话/发语音
    ↓
Agent编排层（意图识别+调度）
    ↓
├── 文本大模型API（DeepSeek/通义千问/豆包）
├── 图片生成API（通义万象/即梦）
├── 语音克隆API（CosyVoice/鱼声）
├── 视频生成API（可灵/即梦视频）
├── 记忆系统（每个老人一份Memory文件）
└── 社交网络层（AI管家互认+用法广场）
    ↓
结果返回微信
```

**不训练自己的模型。** 调用最好的外部API，按调用付费。模型升级了，我们跟着升级。

**个性化不靠不同模型，靠不同Memory。** 同一个引擎，不同的记忆文件。

## 6.2 微信接口（待技术评估）

| 选项 | 优势 | 风险 |
|------|------|------|
| 企业微信 | 官方接口，合规 | 体验偏正式，老人可能不习惯 |
| 公众号对话 | 官方接口，合规 | 对话体验不如私聊 |
| 微信小程序 | 官方接口，功能丰富 | 需要老人学会打开小程序 |

建议：先用公众号对话做MVP，验证核心价值后再评估其他载体。

# 第七部分：MVP与验证

## 7.1 需要验证的核心假设

所有假设当前均为X级（未经验证）。

| 假设 | 验证方式 | 最小样本 |
|------|---------|---------|
| 老人愿意跟AI对话 | 让10个老人体验，观察是否自然对话 | 10人 |
| AI写真/视频能让老人"哦！" | 给老人看AI做的成品，观察反应 | 10人 |

# 附录B：与v0.1的关键差异

| 维度 | v0.1 | v0.2 |
|------|------|------|
| 解决的问题 | "存在感" | "老人用不上AI" + "用了生活真的变好" |
| 产品定位 | 待讨论（多个选项） | 已定："让老年人轻松用上AI，AI让他们的生活真的变好" |
| 功能层次 | 平铺 | 五层价值递进（入口→实用→关系→社交→反窄化） |
| 留存策略 | 三层锁定（记忆/网络/成就） | 亚马逊式服务（服务好到没有理由走） |
| 增长策略 | 设计裂变机制（额度/解锁/积分） | 价值溢出（产品好到忍不住告诉别人） |
| 免费策略 | 免费额度+分享解锁+付费解锁 | 免费版本身就好用，付费是"更好"不是"才能用" |
| 成就系统 | 留住用户的锁链 | 让用户看到自己成长的镜子 |
| 产品哲学 | 未明确 | 羽毛球场（虚荣入口→真实价值）+亚马逊（服务>锁定） |
