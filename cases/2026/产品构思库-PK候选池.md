## PK规则

积累到10个后，每个构思用以下5个维度打分（各10分）：

| 维度 | 说明 |
|------|------|
| 中国用户明确性 | 能说出具体是哪群人，不是泛泛说"有市场" |
| 技术可行性 | 用现有API能实现吗？需要多少资源？ |
| 不对称优势 | 我们有没有别人没有的东西（理解/渠道/关系）|
| 时间窗口 | 现在做是不是好时机，有没有信号支撑 |
| 变现路径清晰度 | 谁付钱，为什么付，付多少 |

总分最高的1-2个进入MVP阶段。

### IDEA-001｜老年AI管家

**来源**：【自有构思】+ 已有大量推演积累
**核心重新定义**：不是"照顾老人"，而是"让老年人用上AI，让他们的生活更好"
**用户洞察**：一部分生活条件还可以的老年人，核心需求是"被看见"——让人注意到自己虽然老了，但存在价值貌似比周围人还强（社交虚荣+真实价值的双层结构）
**目标用户**：有一定经济基础、在社交圈有一定地位、对新事物有好奇心的老年人（"圈子潮人"）
**北极星**：让老年人用上AI，从而生活真的变得更好
**当前状态**：已完成v0.5产品设计，已跑过完整推演SOP，已有七份红队报告，暂停推进等内容线先跑起来
**关键文件**：`cases/2026/老年AI管家/` 目录
**待补**：执行合伙人（银发经济行业背景），找到后重启

### IDEA-002｜活书（KAIROS Publishing）

**来源**：【自有构思】，有详细商业计划书（KAIROS Publishing BP v2.0）
**电梯推介**：智慧版Spotify——将世界上所有伟大的书籍，从静态文字，解放为可以与你的个人生活实时交互的"AI人生导师"

**核心重新定义**：
- 原来：书是"专辑"——用户"读"书，费时解码，学了用不上
- 现在：书是"播放列表"——用户"用"书，带着真实困境来，AI引导你用书中思维框架解构问题
- 不给你答案，而是成为顶级"提问官"——基于作者思想，向你提出能帮你"看清自己"的高质量问题

**产品核心体验**：
用户拿着"要不要接受这个工作机会"的困惑来，卡尼曼的AI化身不给答案，而是问："你现在更害怕'失去'现有稳定，还是更渴望'得到'新机会？你是否陷入了损失厌恶？"
更进一步：召唤卡尼曼+达利欧同时会诊，从认知心理学和绝对理性两个视角给决策框架。

**三层价值**：
- 第一层（免费）：动态阅读——角色化阅读，高效输入理论框架
- 第二层（核心订阅）：情境赋能——带着真实问题来，获得"大师提问清单"和"事后认知复盘"
- 第三层（生态）：作者合作——思想影响力即服务，作者授权+长期收益分成+《思想应用报告》

**滩头市场**：高风险决策辅助（创始人/高管/投资人），付费意愿最强
**商业模式**：
- 实践者版：$49/月（无限调用大师情境赋能）
- 架构师版：$149/月（解锁"AI史官"，构建个人认知资产）
- 企业版：定制报价

**护城河（飞轮）**：
1. 独特理论内核（CWET认知理论）→ 极致单点体验
2. 高质量种子用户+信任社区（1000名"信徒"的密度壁垒）
3. AI史官驱动的个人认知资产沉淀（不可移植的用户锁定）
4. UGC×顶级内容双螺旋（高质量用户吸引顶级思想家，顶级内容吸引更多高质量用户）

**MVP**："芒格逆向思维练习器"——极简Web应用，只做"逆向思维"一个思维模型，集成"决策卡片"分享功能
**MVP验证三个假设**：
1. 用户认为"与大师AI陪练"远超阅读（价值假说）
2. 用户愿意为解锁下一个思维模型付费（付费意愿假说）
3. 用户愿意分享生成的"决策卡片"（分享意愿假说）

**与IDEA-003的关系**：003是"把自己的智慧AI化传承给后辈"，002是"调用别人的智慧解决自己的问题"。方向互补，用户群有交叉。

**海外对标**：
- Spotify（分发+个性化推荐的范式）
- Curology（订阅制+个性化定制的商业模式）
- 尚无直接竞品——"认知增强即服务（Cognitive Augmentation as a Service）"是新品类

**待验证**：
- 中国知识付费用户（得到/樊登读书的用户）是否接受"交互式智慧"而非"知识灌输"？
- 版权合作路径：如何获得顶级畅销书授权？
- 定价是否适合中国市场（$49/月≈350元/月，偏高）？

**当前状态**：有完整BP v2.0，构思成熟，未立项推进

### IDEA-003｜交互式自传·智慧传承产品

**来源**：【自有构思】，有部分海外参照（MUSK 2008类产品方向）
**核心重新定义**：自传不只是记录，而是一个能"陪伴后辈成长"的智慧载体——在后辈的生活和工作中进行培训、教育，关键时刻能一起讨论
**用户洞察**：成功的老年人（企业家、管理者、专业人士）希望智慧得以传承，但传统自传是单向的、死的。这个产品让它变成双向的、活的。
**产品形态**：
- 输入层：结构化采集当事人的经历、决策、价值观、方法论
- 交互层：后辈/继承者可以向"数字祖辈"提问，获得基于其真实思想的回答
- 培训层：可以设计学习路径，在关键人生节点主动推送相关智慧
**目标用户（付费方）**：有一定财富积累和社会地位的老年成功人士，或其家族
**潜在变现**：高客单价（家族定制级别），非标服务
**与IDEA-001的关系**：用户群有重叠（老年人），但产品逻辑完全不同。001是高频日常助手，003是低频高价值传承产品
**与IDEA-002的关系**：002是调用别人的智慧，003是把自己的智慧AI化
**待验证**：中国用户愿不愿意为"数字化自己的智慧"付费？谁来启动（本人还是子女）？

### IDEA-004｜零碎时间赚零花钱·AI副业助手

**来源**：【自有构思】+ 已有详细文档（`cases/2026/产品构思-通用任务代理与副业AI助手.md`）
**核心重新定义**：不是"AI帮你暴富"，而是"AI带着你把轻副业持续做下去"
**用户洞察**：学生、宝妈不缺想法，缺的是持续推进力——大多数副业死于"不知道下一步做什么"和"中途放弃"
**目标**：每月额外收入500-2000元，不期望大钱，是赚零花钱
**目标用户**：学生、宝妈、有碎片时间的上班族
**MVP验证三件事**：
1. 用户愿不愿意把赚钱任务交给AI推进
2. AI能不能把副业方向拆成每日/每周动作
3. 用户是否比没有AI时更容易坚持
**变现路径**：订阅制（低客单价，高频使用）
**关键文件**：`cases/2026/产品构思-通用任务代理与副业AI助手.md` 第二部分
**待验证**：这类用户付费意愿有多高？月费能到多少？

### IDEA-005｜AI长期健康管理·从皮肤管理切入

**来源**：【自有构思】+ 深度讨论记录
**核心方向**：AI预管理+长期跟踪+个性化——把美容从"玄学"变成"皮肤健康工程"

**核心重新定义**：
- 原来：美容院是"事后救火"——已经长斑/过敏了才来处理
- 现在：AI是"皮肤科的住院医师"——持续监测+预测+干预，在问题发生前解决
- 卖的不是产品，而是"确定性"——"我可以不再焦虑地照镜子"的心理免责权

**用户洞察**：
- 消费者：试错成本极高（买错产品/做错项目），决策充满后悔厌恶；需要"确定性"不是"希望"
- 美容机构（B端）：AI操作出问题归咎于技术，不归咎于技师；"心理免责权"是B端采购AI的核心动力

**巨头的结构性盲区（为什么大厂做不了）**：
1. 产品逻辑诅咒：欧莱雅的AI是为了"卖得更多"，这个产品的AI是为了"卖得更准"（甚至少买）——价值观冲突
2. 渠道反噬：BA靠推销提成，AI说真话会断BA财路，巨头不敢让AI太聪明
3. 数据孤岛：巨头的AI是"横向平均"（百万人画像），你的AI是"纵向深潜"（同一个人6个月连续数据）
4. 合规恐惧：巨头不敢"医疗暗示"，你可以擦边医疗（与互联网医院合作）

**产品架构（三层闭环）**：
- 感知层：家用皮肤镜（手机摄像头，<200元）+ 每周自拍 + 环境传感器 + 行为数据
- 决策层：预测模型（7天后皮肤状态预测）+ 干预推荐（预测屏障下降时自动调整配方）
- 交付层：个性化护肤品（中央工厂按需微批次生产）+ 轻干预服务导流

**商业模式**：
- 基础版：99元/月（AI监测+基础保湿配方）
- 进阶版：299元/月（动态抗衰管理+光老化预警）
- 尊享版：599元/月（皮肤科医生季度远程咨询+优先服务预约）

**五维差异化（非对称优势）**：
1. 极端人群深耕：玫瑰痤疮/重度敏感肌——巨头不做，用户付溢价，数据壁垒极高
2. 动态配方引擎：根据实时数据每周调整成分（"鲜做护肤品"，保质期3个月）
3. 皮肤数据主权：用户拥有自己的皮肤数据，可授权变现（Web3范式）
4. 医生IP绑定：顶级皮肤科医生数字分身（医生不为化妆品品牌背书，但愿为"严肃健康项目"站台）
5. 负SKU策略：AI经常建议"少买"——短期牺牲GMV，换取长期信任资产

**MVP路径**：
- Phase 1（0-6个月）：纯软件"皮肤日记"App——验证用户是否愿意为"看得懂的数据"持续记录
- Phase 2（6-12个月）：轻硬件+月度配方——免费送皮肤监测镜，绑定年会员
- Phase 3（12个月+）：闭环验证——接入更多生物传感器，建立"皮肤健康指数"行业标准

**海外对标**：
- Proven Skincare（美国，AI定制护肤品，已获多轮融资）
- Curology（针对痤疮的远程医疗+定制药膏，已验证）
- SkinPilot（智能皮肤镜）

**待验证**：
- 用户对"AI预测皮肤状态"的接受度
- 医疗合规边界（不能诊断疾病，只能描述指标）
- 微批次生产的供应链成本

**注意事项**：
- 禁区：按摩手法（情感劳动，机器无法替代）、创意美妆、高端SPA
- 安全表述："您的屏障功能指数低于基线20%"（不说"您有X病"）
- 必须与持牌互联网医院合作，AI提供"健康建议"，医生做"诊疗确认"
