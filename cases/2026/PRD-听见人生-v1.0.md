## 一、产品概述

**产品名称：** 听·见人生（Living Autobiography Platform）

**一句话定义：** 用AI扮演的传记师，通过剧情对话"骗"出老人一生的故事，实时生成有文学质感的个人自传。

**核心用户：** 60-85岁老年人，以及想给父母留下记忆资产的子女。

**本质是什么：** 老人不需要会写作，不需要懂技术，只需要像聊天一样开口说话，就能拥有一本真正属于自己的回忆录。

## 二、核心机制：剧情引导系统

> 这是整个产品最关键的创新，也是最需要打磨的部分。

我们不直接说"来写自传吧"——这样老人会觉得麻烦、没东西可写、或者不好意思。我们用精心设计的叙事剧情，让讲故事这件事变成一种享受，甚至是一种使命感。

### 三套叙事框架

**框架一：国家档案馆**

AI扮演一位来自"中国民间记忆档案馆"的年轻研究员，正在做"百年口述历史"项目，需要采访亲历过特定年代的普通人。

老人会觉得：自己的经历是历史的一部分，被社会需要，值得被记录。问题方式也变得自然，比如"您那个年代物价是什么样的""您在哪里经历了那段岁月"——老人在不知不觉中讲了自己的故事，而不是觉得在"写作文"。

**框架二：给孙辈的时光信**

AI扮演"未来信使"，帮老人给30年后的孙子孙女写一封时光胶囊信。

这个框架触发的是老人对家族传承的责任感和对孩子的爱。情感投入极深，故事细节往往最丰富、最真实。"您希望孩子知道，您这一生最骄傲的选择是什么"——这类问题能挖出普通问卷永远挖不出的答案。

**框架三：人生侦探**

AI扮演"人生故事侦探"，声称要帮老人找到"被遗忘的人生宝藏时刻"，像解谜游戏一样推进。

"根据您说的线索，我推断您28岁那年一定发生了什么大事……"——这种方式对60-70岁相对活跃的用户效果很好，趣味性强，主动性高。

### 剧情推进规则

- 自传按"童年—青年—成家—事业—暮年感悟"**五章**推进，每章3-8次会话完成。
- AI在老人给出浅层回答后，会用2-3次追问挖掘细节，追问维度包括：具体的人、具体的地点、当时的情绪、感官记忆（气味、声音、天气）。
- 检测到老人提及敏感话题（丧亲、疾病、历史创伤）时，AI主动放缓节奏，给予共情回应，不强迫继续。
- 每次会话结束前埋下**"悬念钩子"**：比如"您刚才提到了一个很重要的人，下次我们来好好聊聊她"——让老人期待下一次。
- 完成一章后，系统生成**该章自传预览**作为成就奖励，看到自己的故事变成了漂亮的文字，是最强的留存动力。

## 三、前端设计方向

### 视觉基调：墨迹美学

不是现代APP的那种干净白色，也不是老气的传统风格。目标感觉是：老旧宣纸、光线穿过窗帘的午后、一支好钢笔——有克制的温度，有岁月的质感，但绝不沉闷。

**设计规则：**
- 色彩：暖米白底色，深墨色主文字，旧金色点缀，**禁止**高饱和色和渐变彩虹
- 字体：正文衬线体，标题手写感字体，参考民国出版物排版气质
- 背景：轻微纸纹和噪点滤镜，**禁止**玻璃拟态和扁平白色
- 动效：慢，文字像被手写出来，**禁止**弹跳和快切
- 布局：大留白，内容居中，像书页排版而不是信息流

### 主界面：书桌

进入产品，用户看到的是一张有质感的老式书桌。

- 左侧："今日的故事"对话入口
- 右侧："我的传记"书本图标
- 中间：手写风格的待续页面，显示上次讲到哪里

整体是一个仪式感很强的"回到书桌"的动作。

### 对话界面：墨水流

- 对话气泡设计为手写纸条风格，AI一侧用墨绿色，老人一侧用暖米色
- 文字以打字机速度逐字出现，让老人感觉到被认真倾听
- 语音输入时，页面中央显示声波动画，像声音变成墨水流入页面
- 每段对话后，右侧书本图标有轻微翻页动效，暗示故事在生长——让老人感受到"我又写了一点"

### 自传预览：书页

- 模拟一本打开的精装书
- 左页：插图（老照片或AI生成的手绘风格插画）
- 右页：文字，排版采用仿宋或楷体，行距宽松
- 每章开头有手绘风格的章节小插图
- 心智目标：**这不是APP界面，这是一本书**

### 输入方式

**文字输入：**
- 大号输入框，字体18pt，行距1.8
- 内置常用句快捷提示："那是……年的事"、"我记得当时……"

**语音输入（重头戏）：**
- 大圆形录音按钮，触感明确
- 录音时实时声波可视化
- 说完后AI复述确认："您刚才说的是……对吗？"
- **必须支持方言**，粤语、闽南语、四川话是第一批优先级

## 四、功能模块

### MVP必做（第一版）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 剧情引导对话系统 | 含三套叙事框架和AI多轮追问逻辑 | P0，产品灵魂 |
| 语音输入 | 普通话+主要方言识别 | P0 |
| 实时成文引擎 | AI将对话内容润色为自传段落，流式输出到书本界面 | P0 |
| 书本预览 | 已完成章节的精排展示 | P0 |
| 用户账户 | 手机号注册，进度云端保存，断点续聊 | P0 |
| PDF导出 | 精装版排版可下载打印 | P1 |

### 第二版迭代

| 功能 | 说明 |
|------|------|
| 子女端只读账号 | 让家人能看到老人的传记进度 |
| 老照片上传 | 与相关故事段落关联展示 |
| AI章节插图 | 根据故事内容生成手绘风格章节插图 |
| 实体书印刷 | 接入按需印刷服务，订购精装实体书 |
| 时间线视图 | 将故事事件按年份可视化呈现 |
| 朗读功能 | AI用合成声音朗读自传（进阶版可用老人本人声音录制） |

## 五、技术架构要点

| 层 | 技术选型 | 备注 |
|---|---------|------|
| 对话引擎 | Claude API + 自定义剧情系统提示 | 核心，不可外包 |
| 语音识别 | 阿里云或讯飞 | 方言模型支持是关键卖点 |
| 前端 | React Native（移动端为主）+ Web端为辅 | 老年用户主要在手机上 |
| 后端 | — | 按需设计 |

### 数据隐私（三条底线，必须在产品内显著呈现）

1. 用户故事数据不用于模型训练
2. 支持一键删除全部数据
3. 子女共享需老人明确授权

## 六、商业化模型

| 产品 | 价格 | 面向用户 |
|------|------|---------|
| 免费版 | ¥0 | 体验3次对话，生成第一章预览 |
| 月度会员 | ¥38/月 | 无限对话，完整成书，语音输入 |
| 家庭年卡 | ¥298/年 | 主账号+3个子女只读账号 |
| 传世礼盒 | ¥599起 | 精装实体书+电子书+二维码朗读 |

### 核心获客逻辑

**第一步：主打"子女送礼"**，而非老人自购。
- "比任何礼物都有意义的孝顺礼品"
- 情感驱动的子女付费决策快，客单价高，传播效果好

**第二波：B端合作**
- 养老院和社区中心：团购授权
- 临终关怀机构：生命叙事疗法版本（单独打磨）

## 七、成功指标

| 指标 | 目标 | 验证的假设 |
|------|------|----------|
| 7日留存率 | ≥40% | 剧情是否真的有粘性 |
| 人均月对话次数 | ≥6次 | 老人是否真的在持续讲故事 |
| 完成一章比例 | ≥60% | 核心价值是否被实现 |
| 付费转化率 | 目标8%，种子期不急 | 产品价值是否被认可 |

---

## 八、核心风险与应对

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| 老人不愿讲某些历史片段 | 中 | 任何话题都可以跳过，不强迫，不记录拒绝行为，让老人有完全掌控感 |
| 方言识别准确率低 | 高 | 引入人工辅助纠错通道；差的识别结果可直接反馈修正；用这些数据优化方言模型 |
| AI成文失真 | 高（信任风险） | 每段AI润色后的内容必须有"这是您说的吗？"确认步骤，老人有权修改或推翻，这一步不能省 |
