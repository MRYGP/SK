# 老年AI管家·技术评估清单 + 迭代路线图

> 日期：2026-03-16
> 状态：待评估
> ⚠️ 重要说明：本文档中的"数字人视频通话"方向来自一次讨论中的灵感闪现，尚未经过真正的反思和推敲。不要把灵感当结论。需要技术评估+用户测试双重验证后才能决定是否走这个方向。

---

# 第一部分：灵感的来源和需要反思的地方

## 灵感

讨论老人日常场景时，创始人提出：
- AI管家应该主动给用户打视频电话
- 以一个真实的AI年轻人形象出现（用户可定制）
- 大部分交互通过视频通话完成
- 必须明确告知是AI（透明原则）

## 为什么这个灵感值得认真对待

- 老人跟微信好友的核心交互是语音和视频，不是文字
- "有脸的人"跟"聊天框里的文字"在情感浓度上差10倍
- 视频通话是老人已经会用的交互方式，零学习成本
- 可定制形象 = 老人对AI管家有情感投射 = 更深的关系

## 需要反思的问题（灵感≠结论）

1. **老人真的想跟一个AI的脸视频聊天吗？** 还是会觉得诡异？必须找真实老人测。
2. **"明确告知是AI"之后，老人还愿意视频聊天吗？** 可能知道是假的就不想看了。也可能"知道是假的但还是喜欢"（就像知道电视剧是演的但还是哭）。不确定。
3. **视频通话是"更好"还是"过度"？** 也许语音+图片就够了，视频是我们以为的好，不是老人要的好。
4. **技术成熟度够不够？** 延迟超过2秒，体验就从"对话"变成"对讲机"，老人会烦。
5. **成本能不能hold住？** 如果每月成本从15元涨到80元，99元会员就不赚钱了。
6. **载体问题：** 微信能不能跑实时视频流？如果必须做独立App，"零门槛"的优势就没了。

**这些问题没有答案之前，不要在产品设计文档里写死"视频通话为主"。**

# 第二部分：技术评估清单

## 评估项1：实时数字人视频通话的技术链

完整链路：
```
老人说话（语音输入）
  → ASR语音识别（语音→文字）
  → 大模型理解+生成回复（文字→文字）
  → TTS语音合成（文字→语音）
  → 数字人驱动（语音→口型/表情/动作）
  → 视频流编码+推送
  → 老人手机上看到数字人在说话
```

需要回答：
- 端到端延迟是多少？（目标：<2秒）
- 哪个环节是瓶颈？
- 国内网络环境下稳定性如何？（老人用的可能是3G/4G）

## 评估项2：可选技术方案对比

| 方案 | 类型 | 需要评估 |
|------|------|---------|
| 硅基智能 | 商用SaaS | 延迟、画质、价格、API对接难度 |
| 商汤数字人 | 商用SaaS | 同上 |
| 腾讯智影 | 商用SaaS（腾讯系，微信对接可能更顺） | 同上 |
| 字节火山引擎 | 商用SaaS | 同上 |
| 阿里通义实验室 | 商用SaaS | 同上 |
| SadTalker/MuseTalk | 开源，可自部署 | 效果、延迟、部署成本 |
| LivePortrait | 开源 | 同上 |

每个方案需要实际跑一次demo，不要只看官网介绍。

## 评估项3：载体/端问题

| 方案 | 能否跑视频流 | 老人安装难度 | 微信生态内 |
|------|------------|------------|-----------|
| 微信小程序 | 待验证（WebRTC支持有限） | 低（微信内打开） | ✅ |
| 微信公众号H5 | 待验证 | 低 | ✅ |
| 独立App | ✅ | 高（要下载安装） | ❌ |
| 企业微信 | 待验证 | 中 | 部分 |
| 微信视频号直播技术 | 待验证 | 低 | ✅ |

**关键问题：有没有一种方式在微信生态内实现实时视频流，同时不需要老人下载任何东西？**

## 评估项4：成本模型

需要拿到真实报价，不要估算。

| 项目 | 需要的数据 |
|------|-----------|
| 数字人视频通话 每分钟成本 | 向3家以上供应商询价 |
| 每天通话10分钟 × 30天 = 月成本 | 计算 |
| 每天通话30分钟 × 30天 = 月成本 | 计算 |
| 对比：纯语音对话的月成本 | 计算 |
| 对比：纯文字对话的月成本 | 计算 |
| 会员定价需要多少才能覆盖？ | 计算 |

## 评估项5：可定制形象

| 问题 | 需要回答 |
|------|---------|
| 预设形象库需要多大？ | 5个？20个？100个？ |
| 用户上传照片生成专属形象可不可行？ | 技术+成本+版权 |
| 生成一个新形象需要多长时间？ | 实时？几分钟？几小时？ |
| 形象的表情丰富度如何？ | 能不能笑、皱眉、点头？ |

## 评估项6："明确告知是AI"的设计

| 方案 | 说明 |
|------|------|
| 视频画面左上角常驻"AI助手"标识 | 最透明，但可能影响沉浸感 |
| 每次通话开始时AI自我介绍 | "张叔您好，我是您的AI管家小丽" |
| 首次使用时明确告知，后续不重复 | 老人知道了就够了 |
| 形象设计本身带AI特征（比如头上有标识） | 视觉提醒，不靠文字 |

需要跟真实老人测试哪种方式接受度最高。

# 第三部分：小步快跑·迭代路线图

**核心原则：不要一步到位做视频通话。每一步都独立验证一个假设，每一步都能独立运行。**

## Step 0：纯文字对话（最小验证）

**验证的假设：** 老人愿不愿意跟AI管家聊天？

做法：
- 微信公众号/企业微信对话
- 老人发文字或语音，AI回文字
- 加上AI创意内容（早安图、写真）
- 每天一条主动问候（文字）

成本：极低（3-5元/月/人）
时间：1-2周开发，1-2周测试
样本：10个老人

**如果老人根本不愿意跟AI聊 → 后面所有步骤都不用做了。**

## Step 1：语音对话（体验升级）

**验证的假设：** 语音交互是否显著提升老人的使用频率和满意度？

做法：
- 在Step 0基础上，AI的回复从文字变成语音
- 老人发语音，AI回语音
- 主动问候也变成语音
- 加上个性化语音（温暖的年轻人声音）

成本：中（5-10元/月/人）
时间：1周升级
样本：同一批10个老人，对比Step 0的数据

**如果语音比文字显著好 → 继续。如果差不多 → 省成本，停在语音。**

## Step 2：数字人短视频（非实时）

**验证的假设：** 老人对"有脸的AI"是什么反应？

做法：
- AI管家的主动问候从语音变成短视频（预生成，不是实时）
- 一个数字人的脸，说着个性化的问候语
- 不是视频通话，是"发来一条视频消息"
- 老人回复还是语音

成本：中（10-20元/月/人，预生成比实时便宜很多）
时间：1-2周开发
样本：同一批老人

**这一步的关键观察：**
- 老人看到数字人的脸，是觉得亲切还是觉得诡异？
- 老人会不会把这个视频转发给朋友？（裂变信号）
- 老人会不会说"能不能跟她/他直接聊天"？（需求验证）

**如果老人觉得诡异 → 退回语音，不做视频。**
**如果老人喜欢并且主动要求"视频聊天" → 进入Step 3。**

## Step 3：数字人实时视频通话（最终形态）

**验证的假设：** 实时视频通话的体验是否值得额外成本？

做法：
- AI管家可以"打视频电话"给老人
- 实时对话，数字人有表情、有口型
- 老人可以选择/定制数字人形象
- 保留语音模式作为备选（网络不好时自动降级）

成本：高（40-80元/月/人）
时间：需要技术评估后才知道
样本：从Step 2中筛选出"最活跃的老人"先试

**只有Step 0-2都验证通过了，才走到这一步。**

## 迭代路线图总结

```
Step 0（纯文字）→ 验证"老人愿不愿意跟AI聊"
        ↓ 愿意
Step 1（语音）  → 验证"语音是否更好"
        ↓ 更好
Step 2（数字人短视频）→ 验证"老人对有脸的AI什么反应"
        ↓ 喜欢
Step 3（实时视频通话）→ 验证"实时视频值不值得额外成本"
        ↓ 值得
正式版本
```

**每一步之间间隔1-2周。** 每一步都有明确的"继续/停止/回退"判断标准。

**任何一步验证失败，就停在上一步。** 停在语音也是一个好产品。停在短视频也是一个好产品。不一定非要走到实时视频通话。

# 第四部分：提醒自己

1. **灵感是灵感，不是结论。** "数字人视频通话"听起来很酷，但老人要不要是另一回事。
2. **问老人，不要猜。** 所有假设都是X级，只有跑了才知道。
3. **每一步都要能独立活。** 如果Step 3做不了，Step 1的语音产品本身也有价值。
4. **成本是硬约束。** 再好的体验，如果成本cover不住，就不是好产品。
5. **"零门槛"是铁律。** 如果视频通话要求老人下载App，那就不做视频通话。
6. **羽毛球原则不变。** 不管是文字、语音还是视频，核心都是"让老人的生活真的变好"。形态是手段，不是目的。
