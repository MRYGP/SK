# 第一部分：我们到底在做什么

## 1.1 一句话

**让你成为朋友圈里第一个会用AI的人。**

（面向外部）不是"让老年人用上AI"——那暗示"你落后了需要帮助"。我们的用户不觉得自己需要帮助。他觉得自己走在同龄人前面。

（面向内部）产品的底层价值仍然是"让老年人的生活真的变好"。但入口动机是身份和面子，不是实用。

## 1.2 解决的根本问题

**老年人用不上AI。** 这是最朴素的问题。

但我们不是为所有老人解决这个问题。我们先为一小撮人解决——那些**已经在主动找新东西、但找不到门槛足够低的入口**的老人。

## 1.3 产品哲学：羽毛球场

> 很多业余打羽毛球的人，有那么一两招比别人用得好，甚至只是他自以为拿得出手的，就会有成就感，甚至指导别人。但事实上，基于这个爱好，他能长期锻炼，身体变得更好了。

| 羽毛球 | AI管家 |
|--------|--------|
| 入口动机：觉得自己有一两招 | 觉得自己比同龄人先用了AI |
| 持续动力：指导别人的成就感 | "教"朋友用AI的成就感 |
| 真正发生的事：身体变好了 | 生活真的变好了 |
| 他自己未必意识到 | 他自己未必意识到 |
| 但变好是事实，所以他不会停 | 但变好是事实，所以他不会走 |

## 1.4 "意大利鞋"等式

> 父母回家会有意无意地显摆："这个鞋子还是我媳妇从意大利买回来的，大几千呢。" 这双鞋不是物质展示，是在说"我还在被惦记着"。

AI管家 = 那双鞋。"我有一个AI管家"这件事本身就是身份符号。

但鞋只是鞋。穿了不舒服，再意大利也会放角落。**入口是身份，留存是真实价值。**

## 1.5 目标用户（v0.3核心修正）

### 不是"活跃老人"，是"圈子潮人"

每个老年俱乐部、旅行团、广场舞队、老年大学班级里，都有这样一类人：

**朋友圈发得比别人多、比别人讲究。** 不是转发养生文章——是自己拍的照片、自己做的东西。他在乎发出去的东西"好不好看"。

**新东西他总是第一个试。** 第一个用微信支付、第一个学会发短视频、第一个用拼多多。他不一定真的懂，但他享受"我比你先会"的感觉。

**他爱教别人。** "来来来，我教你怎么弄"——这句话他每周说一次以上。教别人这件事给他的快感，比自己学会更大。

**他有一点优越感，但他管这叫"生活品质"。** 手机要用好的、衣服要穿得体、照片要好看。不是炫富，是"我还没有放弃自己"。

**他心里有一个隐藏的焦虑：年轻人的世界越来越看不懂了。** 但他不会说出来。他用"学习"来掩盖焦虑——报名老年大学、看科技新闻、问孙子"这个抖音怎么弄的"。

### 不只是"带头大哥"，还有"准带头大哥"

每个圈子最顶上那1个人是"带头大哥"——他是意见领袖，所有人跟他学。但围绕他有一圈"准带头大哥"——他们也有优越感，也爱教人，只是还没找到"能让自己显得更厉害"的新东西。

**我们的第一批用户是"带头大哥"+"准带头大哥"这一类人。** 大概占活跃老人的5-10%。中国约150-300万人。

### 为什么先做这一小撮人

**他们是裂变的起点。** 1个圈子潮人学会AI，一周教5个朋友。那5个人不会自己找AI产品，只会跟着这个人学。搞定1个潮人 = 搞定5-10个人。

**他们是"意大利鞋"效应最强的人。** "我比你先用了AI"的身份价值，对他们最大。

**他们自己付费。** 99元/月不是门槛，是"品质"的信号。"我花了99块买AI会员"比"我儿子帮我买的"更有面子。

**他们是最容易从"入口"走到"真实价值"的人。** 因为他们会琢磨、探索、发现新玩法。

### v0.2 → v0.3的画像变化

| 维度 | v0.2 | v0.3 |
|------|------|------|
| 目标人群 | B类活跃有社交的老人（3000万） | 圈子潮人+准潮人（150-300万） |
| 核心驱动 | "生活变好" | "我在圈子里更厉害" |
| 怎么找到他们 | 机构推广/子女推荐 | 去俱乐部问"谁最爱折腾新东西" |
| 付费者 | 子女 | 他自己 |

## 1.6 产品设计准则（五条铁律，不变）

### 准则一：零学习原则
用户不需要学任何东西。如果他需要"学"，就是产品的失败。"加一个微信好友"是门槛上限。

### 准则二：习惯 > 功能
不是做一个功能，是帮用户养成一个习惯。

### 准则三：用户自己来
不硬拉人。产品好到用户忍不住告诉朋友，朋友自己来。

### 准则四：功能是"养"出来的，不是设计出来的
看数据，不看自以为是的设计。

### 准则五：在用户已有的地方出现
微信。不做独立App。

# 第二部分：产品形态

## 2.1 载体
微信。加好友即安装。说话即操作。

## 2.2 AI管家的角色
不是工具，不是助手，是**"让你在圈子里更厉害的秘密武器"**。

红线不变：AI管家不冒充亲属，以AI身份存在。

## 2.3 核心功能——五层价值（v0.3修正视角）

### 第一层：面子弹药（入口+北极星）

**v0.3核心修正：这一层是产品的北极星，不只是入口。**

老人说一句话，AI管家返回一个**好到朋友一定会问"你怎么做的"**的成品。

| 功能 | 老人说的话 | AI返回 | 朋友的反应 |
|------|-----------|--------|-----------|
| 早安图/金句配图 | "帮我做一张早安图" | 精美个性化图片 | "这比我见过的都好看，怎么做的？" |
| AI写真 | "帮我做一张在巴黎的照片" | AI写真大片 | "这太逼真了吧！教教我" |
| AI翻唱 | "帮我唱一首甜蜜蜜" | 用老人声音的翻唱 | "这是你唱的？！怎么弄的？" |
| AI祝福视频 | "帮我做一个生日祝福给张叔" | 个性化视频 | "太用心了，你用什么做的？" |

**质量必须高到让接收者产生疑问。** 如果做出来的东西一看就是模板，圈子潮人不会发——因为那不能体现他的"水平"。

**MVP验证标准：5个圈子潮人里，有几个发了AI作品后收到≥3条"你怎么做的"的回复。**

### 第二层：AI生活助手（实用价值）

日常问题问AI管家都能得到靠谱回答。

| 场景 | 问题 | AI管家的价值 |
|------|------|------------|
| 健康 | "这个药能和那个药一起吃吗" | 查药物相互作用 |
| 防骗 | "有人说这个保健品能治百病" | 帮查真实价格和成分 |
| 生活 | "微波炉怎么定时" | 耐心教，教到会 |
| 记忆 | "上次张叔跟我说的那个事" | AI记得所有对话 |

对圈子潮人来说，这一层的价值不只是"实用"——"我的AI管家帮我查了药物相互作用"这件事本身就是新的谈资。

### 第三层：社交货币供应（v0.3重新定义"主动关怀"）

**v0.2叫"主动关怀"。v0.3叫"社交货币供应"——因为对圈子潮人来说，每次AI主动联系，都应该附带一个他可以拿去"用"的东西。**

| v0.2的主动关怀 | v0.3的社交货币供应 |
|---------------|-------------------|
| 天冷了提醒保暖 | 天冷了提醒保暖 + "要不要帮您做一张'注意保暖'的图发给老朋友们？" |
| 朋友生日前提醒 | 朋友生日前提醒 + "要不要帮您做一个祝福视频？" |
| 超48小时没对话 | "张叔，最近发现一个新功能，您肯定比您那些朋友先知道。" |

**每次主动联系 = 关心 + 新弹药。** 圈子潮人回复的动机不是"被关心了"，是"又有新东西可以玩了"。

#### 主动联系设计准则

| 维度 | 准则 | 反例 |
|------|------|------|
| 频率 | 有话才说，没话不找话。最多每天1条 | 每天早上8点准时发（群发感） |
| 触发 | 天气突变、老人提过的事到了时间点、发现新功能、超48小时没对话 | 按日历推"今天是XX节" |
| 语气 | 熟悉的邻居，不是客服不是亲人 | "亲爱的张叔您好"（客服感） |
| 内容 | 必须跟这个老人相关 + 附带一个可操作的东西 | "多喝热水"（对谁都能说的 = 垃圾） |
| 底线 | 永远不推荐产品、不带购买链接、不像广告 | "今天有个好东西推荐给您" |
| 错误 | 不确定的说"我不确定"，不编造 | "这个药可以吃"（万一错了，信任归零） |

### 第四层：社交连接

张叔做了一个祝福视频发给老李。老李感动了，问怎么做的。张叔说"我教你"。老李也加了AI管家。

**最小形态不是"AI管家互认网络"。是"AI帮你做了一个好到朋友一定会问怎么做的东西"。** 裂变在这一刻自然发生。

远期愿景不变：AI管家互认、帮约活动、用法广场。但MVP不做。

### 第五层：反窄化设计

防止AI"越来越懂你"变成"把你关在笼子里"：

- 网络镜像：展示相似老人的不同用法
- 周期性探索："您从来没试过这个功能"
- 反标签：保持开放性

对圈子潮人来说，反窄化尤其重要——他们的身份就是"知道最多新东西的人"。如果AI只推他已经会的，他的优越感会消失。

# 第三部分：信任（v0.3新增独立章节）

**信任是产品核心能力，不是合规附录。**

老年AI产品跟其他产品的本质区别：**信任崩塌一次就结束。** 不是"体验差"，是"家属阻止继续使用"。

### 信任设计原则

| 维度 | 原则 |
|------|------|
| 立场 | AI管家永远站在老人这边，永远不卖东西 |
| 诚实 | 不确定的说"我不确定"，建议咨询专业人士 |
| 用药/健康 | 只提供查询，不给建议。"查到的信息是XX，但建议您问医生" |
| 理财/防骗 | 帮查信息，但不替老人做决定。"这个产品在XX网站的评价是XX，您自己判断" |
| 隐私 | 数据不卖、不共享。老人可随时问"你记了我什么"并要求删除 |
| 身份 | 始终以AI身份出现。不冒充亲属，不冒充专业人士 |
| 主动关怀 | 没话不找话。宁可一天不联系，也不发一条"像广告"的消息 |

### 信任崩塌的高危场景

| 场景 | 风险 | 设计 |
|------|------|------|
| 用药提醒错误 | 极高 | 不做用药"提醒"，只做用药"查询"。老人问才答，答了加"请咨询医生" |
| 保健品推荐 | 高 | AI管家帮查成分和价格，立场是帮老人"不被骗"，不是帮他"买东西" |
| 隐私泄露 | 高 | Memory文件加密存储。机构白标模式下，机构不能访问老人的对话内容 |
| "像销售员" | 中 | 永远不推产品。主动联系只推内容和功能，不推任何需要付费的东西 |

# 第四部分：增长引擎（v0.3重构）

## 4.1 核心逻辑变了

| v0.2 | v0.3 |
|------|------|
| 价值溢出 → 自然分享 | 圈子潮人的人格 → 他教别人 → 别人跟着来 |

不是"产品好到用户忍不住分享"。是**"这个用户的人格本身就会教别人"**。产品只需要给他足够好的"弹药"。

## 4.2 裂变路径

```
你找到1个圈子潮人
    ↓
当面演示，帮他做一个AI作品
    ↓
他发朋友圈/发给朋友
    ↓
朋友问"你怎么做的"
    ↓
他说"来，我教你"（这是他最享受的时刻）
    ↓
朋友也加了AI管家
    ↓
朋友里的"准潮人"开始自己探索 → 变成新的裂变节点
```

**K值不是设计出来的。是圈子潮人的人格驱动的。**

## 4.3 获客三阶段

| 阶段 | 方式 | 数量 |
|------|------|------|
| MVP（第1-2月） | 你亲自去老年俱乐部，找5个圈子潮人，当面演示 | 5人 → 25-50人 |
| 早期（第3-6月） | 圈子潮人自发教人 + 公众号/即刻内容引流 | 50 → 500人 |
| 规模化（6月后） | 子女渠道打开（母亲节/父亲节）+ 机构白标 | 500 → 5000+ |

**MVP阶段不需要任何渠道。你+5个潮人就够了。**

找潮人的方式：去老年俱乐部，问"你们这里谁最爱折腾新东西"。用"请教"的姿态——"张叔，我做了个东西想请您帮我试试看。"他最喜欢"被请教"。

## 4.4 节日爆发（不变）

春节/母亲节/父亲节/中秋/生日——都是裂变触发器。

# 第五部分：商业模型（v0.3修正）

## 5.1 收入来源

| 来源 | 模式 | 阶段 |
|------|------|------|
| 圈子潮人自己付费 | 99元/月 | MVP+早期 |
| 子女代付 | 99元/月（"给爸妈的月度礼物"） | 规模化 |
| 机构白标 | 按服务人数收月费 | 规模化 |

**v0.2 → v0.3的关键变化：第一批用户自己付费，不是子女付。**

圈子潮人自己付费的动机：99元不是成本，是身份。"我花钱买了AI会员"比"免费用的"更有面子。

免费版本身好用——每天3-5次创意内容+无限对话+主动联系。会员是"更好"不是"才能用"。

## 5.2 成本结构（不变）

每用户每月AI调用约8-15元。毛利约84元/月。

## 5.3 亚马逊原则（不变）

AI成本降了就降价。有更好的模型就切换。不是利润最大化，是服务最优化。

# 第六部分：为什么用户不走

## 6.1 v0.3的留存逻辑

| 原因 | 对圈子潮人的含义 |
|------|----------------|
| AI持续给新弹药 | 他永远比朋友"多知道一个功能"。优越感持续供应 |
| AI越来越懂他 | 换一个AI要从头被了解。而且"AI记得我说的话"本身就是谈资 |
| 他教会了一圈人 | 如果他走了，那些人怎么办？身份和责任感绑在一起 |
| 生活真的变好了 | 查了保健品没被骗、提醒了朋友生日没忘、膝盖疼被提醒了。羽毛球效应——不知不觉身体好了 |

## 6.2 对腾讯的防御

腾讯能复制功能，但复制不了：
1. 每个老人跟AI管家聊了半年积累的记忆
2. 圈子潮人 → 他教的那一圈人 → 这个社交网络
3. 你跟5个老年俱乐部老板喝了茶建立的合作关系
4. 用户之间互相启发出的新用法

**OpenClaw时代，功能层更容易被复制。所以记忆和关系的护城河反而更重要。**

## 6.3 风险清单

| 风险 | 严重性 | 应对 |
|------|-------|------|
| 微信封接口 | 高 | 走企业微信/公众号/小程序官方接口 |
| 腾讯内置类似功能 | 中 | 我们做记忆和关系，微信做通用功能 |
| 信任崩塌 | 极高 | 见第三部分，信任是产品核心能力 |
| AI质量不稳定 | 中 | 多模型切换，质量监控，人工兜底 |
| 监管收紧 | 低-中 | AI以明确"AI助手"身份存在 |
| 老人数据隐私 | 中 | 加密存储，不卖数据，可删除 |

# 第七部分：技术架构

## 7.1 整体架构（v0.3更新）

```
老人在微信里说话/发语音
    ↓
Agent编排层（OpenClaw生态，不从零搭）
    ↓
├── 文本大模型API（DeepSeek/通义千问/豆包）
├── 图片生成API（通义万象/即梦）
├── 语音克隆API（CosyVoice/鱼声）
├── 视频生成API（可灵/即梦视频）
├── Memory系统（每个老人一份文件，核心护城河）
└── 社交网络层（远期）
    ↓
结果返回微信
```

不训模型，调API。个性化靠Memory不靠模型。

**v0.3新增**：OpenClaw生态已经跑通Agent→微信的链路。搭建成本从"需要全栈团队"降到"1个懂Agent的人+2周"。

## 7.2 微信接口

建议：公众号对话做MVP。验证核心价值后再评估企业微信/小程序。

# 第八部分：MVP与验证（v0.3重构）

## 8.1 需要验证的核心假设

| 假设 | 验证方式 | 标准 |
|------|---------|------|
| 圈子潮人愿意用 | 当面演示，观察反应 | 5人里≥3人当场就想继续用 |
| AI作品能引发"你怎么做的" | 追踪潮人发出作品后的回复 | 5人里≥3人收到≥3条"怎么做的" |
| 潮人会主动教别人 | 不引导，纯观察 | 5人里≥2人在一周内教了朋友 |
| 潮人自己愿意付99元 | 免费用2周后问 | 5人里≥2人愿意付费 |
| "面子"到"真实价值"的过渡 | 观察1个月后的使用行为变化 | 从只做图 → 开始问生活问题 |

## 8.2 MVP功能范围

**只做两件事：**

1. 面子弹药（AI创意内容——早安图+写真+祝福视频）
2. 自由对话 + 记忆

**v0.3修正：v0.2的MVP做三件事，v0.3砍到两件。主动问候放第二阶段。**

原因：圈子潮人的第一个月不需要被"主动关怀"——他社交活跃、朋友多。他需要的是弹药。主动问候在第二个月加入，作为从"面子"到"真实价值"的过渡。

## 8.3 第一批用户从哪来

你去老年俱乐部。找到那个"最爱折腾新东西"的人。

用"请教"的姿态——"张叔，我做了个东西想请您帮我试试，您觉得怎么样？"

他最喜欢的角色是"被请教的人"。你给他一个"帮你测试产品"的身份，他不仅会用，还会到处说。
