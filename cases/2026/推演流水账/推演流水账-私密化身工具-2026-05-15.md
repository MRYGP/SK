# 推演流水账·私密化身工具

## 一、起点

5/15 晚,杨打电话给张(少量头发,不愿意理光头,试过假发但只试一下就走了,觉得不好意思)。张主动说:"如果有 APP,在家试一晚上都不会不好意思"。

这条信息推翻了 5/15 中午"假发 ToC 是手电筒"的判断。原因:
- 之前判断 ToC 弱粘性,基于"用户单次试戴就走"的假设
- 张的真实反馈:用户愿意"试一晚上",这是高粘性场景
- 之前没考虑到"羞耻感"是真实的、能被私密场景解决的、且只有 AI + 在家 + 多轮对话能解的

## 二、从假发到"羞耻型决策"的跳跃

张的核心需求不只是"看到假发戴上的样子",是:
- **私密**(不想在店里被看见试戴)
- **慢**(一晚上)
- **反复**(多次尝试调整)

这三个需求合起来,只有 AI 在家+多轮对话能满足。

把这三个需求抽出来,适用的场景远不止假发:
- 脱发 / 假发 / 发型
- 皮肤 / 痤疮 / 色斑
- 眼镜 / 美瞳
- 整形 / 医美前预演
- 体重 / 体型 / 健身
- 妆容 / 美妆
- 服装 / 穿搭
- 衰老 / 抗衰预演

所有这些场景的共同特征:**羞耻型决策**——用户不想在公开场景被看见决策过程。

## 三、AI 杠杆的体现

之前推演假发时,Claude 用了"两层断层 + 错位母模型",但没考虑"AI 让之前不可能的事变可能"这一层。

AI 在这个场景的真正杠杆:
1. **数据脱敏**:不上传原图,只上传脸型/发型/肤色/五官结构等抽象特征(2024 年后 AI 视觉特征提取技术成熟)
2. **多轮对话调整**:用户说"再短一点,但保留刘海",AI 持续响应(2024 年后多模态 LLM 成熟)
3. **个性化记忆**:用户偏好长期沉淀,迁移成本高(LLM 上下文 + 长期记忆 2024-2025 年成熟)
4. **从工具到顾问**:不是被动响应,是主动建议(Agent 化趋势)

## 四、产品形态

**MVP 第一刀**:脱发 / 假发 / 发型(张电话证明 + 政策无风险 + 商业化最近)

**产品名候选**:试镜 / 造化 / 影 / 换 — 不急定

**形态**:微信小程序

**核心交互**:
- 上传一张照片 → AI 提取脸型/肤色/五官特征 → 删除原图
- 用户:"我想试试中长发,自然一点"
- AI 生成 3 个版本 → 用户选 → 继续微调
- 满意了 → 保存 → 跳转到合作假发商家

**扩展性预埋**(第一天就架构对):
- 化身管理(不是"试假发",是"管理化身")
- 多轮对话(不是上传→输出,是持续对话)
- 数据结构(脱敏特征 + 用户偏好 + 试过方案 = 个人风格档案)
