## 二、v2.0尚未解决的新发现（按重要性排序）

### P0级（必须改）

**1. 飞轮条件②措辞需要修正**

| 来源 | 建议 |
|------|------|
| DeepSeek | 从"体验随规模改善"改为"核心价值随规模强化"，并增加"规模收益性质"检验（递增/恒定/递减） |
| 千问 | 提出"多维度重新定义可能内部冲突"——低价策略可能导致体验恶化 |

**为什么重要：** WeWork满足了"体验随规模改善"的字面意思（更多位置=更方便），但核心价值（社区感）反而随规模稀释。当前措辞让WeWork能"通过"条件②，但它不应该通过。

**修正方案：** 条件②改为"核心价值主张随规模扩大而强化（非稀释）"，并加一个子检验："规模扩大时，这个产品的核心卖点是变强了还是变弱了？"

**2. 遗漏了Gary Hamel的"规则制定者"理论来源**

| 来源 | 发现 |
|------|------|
| Perplexity | Gary Hamel在1996-1998年的"Strategy as Revolution"中已明确定义"rule makers, rule takers, revolutionaries" |

**为什么重要：** 这是Rule概念最直接的理论前身，比Porter的五力模型更贴近。不引用Hamel，学术诚信有问题。

**修正方案：** 在DOC-D022理论溯源表中加入Hamel。

**3. 遗漏了JTBD（Jobs to Be Done）理论来源**

| 来源 | 发现 |
|------|------|
| Grok | R-Redefine的"把问题本身改了"几乎是JTBD的"进步目标"翻版 |
| Perplexity | 也提到了JTBD |

**修正方案：** 在理论溯源中加入JTBD。

### P1级（应该改）

**4. 新增不适用场景：资源垄断型 + 小而美型**

| 来源 | 案例 |
|------|------|
| DeepSeek | LVMH——资源垄断+品牌护城河，RRF完全不适用 |
| DeepSeek | Basecamp——小而美企业，不追求"伟大"，RRF会误判为"必死" |
| Grok | Costco——纯执行力+会员模式，无RRF但持续高增长 |

**修正方案：** 在不适用场景中新增"资源垄断型（奢侈品/矿产）"和"刻意小而美型（不追求规模的利基企业）"。

**5. 飞轮可以被"反转"（质量/信任崩溃时）**

| 来源 | 案例 |
|------|------|
| Perplexity | Friendster——满足飞轮三条件但技术性能糟糕，网络效应反转 |
| DeepSeek | WeWork——"社区感"随规模稀释，反飞轮 |
| Perplexity | 平台失败研究（Cornell）——信任崩溃时飞轮逆转 |

**为什么重要：** 当前v2.0只说"飞轮条件不满足就没飞轮"，没说"飞轮满足后还可能反转"。

**修正方案：** 在"已知局限"中加入"飞轮可反转"的说明——"满足四条件不代表飞轮永远转。技术性能崩溃、信任危机、核心价值稀释都可能让飞轮反转。"

**6. R1（Redefine）和R2（Rule）之间的转化条件不清**

| 来源 | 观点 |
|------|------|
| DeepSeek | 有时因果（Uber），有时独立（特斯拉重新定义了电动车但不制定汽车标准） |
| Grok | 独立而非因果。需要完全不同的能力（洞见 vs 生态构建） |
| Perplexity | 存在三类：A破坏+制定、B破坏不制定、C不破坏但制定 |

**修正方案：** 在DOC-D022"已知局限"中明确说：R1和R2是独立维度，不是因果关系。重新定义问题需要"洞察力"，规则制定需要"生态构建能力"，两者不同，不自动转化。

### P2级（记录待研究）

**7. 是否需要第四维度？分歧很大。**

| AI | 建议的第四维度 |
|----|---------------|
| Claude | D-Dependency |
| Gemini | D-Dependency |
| DeepSeek | T-Timing |
| Grok | D-Dependency |
| Perplexity | D或T |
| 千问 | 无新增维度 |

**当前处理：** v2.0已将依赖度检验放在Rule层内作为子问题。暂不加第四维度，保持框架简洁。但在AI产品拆解中强制做依赖度检验。

**8. L层级扩展建议**

| AI | 建议 |
|----|------|
| DeepSeek | 加L0叙事规则 + L4价值观规则 |
| Grok | 加L4数据/监管规则 |
| 千问 | 加L4政策规则 + L5生态规则 |
| Gemini | 加L0心智/品牌规则 |

**当前处理：** v2.0已在"已知局限"中提到L0和L4的可能性。暂不扩展，等案例验证后再决定。
