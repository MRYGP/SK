# 360-维度跨越-提炼

## 1. 核心结论

1. 360 的本质优势不是技术领先，而是把问题从“杀毒技术竞赛”重定义为“用户电脑体验与安全入口争夺”。
2. 360 的现象级增长来自“免费+小白化+流氓软件清理+强渠道覆盖”的组合，而非单点创新。
3. 360 的上限问题在于长期停留在“规则套利”，未完成从流量与渠道优势向技术与标准输出的跃迁。

## 2. 最有价值的框架/模型

- 三级火箭：基础安全免费 → 入口/分发承接 → 广告/增值变现
- PRDI流程（文中版本）：问题考古 → 六维重构 → 理论对齐与验证 → 商业模式工程化 → 非对称竞争设计 → 执行与迭代
- 规则制定者三重境界：流量规则 → 商业基础设施规则 → 技术标准规则
- 共性公式：Timing + Free/Freemium + Simplicity + Viral Loop

## 3. 可以复用到其他拆解中的规律（现象→本质→可迁移原则）

- 现象：技术不最强却爆发增长  
  本质：问题被重定义后，原有技术优势被降维  
  可迁移原则：先判断“它在解哪道题”，再判断“它比谁更强”。

- 现象：免费策略短期横扫市场  
  本质：不是定价动作，而是价值链重构与非对称变现  
  可迁移原则：评估免费时必须同时验证后端变现闭环是否真实可跑。

- 现象：用户端口碑极强但行业冲突剧烈  
  本质：占据“道德高地+高频刚需入口”会带来快速分发，也会放大对抗成本  
  可迁移原则：拆解时把“增长效率”与“长期关系/监管成本”分开计分。

- 现象：产品长期停在单一增长逻辑  
  本质：成功路径形成组织惯性，抑制第二曲线  
  可迁移原则：每个高速增长模型都要预设“保质期”与“下一次问题重构节点”。

## 4. 与现有案例卡的关联

- 对照 `CASE-002-Gamma`：同样体现“技术不是唯一壁垒”，但 Gamma 的壁垒更偏“产出即分发”，360 更偏“入口与渠道控制”。
- 对照 `CASE-007-Paid`：Paid 在重构的是“价值计费规则”，360 重构的是“安全产品的商业模式”；都属于“先换题再竞争”。
- 对照 `CASE-REF-WHF` / `CASE-REF-PandaCake`：可形成反向比较——前者困于低频与无飞轮，360抓住了高频入口，但在长期价值跃迁上同样暴露结构性上限。
