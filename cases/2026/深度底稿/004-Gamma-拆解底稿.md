## 那一刀确认（Step 1.5 强制卡点）

**新版那一刀（重拆后）**：

"98%的人打开PowerPoint看到空白页就关掉了——不是因为没有想法，是因为他们不知道怎么把想法翻译成别人能读懂的格式。Gamma解决的是这个翻译问题，不是排版问题。"

**对比旧版**：
- 旧版：「最大壁垒不是AI，是每个用户帮它打广告」
- 新版：「最大壁垒不是AI，是Gamma是唯一一个帮98%在空白页上卡住的人，把想法翻译出来的工具」

**竞争对手有没有变？**
- 旧框架：Gamma vs PowerPoint/Keynote/Tome（更好的PPT工具）
- 新框架：Gamma vs「大多数人的想法永远停留在脑子里，因为格式翻译成本太高」
- 新的竞争对手：不是其他演示工具，是**空白页本身**——是那个让98%的人在打开页面后30秒内关掉的心理障碍

**验证：这让PowerPoint看起来在做蠢事了吗？**
→ 是。PowerPoint假设用户能做视觉设计，但98%的用户不能。PowerPoint是为2%的人设计的工具，把98%的人排除在外了。Gamma是第一个承认这件事并修好它的产品。

## Step 0：那个卡住的人

**不是画像，是具体的人：**

他叫什么：32岁，咨询公司顾问，在周三下午3点接到任务——明天上午要给客户做一个提案汇报。

他每天/每周会想到这个问题多少次：每次要汇报前都会想。也许每周1-2次。

**他卡在哪里**：他打开PowerPoint，看到空白页，脑子里有想法，但他不知道：
1. 第一张幻灯片要写什么
2. 要做几页
3. 字体配色要怎么选
4. 图表要怎么放

他不是不会思考，他是不会做视觉翻译。

他花了80%的时间调格式，20%的时间想内容。他发出去的PPT自己也不满意，但他已经没时间了。

**现有的解法为什么不够好**：
- 用模板：模板改起来也很麻烦，还是得自己调
- 找同事帮：欠人情，而且他有的想法说不清楚
- 让设计师做：太贵，而且沟通成本高

**Grant Lee的原话（A级证据，JPMorgan采访，2024）**：
「making slides is a painstaking visual process that's intimidating to a lot of people」

「Our slogan was 'write like a doc, present like a deck'——the insight there was that making slides is a painstaking visual process that's intimidating to a lot of people」

「When you think about the traditional presentation tools like slides, you're often faced with this blank canvas—and the majority of us are not visual designers」

**98%的数据来源（B级）**：
Grant Lee在Generational访谈（2023）中提到：Gamma在AI推出前，98%的用户注册后就离开了（「Well, 2% would and 98% wouldn't」——指能成功从空白页做出PPT的比例）。

这是一个产品近死亡的数据，也是最精确的用户洞察：**98%的人不是不想用，是用不了**。

## 2) 用户画像

**那个卡住的人（Step 0已完成，见上方）**：
32岁顾问，有想法，卡在视觉翻译门槛上，98%的人都是他。

- **目标用户（官方）**：知识工作者、内容创作者、企业用户（B级·Sacra）
- **真实用户（重拆后）**：任何需要把想法翻译成「别人能读」的格式的人，尤其是不会做视觉设计的人
- **用户痛点**：
  - 表层：PPT做起来太麻烦
  - 深层：不会做视觉翻译（没有设计感），导致想法无法有效传播
- **用户获取方式**：Freemium + 产出物品牌露出（「Made with Gamma」） + 微网红营销（1000+个微型KOL）
- **用户留存策略**：文档链接积累（迁移成本）+ Credit-based使用限制（付费门槛）
- **关键指标**：70M用户，600K付费用户，付费转化率约0.86%（B级·Sacra推算）

**40% Fortune 500有Gamma用户**（A级·Sacra，2026）——说明Bottom-up企业渗透已经发生

**这道题是新题还是旧题**：
旧题是「怎么做PPT」，每个人都在解。
Gamma解的新题是「怎么让想法从脑子里走出来进入别人脑子」——这道题没有人系统地解过。

## 3) 产品机制

- **核心功能**：文字/文档输入→AI生成完整演示稿（含结构/排版/配图）；之后可迭代编辑
- **技术架构**：Gamma自研Layout Engine + 第三方LLM API（GPT系列）+ 图像模型
- **新功能（2025年）**：
  - Gamma Agent：AI自动搜索网络、引用来源、生成内容
  - Gamma API：接入Zapier/Make，可程序化生成内容
  - Gamma Imagine：AI图像生成（品牌化视觉资产）
  - 团队功能：品牌一致性管理、多人协作
- **产品迭代节奏**：
  - 2020-2022：传统编辑器（非AI），新格式探索（「card」概念）
  - 2023年3月：AI集成发布，用户数6万→300万（3个月）（A级·JPMorgan，2024）
  - 2025年9月：Gamma 3.0，平台化/API/团队功能
- **核心设计哲学**：「write like a doc, present like a deck」——文档写法+演示呈现

## 4) 商业模式

- **收入来源**：订阅（C端和企业端）
- **定价策略（A级·Sacra，2026）**：
  - Free：有限AI Credits
  - Plus：$8/月
  - Pro：$18/月
  - Ultra：$100/月（高级图像模型+Studio模式）
- **付费转化驱动因素**：Credit用完后必须付费才能继续用AI功能；去除「Made with Gamma」品牌需要付费；自定义域名/密码保护需要付费
- **单位经济推算（C级）**：600K付费用户，假设平均$12/月 = $86M ARR（接近Sacra的$102M估算）
- **CAC**：极低（产出物品牌露出是主要获客渠道，几乎零成本）
- **LTV**：中等（文档积累形成切换成本，但类似工具迁移成本有限）

**B端是不是同一问题的另一面**：

| 检验项 | Gamma情况 |
|------|------|
| C端的不确定感是？ | 「我不知道怎么做视觉翻译」 |
| B端机构的痛点是？ | 「员工的PPT质量参差不齐，品牌一致性难以保证；内容创作效率低」 |
| 两者是否同一问题两面？ | 部分是：员工「不会格式翻译」= 企业「内容品质难标准化」；同一问题不同角度 |
| B端愿意为什么付钱？ | 品牌一致性控制（Ultra/团队版）；自动化内容生成（API+Zapier）|

**结论**：是。但Gamma是Bottom-up渗透B端（个人先用→企业购买团队版），而非直销B端。

## 5) 增长与分发机制

- **主要获客渠道**：产出物品牌露出（Primary）+ 微网红（Secondary）+ SEO（Secondary）
- **分发楔子**：Freemium + AI Credits限制（用了就想付费）
- **留存机制**：
  1. 历史文档积累（用了越多，迁移成本越高）
  2. 分享链接是Web-native（别人打开是动态网页，不是静态PDF）
  3. Credit体系制造使用惯性

**产出物分发机制的精确描述**：

用户做Gamma文档→发给别人→别人打开看到「Made with Gamma」→部分人注册→他们做文档→继续分发

关键特征：**这不是「邀请返利」，是产品核心使用场景自带的分发**。用户发文档不是为了帮Gamma推广，是因为他要开会、要汇报、要分享想法。分发是行为的副产品，不是额外的营销动作。

每天100万+文档被创建（A级·BusinessWire，2025年11月）= 每天100万次免费广告，用户自愿发出。

**微网红策略（B级·Lenny's Podcast，2025）**：
Grant Lee亲自操刀了数月的网红营销，与1000+个微型KOL合作。判断标准：「哪些创作者真的打开了产品再来谈，哪些只是在背稿子」。

## 6) 护城河判断

### 6a) 传统护城河

- **数据壁垒**：有，但有限。Layout训练数据积累有价值，但竞品Canva数据量更大
- **切换成本来源**：已创建文档的迁移（Web-native格式，导出到PPT会失真）；团队版品牌设置
- **技术壁垒**：中等。自研Layout Engine有门槛，但核心AI能力依赖第三方模型
- **平台依赖风险**：依赖OpenAI/Anthropic API；但格式引擎是自有差异化

**AI产品依赖度检验**：
如果OpenAI明天把「文字→演示文稿」内置到Copilot，Gamma还剩什么？
→ 剩：Web-native格式+分发机制+用户历史文档+团队品牌设置
→ 结论：有防御性，但非无懈可击

### 6b) 飞轮检验

**增长循环图**：
```
用户做Gamma文档 → 发给收件人 → 收件人看到「Made with Gamma」 → 部分注册 → 继续创作文档 → 继续分发（回到起点）
```

**飞轮四条件**：

| 条件 | 本产品情况 |
|------|----------|
| ①多边参与 | 创作者+看到文档的人；互相依赖（发出去才有人看） |
| ②核心价值随规模强化 | 是。更多文档在流通 = 品牌渗透度更高；更多数据改善AI质量 |
| ③退出成本递增 | 是。历史文档、分享链接、团队品牌设置积累越多，切换越痛苦 |
| ④单圈正收益 | 是。CAC极低，0.86%付费率×70M用户已支撑$102M ARR |

**飞轮判断**：✅ **真飞轮**（四条件全满足）

## 7) 成功/失败机制

**成功案例。三条核心机制**：

**SM1：空白页问题（最重要）**
机制：98%的用户被空白页阻挡 → AI消除了「怎么开始」这道门槛 → 前30秒体验决定留存
信号：AI推出后3个月用户从6万→300万（A级·JPMorgan，2024）
可复制性：高。任何「开始」成本高的产品都可以用AI降低入门门槛

**SM2：产出物即分发（飞轮核心）**
机制：每次用户分享文档 = 一次不花钱的品牌曝光 → 不需要买量就能持续增长
信号：每天100万+文档被创建（A级·BusinessWire，2025年11月）
可复制性：仅适用于「用户产出物会被别人看到」的场景

**SM3：利润换自由（创始人决策）**
机制：不大量融资 → 盈利 → 拒绝不合适的投资人 → 等到a16z用$2.1B估值来谈
信号：B轮以前团队保持盈利2年以上（A级·BusinessWire，2025年11月）
可复制性：中。需要同时满足「产品能赚钱」和「创始人有定力」两个条件

**最可复制的一条**：SM1——适用于所有「用户在第一步就被吓跑」的产品

**失败对比案例 - Tome（对照）**：
- $81M融资（B级·Crunchbase）
- $3.5M ARR（2024），<2%付费转化率（A级·Sacra，2026）
- 2025年8月被AngelList收购（A级·PitchBook，2025）
- 根本原因：优先用户规模而非付费转化；产出物不带分发机制；走的是「故事叙述」路线而非「想法传达」路线

## 9) 理论映射 + RRF分析链

### 9a) 理论映射

| 理论 | 与本案例关系 |
|------|------------|
| 破坏式创新（DOC-S032）| Gamma是「门槛破坏」变体：不是更便宜，而是「入门更容易」；服务被传统工具排除的98%的非设计师 |
| 从0到1（DOC-S037）| Gamma的「秘密」是：「大多数人的想法因格式翻译成本而无法被传播」——投资人不信，Grant Lee信了 |
| 推演公式（DOC-P002）| 约束条件（小团队+不融资）→唯一解（飞轮必须自转，不能靠买量）→被迫设计出最优分发机制 |

**最核心理论**：从0到1的「秘密理论」。Gamma找到了一个别人都忽视的真实问题，投资人说「最蠢的想法」正是因为他们没看到这个秘密。

### 9b) RRF分析链

**R — 重新定义问题**

| 维度 | 行业原来的定义 | Gamma的新定义 |
|------|--------------|-------------|
| 用户 | 需要做演示的人（2%会视觉设计的） | 任何有想法需要传达的人（98%不会设计的） |
| 价值 | 更好的视觉格式工具 | 消除格式翻译成本，让想法能走出去 |
| 边界 | 演示软件 = PPT/Keynote的竞争 | 演示软件 = 和「想法永远停在脑子里」竞争 |
| 竞争 | 对手是PowerPoint/Canva | 对手是「空白页带来的焦虑和放弃」 |
| 成本 | 高（需要设计技能+时间） | 极低（一句话描述） |

重新定义了**5个维度**→ **范式转移级别**

**检验**：这让PowerPoint看起来在做蠢事吗？
→ 是。PowerPoint服务了2%，把98%的人挡在门外了。

**R — 规则制定**

- 「Made with Gamma」= 控制了传播渠道中的「展示规则」
- Web-native格式（非.pptx）= 在尝试制定新的「文档格式标准」
- Gamma API（2025年9月）= 开始开放平台，吸引第三方构建工作流

**「消失测试」**：Gamma明天消失——40% Fortune 500的文档/API集成会断；普通用户体验大幅倒退

**判断**：**过渡中**——从套利者向规则制定者过渡，API推出是关键转折信号

**RRF综合判断**：

| 维度 | 状态 |
|------|------|
| R-重新定义 | ✅ 5维度转变，范式转移 |
| R-规则制定 | 过渡中（API开放是转折点）|
| F-飞轮 | ✅ 真飞轮 |

**产品段位**：有R有F，R（规则制定）过渡中 → **中等壁垒，有成为伟大公司的基因，规则制定层尚在建立**

## 10-A) 中国机会扫描

**对应的中国用户群**：
与Gamma完全一样的用户：每周需要做汇报/提案/课件的职场人。
估算：中国职场人口约4.5亿，其中频繁做PPT的约1.5亿，能做出质量过关PPT的约5%（750万）。其余1.4亿都是「那个卡住的人」。（C级推断）

**当前中国市场状态**：
- WPS：有AI辅助，但主要是「在旧框架里加AI」——没有解决空白页恐惧
- 美图AI、稿定设计：偏营销物料，不是「想法传达」工具
- 腾讯文档/飞书：协作能力强，演示能力弱
- 直接对标Gamma的（文字→完整可分享演示稿）：**几乎空白**

**差异化切入点**：
1. **微信生态分发版**：中国最强分发介质是微信；设计「用XX制作，点击查看全文」分发钩子，飞轮转速可超Gamma
2. **竖屏演示版**：中国场景大量演示是竖屏（手机观看/小红书/抖音内容），这是Gamma尚未攻克的场景
3. **B端企业内训**：中国企业内训市场约500亿/年，PPT质量是普遍痛点

**时间窗口**：**12-18个月**
**窗口关闭信号**：腾讯/WPS推出「输入一句话→完整演示稿+分发机制」的完整产品

## 11) 可迁移框架提炼（最后一步）

### P1框架升级：「产出物即分发」（完整版）

**框架名称**：P1产出物即分发

**一句话**：
当用户完成核心任务时，他的产出物必然会被别人看到——在那个时刻嵌入品牌，就是一个永续增长引擎，不需要额外的营销预算。

**核心机制**：

Gamma关键设计：每份文档底部出现「Made with Gamma」。这不是水印，因为：
1. 用户发文档是因为要开会/汇报，不是为了推广
2. 收件人看到时，正处于「认为这份文档专业」的正面情绪状态
3. 展示发生在「用户完成了价值」的时刻，不是强迫广告

类比：问卷星的「本问卷由问卷星创建」——填卷人不觉得是广告，只是在做问卷。

**适用边界（重拆核心产出）**：

| 适用条件 | 不适用条件 |
|---------|-----------|
| 用户核心任务 = 「把东西发给别人」（文档/报告/设计/诊断结果/分析）| 用户核心任务 = 「自己用」（健身记录/日记/私人财务）|
| 产出物被收件人看到时，发送者已经证明了产品价值 | 产出物是私密的/不被别人看到的 |
| 品牌露出不干扰收件人使用（角落/底部） | 品牌露出妨碍收件人（水印遮主要内容）|
| 看到产出物的人也有同样的需求 | 看到产出物的人与制作者需求完全不同 |
| Freemium可行（免费版让品牌出现，付费版去掉）| 产品没有免费层 |

**典型适用**：演示文稿工具、问卷工具、设计工具、AI诊断报告、AI分析报告、合同/法律AI工具

**典型不适用**：个人健康追踪、密码管理、个人财务、内部工具（不对外发送产出物）

**对下一个案例（Cactus）的提示**：
Cactus是学术写作AI。用户核心行为 = 写论文 → 论文最终会被提交/发表，有P1机会；但学术规范不接受AI标注工具，P1在Cactus可能不适用，更可能是P3（帮用户练，让写作能力真正提升）。
