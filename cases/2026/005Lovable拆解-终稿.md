# 45个人，8个月做到1亿美金

## Lovable真正厉害的，不是AI写代码

*这是三湘问道第005篇。* *004那篇，我写Gamma。* *它把分发做进了产品。* *这一篇，我想写另一种赢法。*

------

2025年7月，Lovable用45个全职员工， 8个月做到1亿美金ARR。

更猛的是，到2026年2月， 这家公司冲到了4亿美金ARR，团队也只到146人。

很多人看到这个数字， 第一反应是：

AI写代码，太猛了。 程序员工具，又出王炸了。

我觉得不是。

**Lovable真正厉害的，** **不是让程序员更快。** **而是让不会写代码的人，第一次上场。**

这两句话，差很多。

------

## 不是让高手更快，是让门外汉第一次上场

GitHub Copilot、Cursor这批产品， 大多在做一件事：

让会写代码的人，写得更快。

这当然是大市场。 但也是最挤的市场。

Lovable换了一道题。

它盯上的，不只是程序员。 而是那一大群——

有想法， 想做产品， 但不会写代码的人。

创始人、运营、设计师、销售， 甚至一个学生。 他们以前要把想法做出来， 几乎只有两条路：

要么去求人。 要么直接放弃。

Lovable做的，是把第三条路给了他们。

你说一句需求。 它先帮你把东西搭出来。 不一定完美。 但第一次，能跑起来了。

数据也印证了这一点： 超过半数财富500强企业的非技术员工在用它做内部原型。 （B级，创始人公开发言） 一个11岁的孩子用它给学校做了社交网站。

**真正的大机会，** **不是帮会做的人更快做。** **是让不会做的人，第一次做出来。**

------

## 这不是功能创新，这是市场创新

很多人看AI产品， 喜欢盯功能：

模型强不强？ 生成快不快？ 界面顺不顺？

这些都重要。 但还不够。

更值钱的问题是：

**它服务的，** **是不是一群过去根本进不来的人？**

Canva早年为什么能起来？

不是先去帮顶级设计师。 而是让不会设计的人， 也能做出能发出去的图。 Canva官方到今天还在讲这件事： 设计不该只是设计师的领域。

拼多多早年为什么能起来？

也不是先去抢天猫、京东最核心的那批人。 而是先去服务那些原来没被认真服务的人。 IMD当年直接写：e-commerce for the underserved。

Lovable本质上，走的是同一种路。

**最值钱的市场，往往不是现有客户。** **而是过去根本进不来的人。**

**别老盯着高手。** **去找那些一直想做，但一直做不了的人。**

------

## 这也是它和Builder.ai的区别

同样打着"不会写代码也能做软件"的旗号， Builder.ai当年也讲过这个故事。 融资不少，声量也很大。 但到2025年，它还是进了破产程序。（A级，多家媒体报道）

这件事当然不只一个原因。 但有一个更根本的问题：

Builder.ai从来没有真正降低门槛。

它背后不是AI在做事， 是一群程序员在手工写代码，再包装成"AI产品"。（B级，Medium和BBC报道） 门外汉看到的是一个漂亮的入口， 但进去之后，门槛还在那。

如果用003推演公式里的"约束条件"来看—— 当AI能力快速提升的时候， 真正用AI降门槛的Lovable越来越强。 假装用AI的Builder.ai， 成本结构没有任何改善。

**讲一个漂亮故事，** **不等于真的把门外汉带上场了。**

真正的新市场，不是PPT里说出来的。 是你把一群原来进不来的人，真的拉进来了。

------

## 判断一个AI产品是不是大机会

你以后看一个AI产品， 先别急着问：

它能不能让高手再快30%。

先问一句：

**它有没有把门外汉，** **第一次带上场？**

如果有， 这类产品就值得多看一眼。

如果没有， 那它大概率还是在 现有市场里卷功能、卷价格、卷体验。

那也可能是好生意。 但未必是大机会。

------

## 小团队能学什么

004那篇，我用002的财富公式讲了一件事： 小团队要赢，得把增长结构设计进产品。

这一篇我想再补一刀：

**小团队不一定要比巨头更强。** **你也可以先去服务，** **巨头还没认真服务的人。**

不是取巧。 是选战场。

在中国，这个逻辑同样成立。

法律文书、财务报告、课程制作、建筑方案—— 这些今天仍然带着很高门槛的事， 如果有人能用AI把门槛拆掉， 让原来做不了的人也能先做出来， 里面就可能藏着下一个大机会。
