## 二、反例（RRF全满但失败）

> 这些案例证明RRF三条件不是充分条件。飞轮画得再好看，如果单圈经济模型是负的，就是绞肉机。

### Ofo共享单车 — 绞肉机案例

| 维度 | 分析 |
|------|------|
| R-重新定义 | ✅ 从"买自行车/定点桩车"→"随地扫码解锁" |
| R-规则 | ✅ L2级。制定了共享单车的物联网锁标准、城市投放规则 |
| F-飞轮 | ✅（前三条件）铺车越多→找车越快→用户越多→资金越多→铺车越多。但④单圈正收益❌（LTV/CAC持续<1） |
| RRF判断 | R✅ R✅ F❌（绞肉机） — 飞轮转得越快亏得越快。这就是为什么v2.0加入第④条件 |
| 来源 | Gemini红队报告 |
| 状态 | 🔲 待深度拆解（建议加入拆解队列作为失败案例） |

## 三、反例（RRF不满但成功）

> 这些案例证明RRF不是万能框架。在非平台型商业模式中，极致执行力或技术代差也能创造伟大公司。

### Zoom — 极致执行力案例

| 维度 | 分析 |
|------|------|
| R-重新定义 | ❌ 没有改变问题。Skype/WebEx早就在做视频会议。 |
| R-规则 | ❌ 纯套利者。消失测试：用户无缝切换Teams/Google Meet |
| F-飞轮 | ❌ 早期无多边参与，无网络效应（10人用和1000万人用，单个用户音质一样） |
| RRF判断 | R❌ R❌ F❌ — 但Zoom在2021年市值超1000亿。成功原因：极致的产品体验执行力+疫情时机 |
| RRF能否解释 | **不能。** 这说明RRF不适用于"纯执行力驱动"的产品 |
| 来源 | Claude + Gemini红队报告一致引用 |
| 状态 | 🔲 仅做RRF快检 |

| 状态 | 🔲 仅做RRF快检 |
