## 二、按任务找文件

### 任务1：写一篇公众号文章

```
第一步：读 article_template.md（文章结构）
第二步：读 公众号写作指南.md（写作规范）
第三步：用 agent3a_article.md（改写prompt）执行
```

### 任务2：深度拆解一个AI产品

```
第一步：读 评估引擎速查版.md（选评估模式：快速/标准/深度）
第二步：用 agent2_analyzer.md（分析prompt）执行
第三步：输出到 product-teardown-template-v3.0.md 格式
第四步：查 failure_modes.yml（识别失败风险）
第五步：查 evidence_levels.yml（标注证据等级）
```

### 任务3：判断一个产品该复制/工具用/代理/观察

```
读 评估引擎速查版.md → Step 8 四路决策树
```

### 任务4：理解某个洞察背后的理论支撑

```
破坏式创新 → DOC-S032-破坏式创新.md
垄断与秘密 → DOC-S037-从0到1.md
CEO思维对标 → DOC-S047-三大标杆方法论.md
```

### 任务5：了解三智能体工作流程

```
读 智能体工作流-精华速查版.md
```

## 三、每周工作节奏（推荐）

| 时间 | 任务 | 主要文件 |
|------|------|---------|
| 周一（3h） | 收集5-10个AI产品 | 来源：YC / Product Hunt / TechCrunch |
| 周二-周四（每天2h） | 深度拆解2-3个产品 | `评估引擎速查版.md` + `agent2_analyzer.md` |
| 周四下午（1.5h） | 改写成公众号文章 | `article_template.md` + `agent3a_article.md` |
| 周五（2h） | 四路决策 + 下周计划 | `评估引擎速查版.md` Step 8 |

---

## 四、证据与质量底线

所有输出必须遵守：
- 关键断言标注证据等级（A/B/C/X）→ 见 `evidence_levels.yml`
- 每次评估识别至少1个失败模式 → 见 `failure_modes.yml`
- X级证据禁止作为决策依据，直接删除
- 每篇文章必须有1个真实失败案例对比

```

```
