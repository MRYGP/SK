## 二、按任务找文件

### 任务1：产出一个案例拆解PPT

```
第一步：完成深度分析（见任务2）
第二步：提炼到 aichajie-案例模板-v1.0.md（六维度），保存为 cases/2026/CASE-XXX-产品名.md
第三步：基于案例MD 生成 8 张拆解PPT（参照 案例拆解输出标准.md）
第四步：PPT 导出长图，发布
```

### 任务2：深度拆解一个AI产品

```
第一步：读 评估引擎速查版.md（选评估模式：快速/标准/深度）
第二步：用 agent2_analyzer.md（分析prompt）执行
第三步：输出到 product-teardown-template-v3.0.md 格式
第四步：查 failure_modes.yml（识别失败风险）
第五步：查 evidence_levels.yml（标注证据等级）
```

### 任务3：判断一个产品该复制/工具用/代理/观察

```
读 评估引擎速查版.md → Step 8 四路决策树
```

### 任务4：理解某个洞察背后的理论支撑

```
破坏式创新 → DOC-S032-破坏式创新.md
垄断与秘密 → DOC-S037-从0到1.md
CEO思维对标 → DOC-S047-三大标杆方法论.md
```

### 任务5：了解三智能体工作流程

```
读 智能体工作流-精华速查版.md
```

### 任务6：产出讲稿/音频文案（后期启动）

```
第一步：基于 cases/ 下的案例MD
第二步：用 agent3a_article.md（改写prompt）生成讲稿
第三步：参照 article_template.md 调整结构
存储：scripts/2026/CASE-XXX-产品名-讲稿.md
```

## 三、每周工作节奏（推荐）

| 时间 | 任务 | 主要文件 |
|------|------|---------|
| 周一（3h） | 收集5-10个AI产品 | 来源：YC / Product Hunt / TechCrunch |
| 周二-周四（每天2h） | 深度拆解2-3个产品 | `评估引擎速查版.md` + `agent2_analyzer.md` |
| 周四下午（1h） | 生成案例PPT + 导出长图 | `案例拆解输出标准.md` |
| 周五（2h） | 四路决策 + 下周计划 | `评估引擎速查版.md` Step 8 |

---

## 四、证据与质量底线

所有输出必须遵守：
- 关键断言标注证据等级（A/B/C/X）→ 见 `evidence_levels.yml`
- 每次评估识别至少1个失败模式 → 见 `failure_modes.yml`
- X级证据禁止作为决策依据，直接删除
- 每个案例PPT必须有1个真实失败案例对比

## 五、新增案例文件的存放

完成一个产品拆解后，产出双文件：

| 文件 | 格式参照 |
|------|----------|
| `cases/2026/CASE-XXX-产品名.md` | `aichajie-案例模板-v1.0.md`（案例MD，六维度） |
| `cases/2026/CASE-XXX-产品名.pptx` | `案例拆解输出标准.md`（8张拆解PPT） |

命名：CASE-001 起，三位数递增。工作底稿（v3.0 格式）为内部文件，不强制放入 cases。
