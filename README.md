### 为什么写作和读者各有两个

**写作工坊 × 2**：同一份指令，Claude和GPTs各写一版。你选更好的。不同模型的"砍"法不同，交叉出稿提高命中率。

**第一读者 × 2**：两个模型的"读者假设"不完全一样。如果两个都说"这里我差点划走"——真问题。只有一个说——可能是个体偏好。交叉验证更稳。

**深度研究员只有1个**：研究不需要交叉验证，需要的是联网搜索能力。GPTs比Claude Projects更强。

### 日常使用的工作流

```
1. 你在思考Project里拆解产品、做推演 → 输出底稿
2. 把底稿同时丢给两个写作工坊 → 各出一版
3. 你选更好的（或各取所长合并）
4. 把文章同时丢给两个第一读者 → 各给反馈
5. 综合反馈，回写作工坊改 → 改完发布
```

不需要每篇都跑完整流程。赶时间可以只用一个写作+一个读者。但重要的文章（比如001首发）值得跑全流程。
