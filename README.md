## 文件命名规范

### cases/2026/ 文章文件

两类文章，两套逻辑：

**理念型（001-003）**：`NNN-核心概念.md`
```
001-重新定义问题之老年人被看见.md   ← 方法论名称 + 话题锚点
002-财富公式.md                     ← 概念名本身即自识别
003-推演公式.md
```

**案例拆解型（004起）**：`NNN-公司名-核心洞察.md`
```
004-Gamma-产出自然分享.md           ← 公司名在前，洞察在后
005-Lovable-找到还没服务的大众.md
006-Yoodli-让人变得更强.md
007-Paid-按结果收费.md
```

**三条规则：**
- 不加"终稿"——文件夹里都是终稿，无需标注
- 核心洞察写这篇文章真正在说什么，不照搬文章标题
- 案例型：公司名在前，洞察在后，中间用`-`隔开
