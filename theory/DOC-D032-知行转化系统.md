# DOC-D032 知行转化系统

## 一、这次推演真正抓到的东西

这次讨论表面上从两个项目展开:

1. 儿童方向:小火星工程师,让孩子在火星供电任务里边做边学。
2. 电商方向:京东礼品定制培训班课后督学,让新手卖家在真实后台操作中边做边学。

但它们背后的病灶是同一个:

> 学了,不等于会用。知了,不等于能行。

传统学习默认解决的是"知道不知道"的问题。

但真实世界里,大多数失败不是发生在"完全不知道",而是发生在知到行之间:

- 听懂了,但回到任务现场不会操作;
- 会复述,但不知道什么时候该用;
- 看老师做会了,自己独立做不会;
- 在一个场景会了,换个场景又不会;
- 用了一次,但不能稳定复现。

所以这次推演不应该只沉淀为"边学边用"。

"边学边用"只是现象。

更上位的框架应该是:

> **知行转化系统**:不是教人更多知识,而是在真实任务中识别卡点,按需补最小能力包,通过反复行动、反馈和复盘,把"知道"转化为"能做"。

## 二、为什么传统学习根本到不了头

传统学习的默认路径是:

```text
先系统学习 → 做练习 → 以后也许能用
```

这个路径有三个问题:

1. 知识链条太长,大多数人学不到真正要用的地方。
2. 反馈太慢,真正卡住时课程已经结束。
3. 动机太弱,人在没有任务压力时很难吸收抽象知识。

所以用户常常不是"不愿学",而是还没学到能转化为行动的位置,就已经流失。

按需学习反过来:

```text
真实任务 → 先做 → 暴露卡点 → 补最小能力包 → 立刻再做 → 复盘固化
```

它节省的不是几分钟讲课时间,而是把大量"未来也许会用"的低效学习,压缩成"现在必须用"的高密度学习。

## 三、核心命名:任务触发式按需学习

这套机制可以命名为:

> **任务触发式按需学习**

它不是在线课,不是 AI 老师,也不是知识库问答。

它的基本链路是:

```text
任务驱动 → 直觉尝试 → 卡点暴露 → 按需学习 → 立即应用 → 反思固化
```

这里最关键的单位不是知识点,而是卡点。

传统教育问:

> 你该学什么?

新范式问:

> 你现在为什么做不下去了?

这个变化非常大。

因为真正阻止人进步的,通常不是"不知道全部知识",而是某个局部卡点没有被拆开。

## 四、知行转化阶梯

传统教育把路径看成:

```text
不知道 → 知道
```

但真实的知行转化路径至少是:

```text
不知道 → 知道 → 看懂 → 会模仿 → 能独立做 → 能稳定做 → 能迁移
```

可以压缩成五层:

| 层级 | 含义 | 常见断裂 |
|---|---|---|
| 知 | 知道概念 | 能复述,但不一定懂 |
| 懂 | 能解释原理 | 懂道理,但不会操作 |
| 仿 | 照着做 | 老师带着会,自己不会 |
| 行 | 独立完成 | 能做一次,但不稳定 |
| 迁移 | 换场景还能用 | 此场景会,彼场景不会 |

大多数课程只把人带到"知"或"懂"。

但商业价值和能力变化,真正发生在"行"和"迁移"。

所以新教育产品的竞争点不该是"讲得更清楚",而是:

> 能不能把人从知道推到行动。

## 五、四层断裂模型:为什么学了不会用

可以把"学了不会用"拆成四层断裂:

| 层级 | 断裂表现 | 传统教育的幻觉 | 按需学习的解法 |
|---|---|---|---|
| L1 语义层 | 知道词,不懂原理 | 能复述定义就算学会 | 不急着背定义,在任务中让概念浮现 |
| L2 程序层 | 懂原理,不会操作 | 懂了自然会做 | 给当前动作需要的操作方法 |
| L3 情境层 | 会操作,不知何时用 | 做过练习就能迁移 | 在真实情境里触发知识调用 |
| L4 迁移层 | 此处会用,换处不会 | 完成一次就算掌握 | 复盘提取模式,再到新任务验证 |

传统教育主要处理 L1。

任务触发式按需学习的价值,在于把学习直接推到 L2-L3:

- 在操作中补知识;
- 在情境中理解用途;
- 在再次行动中验证是否真的会用。

## 六、六类卡点诊断模型

真正的产品壁垒不是"会讲知识",而是能判断用户为什么卡住。

通用卡点至少有六类:

| 卡点 | 典型表现 | 需要的帮助 |
|---|---|---|
| 概念卡点 | 不知道这个东西是什么意思 | 一个概念 + 一个类比 |
| 操作卡点 | 知道要做,但不知道在哪点、怎么点 | 操作步骤 / 页面指引 |
| 策略卡点 | 有多个选择,不知道选哪条 | 判断框架 / 取舍标准 |
| 迁移卡点 | 听懂了,不会用到当前任务 | 当前场景映射 |
| 心理卡点 | 怕错、不敢提交、不敢继续 | 风险边界 / 小步验证 |
| 反思卡点 | 做错了,不知道错在哪里 | 回看过程 / 对比前后想法 |

过去的教育产品多处理概念卡点。

真正的"用中学"要处理 1-6 类卡点。

## 七、最小能力包

最小能力包不是一节课。

它是完成当前动作所需的最小可用能力。

一个标准最小能力包包含五件东西:

1. 一个概念:这是什么。
2. 一个类比:像什么。
3. 一个判断问题:现在该看哪里。
4. 一个动作步骤:下一步怎么做。
5. 一个即时练习:马上试一下。

可以写成:

```text
概念 + 类比 + 判断 + 动作 + 练习
```

这比"微课"更准确。

微课仍然是课。

最小能力包是行动燃料。

## 八、行动练习飞轮

知到行需要时间,不是因为人笨,而是能力必须在反复行动中被锚定。

真正的练习不是:

```text
学 → 练
```

而是:

```text
做 → 错 → 反馈 → 再做 → 变熟 → 换场景 → 再做
```

可抽象为行动练习飞轮:

```text
真实任务
  ↓
第一次尝试
  ↓
暴露错误
  ↓
即时反馈
  ↓
最小能力包
  ↓
再次尝试
  ↓
复盘固化
  ↓
迁移到新任务
```

AI 最有价值的位置不是讲课,而是:

- 发现错误;
- 降低反馈延迟;
- 生成下一次练习;
- 记录前后变化;
- 提醒用户复盘。

## 九、锚定-漂移-再锚定

学习不是一次性获得,而是动态的锚定循环。

传统学习是:

```text
先锚定(课堂) → 漂移很久(到真实场景) → 锚定丢失(忘了)
```

按需学习是:

```text
进入任务漂移 → 遇阻 → 即时再锚定 → 继续行动
```

这里的关键不是重复看同一节课,而是在不同任务中反复经历:

```text
尝试 → 卡住 → 再锚定 → 再尝试
```

儿童方向里,这表现为:

> 火星供电任务中先接线,失败后补串并联,再回到任务里重新接。

电商方向里,这表现为:

> 新手卖家先上架,卡在标题/详情/客服报价,再补当前页面需要的 SOP。

## 十、AI 是认知外骨骼,不是替代大脑

AI 让这个范式现在成立,不是因为它把课讲得更好,而是因为它可以规模化承担过去真人导师很难承担的工作:

- 观察过程;
- 识别卡点;
- 调取最小能力包;
- 根据任务上下文解释;
- 追踪前后变化;
- 生成过程报告。

可以把时代变化写成:

| 时代 | 学习定义 | 知识位置 | 人的核心能力 |
|---|---|---|---|
| 前 AI 时代 | 把知识装进大脑 | 人脑内存 | 记忆力、理解力 |
| 信息时代 | 知道去哪里找知识 | 搜索硬盘 | 检索力、筛选力 |
| AI 按需时代 | 在情境中调度知识 | AI 外骨骼 | 判断力、调用力、情境感知力 |

关键翻转:

> 传统学习要求人成为知识的容器。按需学习要求人成为知识的情境调度者。

但这里有一条红线:

AI 不能过早给答案。

如果 AI 直接替人思考,用户失去"挣扎-顿悟-再锚定"的过程,能力不会真正长出来。

## 十一、脚手架递减模型

好的按需学习不是一直扶着人走,而是逐渐撤掉辅助。

可以分成四级:

| 阶段 | AI / 系统帮助 | 用户状态 |
|---|---|---|
| 带着做 | 给明确步骤 | 完成第一次动作 |
| 半独立做 | 只给方向提示 | 开始自己判断 |
| 独立做 | 用户自己完成,系统检查 | 能独立交付 |
| 迁移做 | 换场景也能完成 | 形成能力 |

提示也要递减:

1. 第一次:明确步骤。
2. 第二次:只给方向。
3. 第三次:反问用户判断依据。
4. 第四次:让用户先解释,系统再纠偏。

否则产品会从"能力训练"退化成"答案外包"。

## 十二、成长可见性框架

学习产品很难收费,因为家长、老师、老板看不见变化。

所以过程报告不是附属功能,而是商业化入口。

不同场景的报告对象不同:

| 场景 | 谁看报告 | 看什么 |
|---|---|---|
| 儿童教育 | 家长 | 孩子是否完成任务、是否反思、是否坚持 |
| 成人培训 | 学员 / 老师 | 是否完成真实任务、卡在哪、是否突破 |
| 电商培训 | 培训班 | 上架率、任务完成率、进阶课转化 |
| 企业培训 | 主管 | 新人是否能独立处理工作流 |

报告指标不应只看:

- 学习时长;
- 完课率;
- 答题正确率。

更应该看:

- 完成了什么任务;
- 卡在哪里;
- 经过什么提示后突破;
- 是否减少重复错误;
- 是否能迁移到新场景;
- 下一步该练什么。

## 十三、身份重构

学习不只是获得知识,也是改变身份。

一个人从:

> 我是不会做的人。

变成:

> 我是能处理这种任务的人。

这对产品很重要。

儿童方向:

> 我不是听电路课的小孩,我是能修复火星基地的小工程师。

电商方向:

> 我不是听课的新手,我是能独立上架和接待客户的店主。

AI 工具方向:

> 我不是看教程的人,我是能用 AI 完成一个项目的人。

学习的终点不是"我知道了",而是"我能做这件事"。

## 十四、适配边界

不是所有学习都适合任务触发式按需学习。

适合的场景需要七个条件:

1. 有明确任务。
2. 有真实产出。
3. 会遇到可诊断卡点。
4. 学完能马上用。
5. 能反复练习。
6. 有反馈。
7. 有人关心结果并可能付费。

不适合的场景:

- 纯知识欣赏;
- 没有明确任务的兴趣浏览;
- 高风险且不能试错的领域;
- 完全靠长期积累的深理论;
- 没有反馈、没有产出、没有付费方的学习。

## 十五、两个当前项目的映射

### 1. 小火星工程师

映射:

- 任务:火星基地供电。
- 卡点:不懂电路分流、串并联、供电路径。
- 最小能力包:电流像水流 + 串并联区别 + 重新接线。
- 报告对象:家长。
- 核心验证:孩子是否能在卡点后立刻用上能力包。

风险:

- 变成游戏玩具,没有能力变化。
- 变成电路课,丢掉按需学习范式。
- AI 提示过早,剥夺试错。
- 家长报告看不到价值。

### 2. 京东礼品定制课后督学

映射:

- 任务:新手卖家完成开店、上架、客服报价、运营节奏。
- 卡点:不会选类目、写标题、写详情、接待客户、处理报价。
- 最小能力包:SOP + 模板 + 当前后台动作。
- 报告对象:培训班 / 老师 / 学员。
- 核心验证:督学组是否比普通组完成更多真实后台动作。

风险:

- 变成 AI 问答老师,被平台和大模型吞掉。
- 培训班只拿 AI 当噱头,不愿开放真实执行数据。
- 学员不执行,督学也推不动。
- 没有跨班级卡点数据,无法沉淀壁垒。

## 十六、优先验证建议

从商业验证速度看,不建议先押最难的儿童 C 端。

更稳的第一战场是:

> 成人培训 / 职业训练 / AI 工具训练营的课后任务导航系统。

原因:

1. 有现成漏斗,不用从零获客。
2. 有明确结果,任务有没有完成一眼可见。
3. 有付费方,培训班关心完课率、实操率、续费率、进阶课转化。
4. 可以半人工先跑,不用先开发完整系统。
5. 能沉淀通用结构:任务地图、卡点库、能力包、报告系统。

儿童方向可以保留为第二条线:

- 儿童版跑通,提供品牌和长期想象力。
- 成人版跑通,提供现金流入口和更快反馈。

## 十七、最小 MVP

建议先做一个 7 天纸面 / 人工 MVP:

选择一个训练营或技能场景:

- AI 工具训练营;
- 京东开店训练营;
- Cursor / Claude Code 入门训练营;
- 短视频剪辑训练营;
- 小红书运营训练营。

找 10 个学员。

不新增课程,只做三件事:

1. 把课程目标拆成 7 天任务地图。
2. 每天记录学员卡点。
3. 每个卡点只给 1-3 分钟最小能力包,然后让学员继续做。

每天输出:

- 完成了什么;
- 卡在哪里;
- 补了哪个能力包;
- 是否马上用上;
- 明天做什么。

7 天后看四个指标:

1. 任务完成率有没有提高。
2. 学员是否觉得比看课省时间。
3. 老师是否觉得督学成本下降。
4. 培训班是否愿意继续使用或付费。

## 十八、这套框架会怎么死

### 死法 1:重新变成课程

如果产品变成"AI 推荐课程"或"AI 讲知识点",就回到旧范式。

纪律:

> 不到卡点,不讲课;讲完马上用;用不上就不讲。

### 死法 2:任务设计太差

真正难的不是 AI 回答,而是设计一个真实、高频、可完成、能产生结果的任务。

任务不真实,AI 再强也没用。

### 死法 3:AI 自由发挥

AI 不能从零自由生成教学。

更稳的方式是:

- 任务卡;
- 知识卡;
- 操作卡;
- 反馈卡;
- 挑战卡;
- 复盘卡。

AI 负责调度和个性化表达,不负责无约束发挥。

### 死法 4:没有现成漏斗

C 端教育容易死在获客。

早期应优先找:

- 培训班;
- 机构;
- 老师;
- 已有社群;
- 企业内训场景。

### 死法 5:报告没人看

如果家长、老师、老板不看报告,商业闭环会断。

报告不是附属功能,是付费理由。

## 十九、最终收束

这次推演的核心结论:

> 教育的机会不在"AI 老师",而在"任务现场的卡点诊断员"。

过去教育卖课程。

现在应该卖任务推进。

过去学习以知识点为单位。

现在学习应该以卡点为单位。

过去证明价值靠"学完多少节"。

现在证明价值靠:

- 解决了什么问题;
- 完成了什么作品;
- 形成了什么策略;
- 能否迁移到新场景。

所以,"知行转化系统"可以作为 SK 的一个待验证方法论资产:

> AI 的机会不是把课讲得更好,而是把人从知道推到行动。

## 二十、下一步入库规则

这份文档目前只是推演流账,不能直接升级为正式清单。

后续需要至少用三个不同场景验证:

1. 儿童任务式学习:小火星工程师纸面测试。
2. 成人培训课后督学:京东礼品定制 / AI 工具训练营 7 天实验。
3. 企业或软件任务导航:新员工 / 工具学习 / 项目交付场景。

如果三类场景都出现同一结构:

```text
任务 → 卡点 → 最小能力包 → 再行动 → 结果可见
```

并且确实提升任务完成率,再考虑升级到 `core/项目审问清单.md` 或单独沉淀成 SKILL。
