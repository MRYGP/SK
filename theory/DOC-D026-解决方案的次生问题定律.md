## 一、一句话定律

**任何产品解决了一个问题，必然制造新的问题。顶级产品思维不是消灭次生问题，而是为次生问题找到"自然接盘侠"——让新参与者主动消化它，并在消化过程中获得自己的利益。**

---

## 二、为什么这条定律重要（和其他理论的交叉）

### 对 010"好奇心 vs 飞轮"的底层解释

010说："飞轮的本质是'运转一次，下一次成本更低'。"
**本定律回答了这个问题的深层原因**：
- 好奇心型产品 = 次生问题全积累在用户侧 = 每次获客都要重新支付"好奇心消耗成本"
- 飞轮型产品 = 次生问题被系统内参与者消化 = 规模越大越自我强化

**飞轮的本质，就是次生问题被接盘。**

### 对 D018"种树种菜"的底层补全

D018说："种菜要给树输血。"
**本定律指出**：
- 菜地如果次生问题无接盘 → 菜地烂在地里，给不了树任何输血
- 菜地次生问题有接盘 → 菜地才能真正输血

**种菜不是"随便种一块就行"，是"种一块次生问题能闭环的菜"。**

### 对 FM015"无飞轮增长"的病因诊断

FM015说："产品没有飞轮，依赖外部投入。"
**本定律指出**：
- FM015 是**症状**
- "没有给次生问题找到接盘侠" 是**病因**

这是一个因→果关系。FM015只是现象，本定律才是机制。

## 三、核心工具1：次生问题五问

**任何产品立项、设计、评估阶段，必须逐条回答：**

| 编号 | 问题 | 红灯信号 |
|------|------|---------|
| Q1 | 谁为"排泄物"买单？ | 答案是"用户自己"/"我们自己扛"/"社会环境" |
| Q2 | 规模放大10倍后，次生问题是线性还是指数？ | 指数增长 |
| Q3 | 有没有"自然接盘侠"？写出具体身份+主动动机 | 写不出具体身份或主动动机 |
| Q4 | 用户会不会因为次生问题而流失？ | 会（护城河是负的） |
| Q5 | 政策/技术突变后，次生问题会不会变致命？ | 会（结构性脆弱） |

**五问全绿 → 产品有自闭环能力**
**任何一问红灯 → 重新设计，或找新接盘侠**

## 四、核心工具2：矛盾转移闭环图

### 图的结构

```
[用户痛点] → [产品方案] → [次生问题] → [接盘侠?] → [闭环?]
                                            ↓
                        闭环成立：飞轮启动 / 闭环断裂：产品死亡
```

### 判断标准（唯一检验）

> **次生问题的解决方案，同时是某个参与者的利益来源。**

- 骑手"主动抢单"赚顺路外快 → 闭环成立
- 城管"被迫清理"共享单车 → 闭环断裂

### 两个标杆案例

**怪兽充电宝（断裂）**：
手机没电 → 扫码借宝 → 归还难 → **用户自己扛+代理扯皮** → 流失+投诉2.1万条 → 市值蒸发90%

**美团充电宝构想（成立）**：
手机没电 → 扫码借宝 → 归还难 → **骑手顺路带走（主动抢单）** → 用户爽+骑手赚+美团锁 → 飞轮启动

## 五、AI创业者的典型次生问题陷阱

| AI产品 | 解决的A | 制造的B | 接盘侠 |
|--------|---------|---------|--------|
| AI客服 | 人工成本 | 用户答非所问的挫败感 | **无** |
| AI写作 | 写作慢 | 内容同质化+垃圾信息 | **无** |
| AI配图 | 设计成本 | 审美疲劳+侵权风险 | **无** |
| AI陪伴 | 孤独 | 现实关系退化+心理依赖 | **无** |

**几乎所有AI产品立项时只想了"解决什么"，没想"制造什么"。**
这是AI产品"初期惊艳、中期停滞、后期口碑崩"的结构性原因。

## 六、实操金句（贴墙用）

1. **不要做一个没有"接盘侠"的解决方案。**
2. **每次设计功能时，先画"矛盾转移闭环图"，画不出闭环就不开工。**
3. **顶级产品思维不是没有问题，而是问题在系统内闭环消化。**
4. **AI产品降低A成本时，必须想清楚增加的成本B由谁承担；提升A效率时，必须想清楚被压缩的环节C会不会反噬；创造A便利时，必须想清楚制造的垃圾D（数据/认知/关系）谁来清理。**
