# DOC-D030 错位母模型

## 一、母模型

**核心结构**:

> **A 应该在 B 的位置,但实际在 C 的位置——这是错位**
> **A 在 B 的位置但时机/标尺/对象错了——这是错配**

错位是空间性的(在哪),错配是关系性的(配不配)。两者经常同时出现。

## 二、错位/错配的五种典型形态

**形态 1:对象错位**——A 服务的对象不是 A 真正能服务的对象
- 例:海外案例平移陷阱(产品对象是中国用户,但形态是为美国用户设计的)
- 例:创始人错配(创始人能力是 A,但产品需要的能力是 B)

**形态 2:阶段错配**——A 处在错的阶段
- 例:DOC-D027 阶段错配
- 例:小团队用大厂打法 / 大团队做小切口

**形态 3:标尺错配**——A 被用错的标尺评估
- 例:种菜 vs 种树标尺(用种树标尺评估种菜方向 = 全否决)
- 例:产品评估用"终局 10 亿"标尺 vs 用"3-6 个月现金流"标尺

**形态 4:语义错位**——A 说的和 B 听的不在同一层
- 例:产品讲"零代码 APaaS",用户搜"不用编程怎么做系统"
- 例:认知翻译层断层
- 例:供给侧叙事 vs 需求侧实际

**形态 5:身份错位**——A 用 B 的姿势做 C 的事
- 例:创业者节奏 vs 投资人节奏 vs 教练节奏(三种身份节奏不同,混着用 = 都做不好)
- 例:用"判断力变现"的姿势做"创业者"的事

## 四、错位识别的元能力

**"看见错位"的认知姿态是稀缺的**。多数人看世界看的是"对称、规律、共性",少数人看的是"不对称、不一致、错位"。

后者是大投资人/创业者的核心能力:
- 索罗斯:认知和现实的反身性错位
- 巴菲特:价格和价值的错位
- 西蒙斯:市场噪音和真实信号的错位
- 贝索斯:基础设施成本和当前定价的错位
- 马斯克:物理极限和工程惯例的错位

**杨的核心能力**:在 AI 应用方向上识别错位。SK 库 90+ 案例本质上都是错位库:产品做的 vs 用户要的、技术能的 vs 商业能的、海外行的 vs 中国能的。

## 五、错位识别的方法论步骤

任何场景下,识别错位的标准动作:

**步骤 1:问"应该是什么样"**
- 用户应该得到什么(对象层)
- 这个阶段应该做什么(时间层)
- 这个事该用什么标尺(标尺层)
- 这件事该谁来做(身份层)

**步骤 2:问"实际是什么样"**
- 实际给了什么 / 在做什么 / 用了什么标尺 / 谁在做

**步骤 3:量化错位距离**
- 距离 = 应然 - 实然
- 距离极大 → 这是机会(可以做产品填补)或风险(踩坑)
- 距离极小 → 没有错位,不是机会

**步骤 4:判断错位的可逆性**
- 错位是结构性的(基础设施级,几年不会变)→ 创业机会
- 错位是周期性的(短期不平衡,自然回归)→ 投机机会,不创业
- 错位是认知性的(只是没人想到)→ 内容/咨询机会

**步骤 5:选择介入位置**
- 在错位的一端介入(供给侧)→ 改造供给
- 在错位的另一端介入(需求侧)→ 教育需求
- 在错位中间介入(翻译层)→ 做认知翻译产品

## 六、为什么这是元理论而非术层

错位/错配模型**不告诉你具体怎么做**,只告诉你"在哪看"——

- 它是其他所有 SKILL 的元判据(每个 SKILL 都在找某种特定错位)
- 它是评估机会的元过滤(没有错位的方向不值得做)
- 它是判断身份的元工具(我现在的身份和动作错位了吗)

**因此放在道层(theory/)而非术层(core/)**,作为 DOC-D030。

## 八、使用流程

遇到一个新方向 / 新候选 / 新困境时,先过 5 个错位检查:

1. **对象错位?**——这件事服务的对象,真的是它该服务的对象吗?
2. **阶段错配?**——这件事处在它该处的阶段吗?
3. **标尺错配?**——我在用对的标尺评估它吗?
4. **语义错位?**——我说的和受众理解的在同一层吗?
5. **身份错位?**——我用对的身份姿势做这件事吗?

任一触发 → 暂停往前推进,先消除错位再继续。
