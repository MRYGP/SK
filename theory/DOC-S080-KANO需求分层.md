# KANO需求分层：什么是"正确的用户预期"

**作者：松阳 | 三湘问道 | 2026.04**
**分类：术层 / Skill | DOC-S080**

> **SK仓库用途**：精确化"超出用户预期"这句一直在用但从没定义清楚的话。拆解AI产品时的校正工具，判断产品的核心卖点属于哪一层用户需求、时间窗口会怎么关闭。

---

## 一、为什么味道更好的火锅店打不过海底捞

你有没有想过一件事：

海底捞的火锅味道只是中上，但它是中国餐饮品牌的天花板。而很多味道明显更好的火锅店，生意并不好。

这不是一个审美问题，也不是一个运气问题。这是一个**结构性**问题——这些店在错误的战场上比赛。

味道更好的店，在拼什么？在拼用户**明确说得出口**的东西：更香、更辣、更新鲜、更好的肉。

海底捞在拼什么？在拼用户**说不出口甚至没想过**的东西：等位时有人给你做美甲、生日时有人给你拉横幅、一个人吃饭对面放个毛绒玩具。

**这两个东西，不在同一层。**

## 二、KANO的核心切分：用户预期有三层

东京理工大学教授狩野纪昭（Noriaki Kano）1984年提出的 KANO 模型，把"用户需求"按心理机制切成了三层。这个切分是如此精确，以至于当你理解它之后，"用户预期"这四个字再也不会是一个模糊的东西。

### 第一层·基本型需求（Must-be）

用户**默认必须**的东西，但他不会主动说出口。

> "外卖不会迟到"
> "酒店房间干净"
> "打车司机不绕路"

做到了：零分（用户觉得本来就该这样）
做不到：大幅扣分（立刻投诉、差评、再也不用）

**这是最憋屈的战场——你花了巨大成本，只能避免被骂，不能让用户喜欢你。**

### 第二层·期望型需求（One-dimensional）

用户**明确说得出口**、会明确比较的东西。

> "续航更长""配送更快""价格更便宜""屏幕更大"

做多好得多好，是线性关系。

这是绝大多数企业在拼的战场——也是**最容易陷入军备竞赛**的战场。因为用户的期望会持续抬高：今天你送3公里免费，明天别人送5公里，你不跟上就输了；而你跟上的那一刻，你的利润就被吃掉了。

**在期望型需求上卷，你最好的结局是"还活着"。**

### 第三层·兴奋型需求（Attractive）

用户**想不到、说不出、但做出来后会炸裂**的东西。

> 第一次用苹果的两指缩放
> 第一次被海底捞的服务员记住名字
> 第一次被 Gamma 的"输入一句话自动生成PPT"

做到了：满意度指数级飙升，培养死忠粉、口碑传播、零成本获客。
不做：零分（因为用户压根没想过这个需求，没做到也不会生气）。

**这是唯一能产生"溢价 + 护城河"的战场。**

## 三、关键规律：兴奋型会腐烂

这是整个 KANO 模型里**最容易被忽略、但对创业者最致命**的一条规律：

**今天的兴奋型，明天是期望型，后天是基本型。**

- 第一家给你等位美甲的火锅店：兴奋型（哇！）
- 第十家给你等位美甲的火锅店：期望型（理所当然了，没有的扣分）
- 第二十家：基本型（不做就是差评）

苹果的两指缩放，2007年是兴奋型；2012年是期望型；2015年之后，任何触屏设备没有就是 bug——变成基本型。

**这条规律意味着三件事：**

**1. 兴奋型不是一次性动作，是持续机制。** 谁能**持续创造**新的兴奋型，谁才有真正的护城河。海底捞不是"服务好"的护城河，是"持续能发明出让人说哇的小事"的护城河。

**2. "时间窗口"有了底层机制。** 我们在拆解模板第10章一直在问"时间窗口还有多久"。KANO 告诉你窗口关闭的动力是什么——兴奋型向期望型的单向降级。**第一个做的人定义了标准，后来者只能被动跟随**。所以时间窗口 ≈ 这个兴奋点从"哇"降级到"应该有"的时间。

**3. 第一个做的人获得双重收益。** 既吃到了兴奋型阶段的爆发增长，又定义了后来者必须遵守的基础设施。Gamma 定义了"AI生成PPT"的交互标准，后来者哪怕做得更好，只要不这么做就被用户骂"不如Gamma好用"。

## 四、什么是"正确的预期"：三种错法

"超出用户预期"这句话，我们系统指令里一直在写，但从没定义过。KANO 让这句话精确到可以判对错：

**"超出用户预期" = 满足了用户说不出口的那一层兴奋型需求。**

但兴奋型不是你说它是就是。以下是三种最常见的错法——每一种都让创业者自以为在创造，实际上是在消耗资源：

### 错法1：把自己的预期当成用户预期

你觉得重要的功能，用户可能压根不在乎。

这是创业者最常犯的错。你花了三个月打磨的"独创算法""极致性能""优雅架构"——用户看到的只是一个界面。

**识别方法**：你说的兴奋点，必须有**具体的人说出具体的话**。如果你找不到一个真实的用户能用他自己的话描述这个"哇"的瞬间，那它不是兴奋型，只是你的自我感动。

### 错法2：把期望型当成兴奋型

"更快""更便宜""更全"这些，用户嘴上会夸，但心里已经默认你应该做到。

你以为自己在创造惊喜，其实只是在红海里拼命。

**识别方法**：问一句——如果你做的这件事，用户需要**你解释一下**才知道它好在哪，那它大概率是期望型。真正的兴奋型，用户一眼看到就懂，而且会主动告诉朋友。

### 错法3：把无差异型当成兴奋型

这是最惨的一种——你以为的创新点，用户压根不关心，甚至反感。

> 一个视频网站加了"AI 智能摘要"功能，但用户来看视频就是为了看视频，摘要对他毫无意义，还占了界面。
> 一个购物APP加了"社交动态"功能，但用户只想买东西，不想看别人买了啥。

这些功能研发成本巨大，但对用户**做不做没差别**，甚至做了还反感。

**识别方法**：做个 A/B 测试，看有这个功能和没这个功能的用户留存有没有差异。没差异的，不管你觉得多酷，都是无差异型。

## 五、在AI产品拆解中怎么用

每次拆解 AI 产品，在第10章做四路决策之前，加两组检验：

### 检验1·需求分层三问

**第一问：这个产品的核心卖点属于哪一层？**

- 基本型 → 军备竞赛前置条件，做不好就死，做好了也没溢价。**不值得押注，值得观察**
- 期望型 → 红海战场，除非你有结构性成本优势（S032破坏式创新）或B端杠杆（S070），否则会被持续挤压利润。**谨慎复制**
- 兴奋型 → 值得深推。**立刻问窗口**

**第二问：这个兴奋型有没有具体用户的具体话术支撑？**

- 有用户原话引用、截屏、社交媒体自发传播 → 证据等级 A/B，真实存在
- 只有团队自述、媒体稿、融资 PR → 证据等级 C/X，可能是自我感动

**第三问：它和致命变量的关系是什么？**

这个兴奋型，是**直接突破了这个行业所有人都在优化的那个核心变量**，还是只是次级体验优化？
- 突破了致命变量 → 真兴奋（例：Gamma 突破了"PPT制作耗时"）
- 只是次级优化 → 可能是伪兴奋（例：某些AI产品加"更好看的主题皮肤"）

### 检验2·兴奋型降级估算（给"时间窗口"加底层机制）

在第10-A章写"时间窗口"时，不要直接拍脑袋写数字。用这个思路推：

1. **标识这个产品的兴奋点**：用户为什么"哇"？一句话说清。
2. **扫描竞品跟随速度**：已经有多少家在抄这个点？抄到了几成？
3. **估算降级节点**：
   - 还没人抄 → 兴奋型阶段，窗口12-24个月
   - 头部玩家开始抄 → 期望型过渡期，窗口6-12个月
   - 行业普遍有了 → 已降级为期望型，窗口已关闭，进入卷的阶段
   - 底层模型/平台内置化 → 直接降级为基本型（任何新品没有就是bug）

**参考例**：
- Gamma 2022年做"AI一句话生成PPT"是兴奋型；2024年 PPT、飞书妙记都有了类似功能 → 降级为期望型；2026年如果Office内置，直接变成基本型。窗口已经从"兴奋"关到"期望"，只剩下护城河壁垒能不能守住。
- Cursor 的"AI辅助编码"2023年是兴奋型；2024年 Copilot、Windsurf 全面跟进 → 期望型。但 Cursor 用"工作流锁定 + 迁移成本"把窗口延长了（见 DOC-D022 RRF 章的依赖度检验）。

## 六、和仓库已有理论的串用

| 理论 | 串用方式 |
|------|---------|
| **DOC-D023 伪需求陷阱** | 伪需求陷阱筛出"是不是真需求"；KANO 筛出"这个真需求属于哪一层"。串用顺序：先过伪需求三问 → 真需求 → 再过KANO三问 → 兴奋型才值得押注 |
| **DOC-D021 维度跨越** | 维度跨越提供**怎么找到兴奋型**的路径（换到一个全新维度上）；KANO 提供**你找到的是不是真的兴奋型**的验收标准 |
| **DOC-S074 换题思维** | 同上——换题是方法，KANO 是验收 |
| **DOC-S032 破坏式创新** | 破坏式创新的本质，就是在原行业玩家认为无关紧要的维度上，创造被忽视用户的兴奋型需求 |
| **DOC-S037 从0到1** | Thiel 说"垄断的秘密"在哪里——在KANO语言里就是兴奋型需求被首先满足的那个点 |
| **DOC-S076 透过现象看本质** | 识别真实用户需求是KANO分类的前提。用户嘴上说的（期望型）往往不是他真正想要的（兴奋型） |
| **DOC-D022 RRF** | RRF 的 F（飞轮）本质是"持续创造兴奋型需求 → 规模放大 → 更便宜地持续创造 → 继续强化兴奋"的自强化循环 |

## 七、核心洞察金句

1. **"兴奋型需求会腐烂。今天的惊喜，明天是标配，后天是地基。"**
2. **"在期望型需求上卷，你最好的结局是'还活着'。"**
3. **"正确的用户预期，是用户说不出口但会'哇'一声的那一层。嘴上说出来的，都是红海。"**
4. **"时间窗口不是外部时钟，是兴奋型向期望型单向降级的速度。"**
5. **"你以为的兴奋点，必须有具体的人说出具体的话。否则它就是你的自我感动。"**

## 八、在仓库中的位置

**归属判断**：术层（Skill），不是道层（Theory）。

因为 KANO 是一个**可操作的判断工具**，不是一个根本性的世界观。它用于拆解时校准"这个产品的卖点在哪一层"。世界观级的理论（D类）是破坏式创新、财富公式、RRF 等。

**触发场景**：
- 拆解时做第9章理论映射，产品卖点是体验/交互层面 → 调用本文
- 拆解时写第10-A章时间窗口 → 调用本文第五节兴奋型降级估算
- 推演新方向时判断"用户是真哇还是你自以为" → 调用本文第四节三种错法
- 伪需求陷阱筛过之后，下一步判断需求在哪一层 → 调用本文
