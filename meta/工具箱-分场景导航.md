# 工具箱·分场景导航

## 元规则

SK 库有 30+ 个工具/SKILL/理论。**任何具体场景下都不需要全部用上**。
- 同时打开 5 个以上工具 → 工具掩盖判断,变成炫技
- 同时只打开 1 个工具 → 单维分析,容易漏掉关键
- 合理范围:**每个场景 3-5 个核心工具**

## 场景一:种菜阶段做产品(快速产生现金流的小产品)

**核心工具(7 个)**:
0. **SKILL-菜成树检验**(任何候选方向必过的元过滤器,避免做手电筒)——这是新增的强制第一道关
1. SKILL-两层断层产品形态(选方向时用)
2. SKILL-认知翻译层诊断(设计产品形态时用)
3. 项目审问清单 Part A-D(每个方向必过)
4. failure_modes.yml(demo 前必过)
5. DOC-D029 客户采用六因子收敛律(demo 后用户不用时诊断)
6. DOC-D030 错位母模型(贯穿全程的元判据)

**关键纪律**:种菜阶段任何候选必须**先过菜成树五问**,再过其他工具。不过菜成树五问就先动其他工具,等于在手电筒上花精力。

**暂时不用**:推演 SOP 八步法、第二波下场判据、MTP 构思招募法、AI 研究报告评估法、SK 认知操作系统总图、各种道层理论

## 场景二:种树阶段做大产品(12-24 个月长周期)

**核心工具**:
1. SKILL-第二波下场判据
2. SKILL-推演 SOP 八步法
3. 项目审问清单完整版
4. SKILL-AI 研究报告评估法(多 AI 评估)
5. DOC-D027 阶段错配 + DOC-D029 收敛律
6. 三大标杆方法论 + 破坏式创新 + 从 0 到 1

## 场景三:做诊断 / 给别人产品诊断(诊断邮件路径)

**核心工具**:
1. product-teardown-template v4.7 完整版
2. SKILL-真实产品外部体检 SOP
3. 两层断层产品形态(看断层)
4. failure_modes(看可能死法)
5. DOC-D029(看用户为什么不用)
6. DOC-D030 错位母模型

## 场景四:对话/推演时(SK 库自身使用)

**核心工具**:
1. SKILL-坐标系校正判断
2. SKILL-反漂移检查清单
3. SKILL-问题跃迁触发条件(元 SKILL)
4. SKILL-反向 teardown(必要时跳出)
5. 项目审问清单 Part D 身份照镜子

## 场景五:写公众号 / 内容生产

**核心工具**:
1. article_template v2.6(必过)
2. 公众号写作指南
3. 公众号内容生产经验手册(36 条教训)
4. SKILL-MTP 构思招募法(如果是招募文)
5. 第一读者 GPTs 终审

## 场景六:看别人 BP / 朋友带项目来

**核心工具**:
1. 项目审问清单 Part A 10 秒排除
2. failure_modes 反向匹配
3. SKILL-真实产品外部体检 SOP
4. DOC-D027 阶段错配
5. 用 30 秒答不上来的问题逼他重新想

## 维护规则

- 新增工具时,必须标注"它在哪些场景下使用"
- 每月看一次:有没有工具从未被用上?那些工具该归档到 _archive/
- 不要追求"每个场景都用全工具",追求"每个工具都有自己最适合的场景"
