# Claude / CODEX 分工协议

> 版本：v1.0 · 2026-05-28
> 适用：SK 仓库三方协作 —— 杨 + Claude（Desktop+Filesystem）+ CODEX
> 关联文件：`AGENTS.md`（CODEX 宪法）、`meta/CLAUDE系统指令.md`（Claude 宪法）

---

## 一句话

CODEX 管「物理正确性」，Claude 管「认知正确性」，杨当信使 + 最终决策者。

---

## 不可替代分工

| 维度 | CODEX | Claude |
|------|:-----:|:------:|
| git status / 分支 / 未提交改动检查 | ✅ | ❌ |
| 跨文件状态一致性对账（README / 执行总表 / case-index 三方对齐） | ✅ | ❌（容易漏） |
| 绝对路径 / 断链 / 路径硬扫描 | ✅ | ❌ |
| commit 拆分 + commit message | ✅ | ❌ |
| 归档移动 / 重命名 / 批量同步 | ✅ | ❌（不擅长） |
| 仓库一致性巡检（只输出冲突清单，不自行合并） | ✅ | ❌ |
| 那一刀 / 内容洞察 / 理论映射 | ❌（禁止） | ✅ |
| 联网研究 / 案例补料 / 第三方核实 | ❌ | ✅ |
| 影响链分析（改 A 还要同步 B/C/D） | ❌ | ✅ |
| 方向判断 / 否决判断（仅在四条红线触发） | ❌ | ✅ |
| 起草协议 / SKILL / 理论 DOC 正文 | ❌ | ✅ |

**底层逻辑**：
- CODEX 每次重读仓库 → 长对话不漂移，但缺少跨次记忆和影响链直觉
- Claude 长对话有上下文 → 能做认知/影响链/那一刀，但容易在工程细节上漂移
- 互补不重叠

---

## 标准循环

1. **杨提需求**
2. **Claude 分析**：影响链 + 验收标准 + 真相确认表（不替杨拍真相，列冲突让杨勾选）
3. **杨勾选真相** → 把工单贴给 CODEX
4. **CODEX 执行**：按 AGENTS.md 出 diff 报告 + commit message（不自行合并冲突）
5. **杨把报告带回 Claude**
6. **Claude 验收**：内容/逻辑层验收（CODEX 不做的判断）

## 文件类型分线（防互相覆盖）

| 文件类型 | 谁能写 |
|---------|--------|
| 文章正文 / 理论 DOC 正文 / 拆解底稿 / 推演流水账 | Claude |
| README / 执行状态总表 / case-index / case-cards 索引同步 | CODEX |
| 单文件方法论修订（SKILL / 铁律小改） | Claude |
| 重命名 / 归档 / 断链修复 / 批量路径替换 | CODEX |
| 清单草稿一行追加（零摩擦捕获） | Claude（即时低风险） |
| 新建 meta/ 协议或边界文件 | Claude 起草 → 直接落盘 |
| 跨仓库边界（SK ↔ SKGPT ↔ daiyixia）调整 | 杨判断 → CODEX 执行 |

**冲突仲裁**：同一文件需同时改时，按"谁的本职更近"决定，另一方止步。

## 高价值常规任务（2026-05-28 新发现）

### 仓库一致性巡检

- **频率**：每月一次（建议月末）
- **执行者**：CODEX
- **输出**：只列冲突清单，不自行合并
- **后续**：Claude 出合并工单 → 杨拍真相 → CODEX 执行

**触发本任务的源头**：CODEX 5/28 自主扫出 README / 执行状态总表 / case-index 三处口径冲突（011-014 发布状态 / 018 编号归属 / 021 未入索引 / theory 数量），且严守 AGENTS.md「不得自行合并为看起来合理的结论」——这是 CODEX 最强、Claude 最弱的能力。固化它。

## 风险与防护

| 风险 | 防护 |
|------|------|
| 两个 agent 都能写同一工作区 | 文件类型分线 + commit 边界隔离 |
| CODEX 守"只改本次触发的" → 必然留同步缺口 | Claude 当影响链翻译官，主动产同步工单 |
| ChatGPT Project 不会自动同步 GitHub | 改完 SKGPT 仍需手动上传，CODEX 必须提醒 |
| Claude 在长对话中可能漂移 | CODEX 每次重读仓库 → 重大判断让 CODEX 复核物理事实 |
| LLM 产的"审计/检查报告"本身可能误报 | 清单 5/19 教训：CODEX 报告也是被核查对象，不直接信，先读原文件 |
| 同一文件被两个 agent 同时编辑 | 用 commit 时间戳隔离；杨当信使 = 天然串行化 |

---

## 不变量

- **杨是唯一决策者**；CODEX 和 Claude 都不能替杨拍真相
- **AGENTS.md 是 CODEX 的宪法**；CLAUDE系统指令是 Claude 的宪法；两者不互相覆盖
- **同一文件不应被两个 agent 在同一时间窗口编辑**（用 commit 时间戳 + 杨的信使角色隔离）
- **CODEX 不做战略判断**，Claude 不做 git 操作 —— 边界硬性
- **建造 > 否决**适用于 Claude，不适用于 CODEX —— CODEX 是执行层，不参与方向判断

---

## 升级触发

本协议出现以下任一情况时升 v1.1：
- 出现新的高频协作任务（如季度大归档 / 跨仓库批量同步）
- CODEX 或 Claude 任何一方边界被突破造成事故
- 杨发现新的"不可替代分工"维度
- ChatGPT Project / GPTs 工作流变化导致信使路径变化
