## 第一个问题：建几个Studio？

**建一个就够。** 不需要三个。

Google AI Studio的对话是可以新开的。你在同一个Studio里：

- 每天新开一轮对话，粘贴指令A的系统指令，做快速扫描
- 每周一新开一轮对话，粘贴指令B，做深度扫描
- 偶尔需要深挖某个方向时，新开一轮对话，粘贴指令C

指令A/B/C不是三个"应用"，是三个**Prompt模板**。你每次开新对话时选一个粘贴进去就行。

如果Google AI Studio支持保存System Instruction（系统指令预设），你可以把指令A存成默认的，因为用得最多。指令B和指令C用的时候粘贴。

## 第二个问题：URL从哪里来？

这是关键。你不需要每天去找URL。**信号源清单里已经有了。**

但问题是——你清单里的信号源有些有固定URL（比如网站），有些没有（比如Twitter账号、抖音账号）。让我按类型分一下，哪些能直接给Gemini URL，哪些不能：

**能直接给URL的（Gemini可以自己去看）：**

| 信号源               | 固定URL                                                      |
| -------------------- | ------------------------------------------------------------ |
| Ben's Bites          | https://bensbites.com                                        |
| Product Hunt AI      | https://www.producthunt.com/topics/artificial-intelligence   |
| Reddit r/artificial  | https://www.reddit.com/r/artificial/                         |
| Reddit r/Solopreneur | https://www.reddit.com/r/Solopreneur/                        |
| Reddit r/automation  | https://www.reddit.com/r/automation/                         |
| Indie Hackers        | https://www.indiehackers.com                                 |
| The Rundown AI       | https://therundown.ai                                        |
| TLDR AI              | https://tldr.tech/ai                                         |
| Crunchbase AI        | https://www.crunchbase.com/hub/artificial-intelligence-startups |
| IT桔子               | https://www.itjuzi.com                                       |

**不能直接给URL的（需要你手动喂内容）：**

| 信号源          | 为什么不能              | 怎么处理                         |
| --------------- | ----------------------- | -------------------------------- |
| Twitter/X账号   | Gemini访问Twitter不稳定 | 你刷完复制文字粘贴给它           |
