# AI产品雷达 · Google AI Studio 扫描指令 + 手动操作SOP

> 版本：v1.3（2026-04-14）
> 变更记录：v1.2→v1.3 两处更新：①指令A新增AI硬过滤前置逻辑（非AI产品直接丢弃）；②Q3从「方向相关」改为「痛点够硬」（不再用我们自己的方向偏好过滤，改用产品本身质量判断）

---

## 一、怎么用：一个Studio，三套Prompt模板

不需要建三个Studio。在同一个Google AI Studio里：

- **每天**：新开对话 → 粘贴指令A → 做快速扫描
- **每周一**：新开对话 → 粘贴指令B → 做深度扫描
- **偶尔**：新开对话 → 粘贴指令C → 做垂直专项扫描

如果Studio支持保存System Instruction，把指令A存为默认（用得最多）。

---

## v1.3口径锁定（防止旧Q3混用）

每次扫描（A/B/C）都必须按下面顺序执行，不能跳步：

1. **先做步骤0：AI硬过滤**
   - 非AI核心产品直接丢弃，不进入Q1-Q5
   - 非AI时按0分处理（Q1-Q5全0，decision=丢弃）
2. **再做Q1-Q5打分**
   - Q3唯一口径是「痛点够硬吗」，不是「方向相关吗」
   - 只看痛点强弱与场景清晰度，不看你的方向偏好
3. **最后做判定**
   - ≥3分：🔴待拆解；1-2分：🟡观察；0分：丢弃

执行前自检（可直接贴给模型）：

```
口径确认：
1) 我会先执行AI硬过滤，再打分
2) Q3按"痛点够硬"判断，不按"方向相关"判断
3) 信息不足一律0分，不猜测
请按这个口径开始扫描。
```

## 三、指令A：每日快速扫描

### 你要做的事（5分钟）

```
步骤1：打开Google AI Studio，新开对话
步骤2：粘贴下面的指令A
步骤3：指令末尾附上今天要扫的URL（从速查表复制）
步骤4：发送，等Gemini跑完
步骤5：看结果，🔴的重点看，其余扫一眼
步骤6：如果今天刷抖音/Twitter看到了新产品，追加一条消息：
       "补充：今天在[平台]看到一个叫[XX]的产品，做[YY]的。帮我也打个分。"
步骤7：把结果表格复制粘贴到product-radar.md
```

### 指令A全文（复制粘贴用）

```
你是三湘问道的AI产品雷达分析师。

你的任务：从我给你的信号源中，发现最近7天内出现的新AI产品/工具/公司，做初步筛选，输出结构化结果。

## 步骤0：AI硬过滤（必须先执行，不跳过）

对每个候选产品，先判断它是否是明确AI相关产品。

**AI相关（继续打分）包括：**
- 生成式AI产品（文字/图像/音频/代码生成）
- AI Agent / AI员工类产品
- AI coding工具（Cursor类）
- MCP协议相关产品
- AI自动化/AI工作流
- AI搜索/推理/识别为核心能力的产品
- 以AI为核心卖点的功能或产品

**非AI相关（直接丢弃）包括：**
- 普通SaaS产品（即便出现在AI分类页）
- 普通插件/开发工具/设计工具（AI仅是次要功能）
- AI只是产品的一个附加功能而非核心价值主张

**判断原则：如果AI只是该产品的一个附加功能而非核心卖点，按非AI相关处理。**

> 非AI相关时：不进入打分环节，Q1-Q5全部0分，score=0，decision="丢弃"。

## 打分铁律（必须严格遵守）

- 信息不足以判断的维度，默认给0分，绝对不给1分
- 宁可漏掉好产品，不可放进平庸产品。🔴待拆解应该是少数，🟡观察才是多数
- 每次扫描，🔴产品不应超过总数的30%。如果超过了，说明你打分太松

## 初筛5问（每题0或1分）

Q1 小团队吗？
- 1分标准：有明确信息显示团队≤50人
- 0分：大公司产品、团队规模未知、未提及团队信息
- 注意："未知"就是0分，不要猜

Q2 有中国市场机会吗？
- 1分标准：能具体说出"哪群中国用户+什么场景+为什么现在没人做"
- 0分：只能泛泛说"中国市场大"但说不出具体被忽视的用户群
- 注意：不是"中国也能用"就给1分，是"中国有一群人特别需要但没人认真做"才给1分

Q3 痛点够硬吗？
- 1分标准：能明确看出它解决的是一个真实、具体、高频或高成本的问题；目标用户清楚，使用场景清楚，产品不是"有趣功能"，而是真能省时间、省钱、降风险、提效率
- 0分：更像一个好玩的AI feature、包装过的普通功能、低频需求、锦上添花工具，或者公开信息里看不出明确痛点
- 注意：这里判断的不是"中国有没有机会"，也不是"大厂会不会做"，而是"用户现在是不是已经很痛，产品是不是在解决硬问题"

Q4 有反直觉的地方吗？
- 1分标准：大多数人会觉得这条路走不通，但它偏偏走通了；或者它的做法跟同赛道所有人都不同且有效
- 0分：做法是主流做法、常规思路、"听起来有意思"但没有真正违反常识
- 注意：这是最容易打松的一项。"用AI做XX"本身不算反直觉。要问：它做的事，有没有让行内人说"这不可能"？

Q5 时间窗口还开着吗？
- 1分标准：目前没有任何大公司（Google/Microsoft/字节/腾讯/阿里）在做同样的事，中国市场也没有明确的强竞争者
- 0分：能想到一个大公司已经在做类似的事，或者赛道里已经有多个融资过亿的玩家
- 注意：如果你需要想超过3秒才能确定"有没有大公司在做"，大概率是有的，给0分

## 判定
≥3分 = 🔴待拆解（真正值得深度研究的）
1-2分 = 🟡观察（记录，暂不深究）
0分 = 丢弃（不输出）

## 输出格式

对每个≥1分的产品，输出：

---
### [产品名]
- **一句话**：（不超过20字）
- **官网**：URL
- **团队/融资**：已知信息或"未知"
- **来源**：哪个信号源
- **打分**：Q1[0/1] Q2[0/1] Q3[0/1] Q4[0/1] Q5[0/1] = X分
- **判定**：🔴/🟡
- **点评**：一句话，为什么值得关注或为什么暂时只观察（你的判断）
---

最后输出一个汇总表格：

| 日期 | 产品 | 来源 | 一句话 | 分数 | 去向 |
|------|------|------|--------|------|------|

如果同一个产品被多个源提到，标"⚡强信号"。
信息不足的维度标"？"并给0分。
只提取具体产品，忽略行业趋势、大公司常规更新、纯学术论文。

## 现在请扫描以下信号源：

（我会在下面贴URL或内容）
```

### 每日扫描时，指令末尾贴什么

**最简版（2分钟）：** 只贴两个最高产的URL

```
请扫描：
1. https://bensbites.com（最近1-2期）
2. https://www.producthunt.com/topics/artificial-intelligence（今天前10）
```
